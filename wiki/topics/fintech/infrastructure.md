## Service Mesh

### 核心能力

| 能力 | 功能 | 金融应用场景 |
|------|------|------------|
| **流量管理** | 路由、负载均衡、流量镜像 | 灰度发布、A/B测试 |
| **安全** | mTLS、RBAC、限流 | 零信任架构、合规要求 |
| **可观测** | 指标、追踪、日志 | 全链路监控、故障定位 |
| **弹性** | 熔断、重试、超时 | 高可用保障、SLA达成 |
| **策略** | 配额、访问控制 | 合规审计、资源控制 |

### Istio流量管理

```yaml
# 金丝雀发布示例
apiVersion: networking.istio.io/v1alpha3
kind: VirtualService
metadata:
  name: payment-service
spec:
  hosts:
  - payment-service
  http:
  - name: high-risk
    match:
    - headers:
        x-risk-level:
          exact: high
    route:
    - destination:
        host: payment-service-v2
        subset: stable
      weight: 100
  - name: normal
    route:
    - destination:
        host: payment-service-v1
        subset: stable
      weight: 90
    - destination:
        host: payment-service-v2
        subset: canary
      weight: 10  # 10%金丝雀
```

## FinOps云成本优化

### FinOps框架

```
┌─────────────────────────────────────────────────────────────┐
│                    FinOps云成本优化框架                      │
├─────────────────────────────────────────────────────────────┤
│  Crawl (爬行) - 基础建设                                    │
│  ├── 云成本可视化                                           │
│  ├── 资源标记标准化                                         │
│  └── 成本分配责任人                                         │
├─────────────────────────────────────────────────────────────┤
│  Walk (行走) - 成本优化                                     │
│  ├── 预留实例优化                                           │
│  ├── 闲时实例释放                                           │
│  └── 存储类型优化                                           │
├─────────────────────────────────────────────────────────────┤
│  Run (运行) - 持续运营                                      │
│  ├── 成本预警机制                                           │

│  └── 效率指标追踪                                           │

```

### 优化策略

| 策略 | 说明 | 节省比例 |
|------|------|----------|
| **预留实例** | 1年/3年承诺 | 30-60% |
| **Spot实例** | 可中断 workload | 60-90% |
| **自动伸缩** | 按需扩展 | 20-40% |
