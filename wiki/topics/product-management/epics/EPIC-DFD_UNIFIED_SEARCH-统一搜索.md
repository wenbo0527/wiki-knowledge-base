## 1. 产品总览

### 1.1 一句话定位
**跨域数据资产的智能统一搜索入口，让用户一句话就能找到任何数据资产。**

### 1.2 产品定位 & 核心价值

| 维度 | 描述 |
|:---|:---|
| **目标用户** | 数据分析师、数据工程师、业务人员、产品经理 |
| **核心痛点** | 不知道数据在哪分散在哪些系统中、搜索不精准、结果不全面 |
| **解决方案** | 统一搜索入口 + 语义理解 + 跨源聚合 + 智能推荐 |
| **核心价值** | 数据发现效率提升 10 倍，一次搜索覆盖所有数据源 |

### 1.6 术语词典（本体论级别）

| 术语 | 英文 | 定义 | 示例 |
|:---|:---|:---|:---|
| **统一搜索** | Unified Search | 跨数据源的集中搜索服务 | 输入"用户订单"返回所有相关表/API/文件 |
| **搜索意图** | Search Intent | 用户搜索背后的真实目的 | 搜索"销售"可能是想找销售数据表或销售报表 |
| **召回率** | Recall | 相关结果被返回的比例 | 相关文档有100篇，返回80篇，召回率80% |
| **准确率** | Precision | 返回结果中相关的比例 | 返回100篇，80篇相关，准确率80% |
| **同义词** | Synonym | 表达相同含义的不同词语 | "用户"和"客户"、"Customer"和"Client" |
| **停用词** | Stop Words | 无检索意义的常见词 | "的"、"了"、"和"等 |
| **分词** | Tokenization | 将文本切分为可检索单元 | "数据资产"→"数据"+"资产" |
| **向量检索** | Vector Search | 基于语义向量相似度的检索 | 搜索"人物信息"能找到"用户表"（语义相关但字面不匹配）|
| **搜索热度** | Search Popularity | 搜索词或资源的被搜索频次 | 近期"日活"被搜索1000次 |
| **搜索建议** | Search Suggestion | 搜索过程中的智能补全 | 输入"用户"提示"用户订单"、"用户画像"等 |

## 2. 功能结构（按模块归档）

### 2.1 FEATURE-DFD_SEARCH_ENGINE（搜索引擎）

#### 功能入口 & 作用

**入口位置**：
- 全局搜索入口（顶部导航栏）
- 数据资产中心 → 搜索
- 快捷键唤起（`Cmd/Ctrl + K`）

**核心作用**：
- 提供高性能的全文搜索能力
- 支持多语言、模糊匹配、智能纠错
- 实现跨数据源的统一索引
- 提供丰富的搜索语法支持

#### 核心逻辑

```
搜索引擎处理流程：

Step 1: 搜索词预处理
  - 分词处理（中英文混合、特殊字符）
  - 停用词过滤
  - 拼写检查和纠错
  - 大小写归一化

Step 2: 查询改写
  - 同义词扩展
  - 查询扩展（相关词补充）
  - 伪 relevance feedback
  - 意图识别和路由

Step 3: 搜索引擎查询
  - 构建查询 DSL
  - 多索引并行查询
  - 结果合并和去重
  - 评分计算（TF-IDF、BM25）

Step 4: 结果后处理
  - 高亮标记
  - 分页处理
  - 聚合统计
  - 缓存结果
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---:|
| `search.analyzer` | 分词器配置 | `ik_max_word` |
| `search.min_match` | 最小匹配度 | `70%` |
| `search.boost_fields` | 字段权重配置 | `name^3, label^2, description^1` |
| `search.fuzzy.enabled` | 模糊匹配开关 | `true` |
| `search.fuzzy.fuzziness` | 模糊匹配模糊度 | `AUTO` |
| `search.highlight.enabled` | 高亮开关 | `true` |
| `search.highlight.pre_tag` | 高亮前缀 | `<em>` |
| `search.highlight.post_tag` | 高亮后缀 | `</em>` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **中文分词不准确** | "数据资产"被分成"数据"+"资产"，失去整体语义 | 使用专业中文分词器（ik、hanlp），配置行业词典 |
| **搜索结果不相关** | 排序不合理，用户想要的结果排在后面 | 优化排序公式，增加点击反馈权重 |
| **搜索响应慢** | 大数据量下搜索卡顿 | 优化索引结构、增加缓存、使用异步搜索 |
| **同义词缺失** | 业务术语搜不到对应的同义词 | 建立业务同义词库，定期维护更新 |
| **零结果率高** | 用户搜索找不到结果 | 优化搜索建议、提供分类推荐、增加纠错提示 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 数据资产字典 | 上游 | 搜索的数据来源之一 |
| 数据资源目录 | 上游 | 搜索的数据来源之一 |
| Elasticsearch | 依赖 | 搜索引擎技术栈 |
| Neo4j | 依赖 | 血缘关系检索 |
| 向量数据库 | 依赖 | 语义搜索能力 |

### 2.2 FEATURE-DFD_SEARCH_INTELLIGENCE（智能搜索）

#### 功能入口 & 作用

**入口位置**：搜索引擎结果页 → 智能增强区域

**核心作用**：
- 理解用户搜索意图，提供精准结果
- 支持自然语言搜索
- 提供搜索建议和智能补全
- 实现个性化搜索排序

#### 核心逻辑

```
智能搜索处理流程：

Step 1: 意图识别
  - 关键词提取
  - 实体识别（表名、字段、业务术语）
  - 意图分类（精确查找、模糊探索、问答）
  - 置信度评估

Step 2: 语义理解
  - 向量化表示（Embedding）
  - 语义相似度计算
  - 上下文理解
  - 跨语言理解

Step 3: 查询优化
  - 自动纠错
  - 同义词替换
  - 查询扩展
  - 搜索路由

Step 4: 智能排序
  - 相关性评分
  - 质量评分（数据完整度、更新频率）
  - 热度评分（访问量、使用量）
  - 个性化评分（历史行为、收藏）
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---|
| `intelligen.embedding_model` | 向量模型 | `text2vec-base-chinese` |
| `intelligen.similarity_threshold` | 相似度阈值 | `0.75` |
| `intelligen.intent_confidence_threshold` | 意图置信度阈值 | `0.6` |
| `intelligen.suggestion_count` | 建议数量 | `10` |
| `intelligen.personalization_enabled` | 个性化开关 | `true` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **意图识别不准** | 将"销售"识别为销售域而非销售表 | 增加业务词典、引入上下文信息 |
| **向量检索慢** | 语义搜索响应时间过长 | 优化向量索引、使用近似最近邻算法 |
| **个性化效果差** | 推荐结果不准确 | 增加行为数据积累、优化特征工程 |
| **跨语言搜索效果差** | 中英混搜效果不好 | 使用多语言模型、翻译增强 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| FEATURE-DFD_SEARCH_ENGINE | 依赖 | 底层搜索能力 |
| 向量数据库 | 依赖 | Milvus/Pinecone 等 |
| LLM 服务 | 依赖 | 意图识别、语义理解 |

### 2.3 FEATURE-DFD_SEARCH_ANALYTICS（搜索分析）

#### 功能入口 & 作用

**入口位置**：数据资产中心 → 搜索分析

**核心作用**：
- 监控搜索质量核心指标
- 分析用户搜索行为
- 发现搜索问题和优化点
- 指导搜索体验迭代

#### 核心逻辑

```
搜索分析流程：

Step 1: 数据采集
  - 搜索词记录
  - 点击行为记录
  - 结果满意度记录
  - 零结果记录

Step 2: 指标计算
  - 搜索量（UV/PV）
  - 零结果率
  - 平均搜索深度
  - 点击率（CTR）
  - 转化率

Step 3: 问题发现
  - 高频零结果词
  - 高跳出率结果
  - 低点击率结果
  - 用户负反馈

Step 4: 优化建议
  - 新增收录建议
  - 词库优化建议
  - 排序优化建议
  - 展示优化建议
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---|
| `analytics.zero_result_threshold` | 零结果率告警阈值 | `10%` |
| `analytics.ctr_benchmark` | CTR 基准值 | `20%` |
| `analytics.report_interval` | 报告生成周期 | `daily` |
| `analytics.retention_days` | 数据保留天数 | `90` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **数据采集不完整** | 部分搜索行为未记录 | 完善埋点方案、端到端埋点 |
| **指标定义不统一** | 不同系统对"搜索量"定义不同 | 统一指标口径、严格定义 |
| **分析结果无法落地** | 分析出问题了但不知道怎么改 | 建立问题-解决方案映射库 |
| **数据延迟大** | 分析报表 T+1 更新 | 实时流处理、增量计算 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 数据分析平台 | 依赖 | ClickHouse/Flink 等 |

## 4. 数据相关

### 4.1 核心指标口径

| 指标 | 定义 | 计算逻辑 | 目标值 |
|:---|:---|:---|:---:|
| **搜索量** | 搜索请求次数 | `count(search_requests)` | - |
| **搜索用户数** | 发起搜索的去重用户数 | `count(DISTINCT user_id)` | - |
| **零结果率** | 无结果搜索占比 | `zero_result / total * 100%` | < 5% |
| **平均搜索耗时** | 搜索请求到返回的平均时间 | `avg(response_time_ms)` | < 500ms |
| **搜索点击率** | 有点击的结果占比 | `clicked_results / total_results` | > 30% |
| **搜索转化率** | 搜索后有后续行为的用户占比 | `converted_users / searched_users` | > 50% |
| **搜索满意度** | 用户对搜索结果的满意度评分 | `avg(satisfaction_score)` | > 4.0 |

## 5. 规则与配置中心

### 5.1 搜索权限规则

| 规则类型 | 规则内容 | 说明 |
|:---|:---|:---|
| **数据可见性** | 根据用户权限过滤搜索结果 | 敏感数据对无权限用户不可见 |
| **搜索范围** | 不同角色可搜索的数据域不同 | 普通用户只能搜公共数据域 |
| **结果屏蔽** | 屏蔽指定的数据资产 | 临时屏蔽、下线数据等 |

### 5.2 搜索排序规则

```
综合评分公式：

Score = TextRelevance × 0.3 
      + SemanticSimilarity × 0.2 
      + QualityScore × 0.2 
      + PopularityScore × 0.2 
      + Personalization × 0.1

其中：
- TextRelevance: 文本相关性（Elasticsearch BM25）
- SemanticSimilarity: 语义相似度（向量相似度）
- QualityScore: 数据质量评分（完整性、更新频率）
- PopularityScore: 热度评分（访问量、使用量）
- Personalization: 个性化评分（用户历史行为）
```

## 7. 问题库 & FAQ

### 7.1 常见问题排查

#### Q: 搜索无结果怎么办？

**排查步骤**：
1. 检查搜索词是否正确
2. 尝试简化关键词
3. 检查是否有过滤条件限制
4. 确认数据源是否正常同步

**解决方案**：
- 使用模糊搜索
- 移除筛选条件
- 清除搜索历史
- 联系管理员检查索引状态

#### Q: 搜索结果不准确怎么办？

**排查步骤**：
1. 分析搜索词和结果的相关性
2. 检查排序权重配置
3. 查看同类搜索的用户行为

**解决方案**：
- 反馈搜索问题（点选"不相关"）
- 优化搜索词（添加更多限定）
- 提交词库补充申请

#### Q: 搜索响应很慢怎么办？

**排查步骤**：
1. 检查网络延迟
2. 查看 Elasticsearch 状态
3. 检查是否有大规模查询

**解决方案**：
- 刷新页面重试
- 避开高峰期
- 联系管理员检查系统状态

## 8. 方法论沉淀

### 8.1 搜索质量评估方法

**离线评估指标**：
- 准确率（Precision@K）
- 召回率（Recall@K）
- NDCG（Normalized Discounted Cumulative Gain）
- MRR（Mean Reciprocal Rank）

**在线评估指标**：
- 点击率（CTR）
- 转化率（CVR）
- 搜索深度（Average Click Position）
- 零结果率
- 用户满意度

**评估流程**：
```
1. 定期抽样搜索 query
2. 人工标注相关性标签
3. 计算离线指标
4. 对比历史基线
5. 分析问题案例
6. 输出优化建议
```

### 8.2 搜索优化策略

**文本相关性优化**：
1. 字段权重调优（name > label > description）
2. 同义词库扩展
3. 分词器优化
4. 搜索建议补全

**语义相关性优化**：
1. 向量模型选型（text2vec、BERT 等）
2. 向量索引优化（HNSW、IVF）
3. 语义相似度阈值调优

**排序优化**：
1. 特征工程（质量、热度、时效性）
2. LTR 模型训练和更新
3. A/B 测试验证效果

### 8.3 行业对比

| 对比维度 | 通用搜索（Google）| 电商搜索（淘宝）| 数据搜索（我们）|
|:---|:---|:---|:---|
| 核心目标 | 信息获取 | 商品转化 | 数据发现 |
| 排序因素 | 相关性为主 | GMV 导向 | 质量+热度 |
| 个性化程度 | 中等 | 极高 | 中等 |
| 评价指标 | Relevance | CTR + CVR | 找数据效率 |
| 特殊要求 | 实时性高 | 大促峰值 | 权限敏感 |

### 8.4 工具流程

**搜索优化工具链**：
- **日志采集**：Filebeat → Kafka → ClickHouse
- **指标计算**：Flink → Elasticsearch / Doris
- **模型训练**：Spark ML → TensorFlow
- **效果监控**：Grafana + 自定义报表

**搜索配置管理**：
- 同义词库：Confluence 文档 → 自动同步
- 停用词表：Confluence 文档 → 自动同步
- 排序参数：配置中心（Nacos）→ 热更新
