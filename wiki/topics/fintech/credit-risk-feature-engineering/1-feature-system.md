## 整体框架
消费金融风控特征按照业务阶段分为三大类，不同阶段的特征目标、优先级、要求完全不同：
```
┌─────────────────────────────────────────────────────────────┐
│  消费金融风控特征体系                                        │
├──────────────────┬──────────────────┬──────────────────┤
│  贷前准入特征   │  贷中监控特征   │  贷后催收特征   │
│  目标：评估还款能力与意愿，准入、额度、定价  │ 目标：实时监控风险变化，提前预警、额度调整  │ 目标：预测回款概率，制定催收策略  │
│  要求：强区分度、稳定性、可解释  │ 要求：时效性、实时计算能力  │ 要求：催收场景针对性、还款意愿识别  │
└──────────────────┴──────────────────┴──────────────────┘
```
按照特征价值和区分度排序：**征信类特征>关联网络类特征>行为类特征>基础属性类特征**

## 1. 贷前准入特征 ⭐（价值最高，占风控效果的70%）
### 核心目标
评估用户的还款能力和还款意愿，决定是否给用户放款、放款额度、利率定价，是风控的第一道防线，直接决定坏账率水平。
### 特征分类与优先级
| 优先级 | 特征大类 | 典型特征 | IV值范围 | 说明 |
|--------|----------|----------|----------|------|
| **P0（最强特征）** | 征信类 | 人行征信逾期次数、逾期金额、最近逾期时间、信用卡使用率、贷款查询次数、百行征信多头借贷次数、第三方黑名单 | 0.3-1.0 | 区分度最高，是贷前风控的核心，没有征信数据的风控基本靠猜 |
| **P1（强特征）** | 关联网络类 | 关联黑名单用户数量、关联逾期用户数量、同一设备登录用户数、同一手机号关联用户数、交易对手黑名单占比 | 0.2-0.5 | 识别团伙欺诈、马甲欺诈的核心特征，传统特征识别不出来的风险 |
| **P2（中强特征）** | 历史行为类 | 历史借款次数、历史逾期次数、历史最长逾期天数、历史还款率、历史平均借款金额、历史消费频次 | 0.15-0.3 | 老用户的核心特征，历史行为是未来行为的最好预测 |
| **P3（中等特征）** | 还款能力类 | 收入水平、职业类型、工作稳定性、公积金/社保缴纳基数、银行流水月均进账、资产情况（房产/车产） | 0.1-0.2 | 评估用户的还款能力，收入越高还款能力越强 |
| **P4（弱特征）** | 基础属性类 | 年龄、性别、城市等级、学历、婚姻状况、是否有子女、居住稳定性 | 0.05-0.15 | 辅助特征，单独使用区分度低，和其他特征组合有提升 |
| **P5（补充特征）** | 外部数据类 | 运营商数据（在网时长、话费缴纳情况）、电商数据（消费层级、消费品类）、社交数据、位置数据 | 0.03-0.1 | 补充维度，适合没有征信数据的下沉用户 |
### 贷前准入特征要求
1. **强稳定性**：特征分布不会随时间、客群变化出现大幅漂移，避免模型效果下降
2. **可解释性**：符合业务逻辑，满足监管要求，被用户拒贷申诉时可以解释
3. **无数据泄露**：特征开发时不能使用未来数据，避免线下效果好线上差
4. **合规性**：符合《个人信息保护法》要求，不能采集法律法规禁止的信息

## 2. 贷中监控特征
### 核心目标
用户放款后，实时监控用户的风险变化，提前识别逾期风险，做额度调整、冻结、提前催收等操作，减少坏账损失。
### 特征分类与优先级
| 优先级 | 特征大类 | 典型特征 | 说明 |
|--------|----------|----------|------|
| **P0** | 实时交易行为特征 | 近1/3/7天交易金额、交易频次、交易地点异常变化、大额交易占比、夜间交易占比、异地交易占比、非日常消费品类交易占比 | 识别用户是否出现异常消费、套现、风险上升情况 |
| **P1** | 征信变化特征 | 近7/15/30天新增贷款查询次数、新增贷款笔数、新增逾期记录、信用卡使用率大幅上升 | 识别用户是否出现多头借贷、负债突然上升、逾期风险 |
| **P2** | 还款行为特征 | 最近3次还款是否逾期、还款是否按时、是否经常最低还款、是否多次延期还款、还款渠道变化 | 识别用户还款意愿和还款能力变化 |
| **P3** | 活跃度特征 | APP登录频次、查询账单频次、修改信息频次、联系客服频次 | 活跃度大幅下降可能是用户失联、出现风险的信号 |
| **P4** | 外部风险特征 | 是否进入失信被执行人名单、是否有行政处罚、是否关联新的风险事件 | 识别用户的外部风险变化 |
### 贷中监控特征要求
1. **时效性**：支持T+1甚至实时计算，用户行为变化后特征尽快更新
2. **低延迟**：特征查询响应时间<100ms，支持实时决策
3. **阈值预警**：特征超过阈值自动触发风险预警，比如"近7天新增贷款查询>5次"自动标记高风险
### 典型应用场景
- 风险预警：用户特征符合风险规则时自动进入高风险名单，安排催收提前介入
- 额度调整：风险降低的用户提额，风险上升的用户降额、冻结额度
- 差异化运营：低风险用户提供利率优惠、增值服务，高风险用户限制部分功能

## 3. 贷后催收特征
### 核心目标
用户逾期后，预测用户的还款概率、还款意愿、失联风险，制定差异化的催收策略，提升回款率，降低催收成本。
### 特征分类与优先级
| 优先级 | 特征大类 | 典型特征 | 作用 |
|--------|----------|----------|------|
| **P0** | 逾期行为特征 | 逾期天数、历史逾期次数、历史最长逾期天数、历史逾期后还款时长、是否有过失联记录 | 最核心的催收特征，直接预测回款概率 |
| **P1** | 还款能力特征 | 当前收入水平、负债水平、是否有稳定工作、是否有资产 | 评估用户是否有能力还款 |
| **P2** | 还款意愿特征 | 历史还款率、是否主动联系客服说明情况、是否承诺还款、是否部分还款、催收响应率 | 评估用户愿意还款的概率，优先催愿意还款的用户 |
| **P3** | 失联风险特征 | 手机号是否能打通、最近登录APP时间、最近交易时间、紧急联系人是否能联系上、居住地址是否变化 | 识别用户是否失联，失联用户需要采取上门、外访等特殊催收方式 |
| **P4** | 人口属性特征 | 年龄、职业、城市、学历、婚姻状况、是否有子女 | 不同人群适配不同的催收话术和策略 |
### 催收策略适配
| 用户分群 | 特征表现 | 催收策略 | 回款率预期 |
|----------|----------|----------|------------|
| **高价值低风险** | 首次逾期、历史还款记录好、还款能力强 | 短信提醒、人工客服温柔提醒 | 90%+ |
| **中风险** | 逾期<30天、历史有少量逾期、还款意愿尚可 | 电话催收、告知逾期后果 | 60-90% |
| **高风险** | 逾期>30天、历史多次逾期、还款意愿差 | 高频电话催收、联系紧急联系人、委外催收 | 20-60% |
| **失联用户** | 联系不上、没有还款迹象 | 外访、上门催收、法律途径 | <20% |
### 贷后催收特征要求
1. **场景针对性**：特征要和催收场景强相关，比如催收响应率、失联相关特征
2. **动态更新**：催收过程中用户的反馈、还款行为要实时更新特征，调整策略
3. **合规性**：不能使用暴力催收相关的特征，符合催收监管要求

## 特征价值评估标准
### IV值（信息价值）评估
| IV值范围 | 特征价值 | 说明 |
|----------|----------|------|
| <0.03 | 无价值 | 完全没用的特征，直接丢弃 |
| 0.03-0.1 | 弱价值 | 辅助特征，可以和其他特征组合使用 |
| 0.1-0.3 | 中价值 | 不错的特征，可以加入模型 |
| 0.3-0.5 | 强价值 | 非常好的特征，核心特征 |
| >0.5 | 超强价值 | 极强的特征，注意检查是否有数据泄露、是否符合业务逻辑 |
### KS值（科尔莫戈罗夫-斯米尔诺夫检验）
- 衡量特征区分好坏用户的能力，KS值越大区分度越好
- 单特征KS>0.2就是很好的特征
### 稳定性指标PSI（群体稳定性指标）
| PSI值范围 | 稳定性 | 说明 |
|----------|--------|------|
| <0.1 | 非常稳定 | 可以放心使用 |
| 0.1-0.25 | 一般稳定 | 可以使用，需要关注变化 |
| >0.25 | 不稳定 | 特征分布发生大幅漂移，谨慎使用，分析变化原因 |

## 特征建设优先级建议
### 创业公司/小团队
优先做P0-P1级特征：征信类、关联网络类、历史行为类特征，80%的风控效果来自这20%的特征，投入少见效快，不需要做太多复杂特征。
### 中大型公司
在核心特征基础上，补充P2-P4级特征，构建完整的特征体系，做特征平台、特征治理，进一步提升模型效果和稳定性，降低坏账率。
### 监管合规要求高的场景
优先选择可解释性强的特征，避免使用黑盒的复杂特征，符合监管要求，应对用户申诉和监管检查。
