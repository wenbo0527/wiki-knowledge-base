## 1. 产品总览

### 1.1 一句话定位
**为电销团队提供的一站式外呼管理平台，集成客户列表、拨号工具、话术指引、订单处理和数据统计，提升电销效率和转化率。**

### 1.2 产品定位 & 核心价值

| 维度 | 描述 |
|:---|:---|
| **目标用户** | 电销专员、电销主管、客服人员 |
| **核心痛点** | 手工拨号效率低、客户信息分散、话术不统一、数据统计滞后 |
| **解决方案** | 统一工作台 + 智能拨号 + 客户画像 + 实时质检 |
| **核心价值** | 人均日呼量提升 50%，转化率提升 20%，管理效率提升 60% |

## 2. 功能结构（按模块归档）

### 2.1 FEATURE-MKT_SALES_TASK_MGMT（任务管理）

#### 功能入口 & 作用

**入口位置**：
- 电销工作台 → 任务管理
- 管理后台 → 任务配置

**核心作用**：
- 创建和管理外呼任务
- 配置任务规则和分配策略
- 监控任务执行进度
- 任务效果统计和分析

#### 核心逻辑

```
任务管理流程：

Step 1: 任务创建
  - 选择目标人群（从客群中心导入）
  - 设置任务基本信息（名称、时间、优先级）
  - 配置外呼策略（重呼规则、接听时段）
  - 设置分配规则（手动/自动）

Step 2: 任务分配
  - 自动分配（按技能组、负载均衡）
  - 手动分配（主管指定）
  - 员工主动领取（抢单模式）
  - 分配结果通知

Step 3: 任务执行
  - 员工登录工作台查看任务
  - 按优先级依次外呼
  - 实时记录跟进结果
  - 触发重呼（如未接通）

Step 4: 任务结束
  - 任务到期自动关闭
  - 手动终止任务
  - 生成任务报告
  - 数据归档
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---:|
| `task.max_retry` | 最大重呼次数 | `3` |
| `task.retry_interval` | 重呼间隔（分钟）| `30` |
| `task.call_interval` | 两次拨打间隔（秒）| `15` |
| `task.auto_assign` | 是否自动分配 | `true` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **任务分配不均** | 部分员工任务堆积，部分空闲 | 动态负载均衡、实时监控调整 |
| **重复拨打同一客户** | 多任务重叠导致重复呼叫 | 去重校验、黑名单拦截 |
| **任务进度不准确** | 实际执行与系统记录不一致 | 通话状态实时同步、异常校验 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 客群中心 | 上游 | 导入目标客户人群 |
| 呼叫中心 | 依赖 | 外呼通话能力 |
| 订单系统 | 下游 | 订单创建和同步 |

### 2.2 FEATURE-MKT_SALES_DIALER（智能拨号）

#### 功能入口 & 作用

**入口位置**：电销工作台 → 外呼列表

**核心作用**：
- 提供高效的拨号能力
- 支持多种拨号模式
- 空号检测过滤无效号码
- 通话状态实时监控

#### 核心逻辑

```
拨号流程：

Step 1: 号码准备
  - 从任务队列获取客户号码
  - 空号预检测（可选）
  - 号码脱敏处理

Step 2: 拨号执行
  - 点击拨号 / 自动外呼
  - 呼叫中心线路分配
  - 接通后转接坐席
  - 通话开始计时

Step 3: 通话中
  - 展示客户画像
  - 话术指引展示
  - 快捷记录入口
  - 保持/转接功能

Step 4: 通话结束
  - 记录通话结果
  - 填写跟进信息
  - 自动更新客户状态
  - 进入下一通
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---:|
| `dialer.mode` | 拨号模式（预览/自动）| `preview` |
| `dialer.predictive_enabled` | 预测式外呼开关 | `false` |
| `dialer.number_check` | 空号检测开关 | `true` |
| `dialer.call_timeout` | 接通超时时间（秒）| `30` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **拨号失败率高** | 号码空号、停机占比高 | 加强入线号码质量、空号检测过滤 |
| **预测式拨号掉线** | 坐席未就绪就接通 | 预测算法调优、充足坐席保障 |
| **通话质量问题** | 声音延迟、断断续续 | 专线接入、QoS 配置 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 呼叫中心 API | 依赖 | 实际通话能力 |
| 号码检测服务 | 依赖 | 空号检测 |
| 隐私号平台 | 依赖 | 号码隐私保护 |

## 4. 数据相关

### 4.1 核心指标口径

| 指标 | 定义 | 计算逻辑 | 目标值 |
|:---|:---|:---|:---:|
| **外呼量** | 实际拨打的电话数 | `count(outbound_calls)` | - |
| **接通量** | 成功接通的电话数 | `count(answered_calls)` | - |
| **接通率** | 接通量 / 外呼量的比例 | `answered / outbound * 100%` | > 60% |
| **平均通话时长** | 每通电话的平均时长 | `sum(duration) / call_count` | > 3min |
| **转化率** | 达成目标的客户比例 | `converted / contacted * 100%` | > 10% |
| **人均日呼量** | 每人每天平均外呼数 | `total_calls / agent_count / work_days` | > 100 |
| **客户满意度** | 客户对服务的满意度评分 | `avg(satisfaction_score)` | > 4.5 |

### 4.2 埋点事件字典

| 事件名 | 触发时机 | 携带参数 |
|:---|:---|:---|
| `call_start` | 开始外呼 | task_id, customer_id, agent_id |
| `call_connect` | 电话接通 | call_id, wait_time, ring_time |
| `call_end` | 通话结束 | call_id, duration, result |
| `order_create` | 订单创建 | order_id, amount, customer_id |
| `followup_save` | 跟进记录保存 | customer_id, intent_level, note |

## 5. 规则与配置中心

### 5.1 意向等级定义

| 等级 | 描述 | 跟进策略 |
|:---|:---|:---|
| **A** | 高意向，明确购买需求 | 当天多次跟进，尽快促成 |
| **B** | 中等意向，有购买可能 | 次日跟进，持续培育 |
| **C** | 低意向，仅了解阶段 | 定期关怀，等待激活 |
| **D** | 无意向，拒绝或无需求 | 标记黑名单，减少重复打扰 |

### 5.2 外呼时间规则

```
可外呼时间：09:00 - 21:00（工作日）
可外呼时间：10:00 - 18:00（周末）
禁止外呼时间：21:00 - 次日 09:00

黑名单拦截：
- 用户明确拒接后加入黑名单
- 连续 3 次未接自动进入冷却期（7 天）
- 用户投诉后加入永久黑名单
```

## 7. 问题库 & FAQ

### 7.1 常见问题排查

#### Q: 拨号失败怎么处理？

**排查步骤**：
1. 检查网络连接状态
2. 确认坐席是否在线
3. 查看呼叫中心服务状态

**解决方案**：
- 刷新页面重试
- 重新登录坐席
- 联系 IT 检查呼叫中心

#### Q: 客户反映接到骚扰电话怎么办？

**排查步骤**：
1. 核实通话记录和时间
2. 检查是否在允许外呼时间段
3. 查看客户黑名单状态

**解决方案**：
- 将客户加入黑名单
- 排查违规外呼坐席
- 加强合规培训

### 8.1 电销转化提升方法论

**提升转化率的关键**：
1. 精准人群：选择高意向客户，减少无效拨打
2. 最佳时间：在客户方便的时间拨打
3. 专业话术：标准化话术 + 个性化应对
4. 即时跟进：通话结束立即跟进，不拖延
5. 有效激励：绩效机制引导积极心态
