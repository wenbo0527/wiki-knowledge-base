## 核心洞察

### 洞察1：晋升公式

```
晋升 = 现有优势 × AI放大 + 新技能补位 - 短板拖腿
```

AI时代，PM的核心价值不是"转行做AI"，而是**用AI放大现有优势**。

### 洞察2：定位升级

| 旧定位 | 新定位 | 核心差异 |
|--------|--------|----------|
| AI模型PM | Data+AI平台PM | 从技术到连接 |

**新定位** = 数据→模型→业务的连接器 + 治理者 + 价值放大器

### 洞察3：能力升级路径

不是"从零学AI"，而是：

| 保留强化 | 延伸扩展 | 避免 |
|----------|----------|------|
| 数据治理 | RAG/向量库 | 与算法工程师竞争 |
| 平台设计 | 模型编排/MLOps | 深入模型训练 |
| 业务理解 | AI场景合规 | 纯技术方向 |

### 洞察4：阶段里程碑

| 阶段 | 核心成果 | 可量化指标 |
|------|----------|------------|
| 0-1年 | 落地RAG模块 | 支持3个业务场景 |
| 1-2年 | 主导航剖功能 | 沉淀1套行业解决方案 |
| 2-3年 | 带1-2人小团队 | 成为行业KOL |

### 洞察5：避坑原则

| 坑 | 原因 |
|----|------|
| 纯模型研发PM | 技术深度要求高，PM优势无法发挥 |
| 通用AI工具PM | 竞争激烈，没有行业壁垒 |

## 面试高频题框架

### Q: 如何设计RAG平台？
```
数据治理 → 向量存储 → 检索增强 → 合规审计
```

### Q: 如何平衡效果与合规？
```
算法透明化 + 数据可追溯 + 权限管控
```

### Q: 如何设计AI平台权限体系？
```
模型权限 + 数据权限 + 操作权限（三层架构）
```

### Q: 为什么从数据PM转向AI平台PM？
```
顺势而为：AI普及后，企业需要"数据+平台+AI"的复合人才
```
