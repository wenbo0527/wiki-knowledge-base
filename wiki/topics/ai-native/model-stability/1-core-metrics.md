## 核心指标体系
```
模型稳定性评估指标
├── 群体稳定性指标（PSI）
│   ├── 单特征PSI：诊断哪个特征发生了漂移
│   ├── 模型输入PSI：所有特征PSI的综合评估
│   └── 模型输出PSI：模型预测分的分布稳定性
├── 条件稳定性指标（CSI）
│   └── 条件分箱稳定性，衡量特征在不同分箱下的稳定性
├── 模型效果指标
│   ├── KS值变化：区分度稳定性
│   ├── AUC变化：排序能力稳定性
│   └── 准确率/召回率变化：业务指标稳定性
└── 特征级稳定性
    ├── 特征缺失率变化
    ├── 特征均值/分位数变化
    └── 特征分布变化（KL散度、JS散度）
```

## 1. PSI（群体稳定性指标）⭐（最核心指标）
### 原理
PSI衡量两个分布之间的差异，用于比较模型上线时的分布和当前分布的差异，是金融行业评估模型稳定性的标准指标。
**公式**：
```
PSI = Σ[(Actual% - Expected%) × ln(Actual% / Expected%)]
```
其中：
- Expected%：模型训练/上线时该分箱的用户占比（基准分布）
- Actual%：当前时该分箱的用户占比（实际分布）
### 判断标准
| PSI值范围 | 稳定性评级 | 处理建议 |
|-----------|-----------|----------|
| <0.1 | 非常稳定 ✅ | 可以继续使用，无需任何操作 |
| 0.1-0.25 | 一般稳定 ⚠️ | 需要关注，分析原因，可以继续使用 |
| >0.25 | 严重不稳定 ❌ | 模型效果可能已大幅下降，需要立即修复或替换 |
| >0.5 | 极度不稳定 🔴 | 模型已完全失效，必须立即下线，用旧模型替代 |
### PSI计算示例
```
分箱1（信用评分0-300）：基准分布10%，当前分布15%，贡献=0.0166
分箱2（信用评分300-500）：基准分布30%，当前分布35%，贡献=0.0054
分箱3（信用评分500-700）：基准分布40%，当前分布38%，贡献=0.0011
分箱4（信用评分700-900）：基准分布20%，当前分布12%，贡献=0.0231
PSI = 0.0462 < 0.1，模型非常稳定
```
### 不同维度的PSI
| PSI类型 | 计算对象 | 监控频率 | 阈值建议 | 说明 |
|---------|----------|----------|----------|------|
| 特征PSI | 单个特征 | 每天 | >0.25报警 | 诊断哪个特征发生漂移，是最重要的PSI |
| 输入PSI | 所有特征的综合 | 每天 | >0.2报警 | 模型整体输入分布稳定性 |
| 输出PSI | 模型预测分 | 每天 | >0.25报警 | 模型整体稳定性，最核心的监控指标 |
| 评分PSI | 信用评分（分箱后） | 每周 | >0.1报警 | 业务人员使用的评分分箱稳定性 |

## 2. CSI（条件稳定性指标）
### 原理
PSI只衡量整体分布，CSI可以衡量在不同条件下（如好用户/坏用户）的分布稳定性，能够发现PSI发现不了的问题。
**公式**：
```
CSI = PSI(坏用户分布) - PSI(好用户分布)
```
CSI越大，说明模型对坏用户的排序能力变化越大，模型效果可能已下降。
### 判断标准
| CSI值范围 | 含义 | 处理建议 |
|-----------|------|----------|
| <0.05 | 非常稳定 | 继续监控 |
| 0.05-0.1 | 轻度漂移 | 关注，可能需要模型迭代 |
| >0.1 | 明显漂移 | 模型效果可能已下降，需要详细诊断 |
| >0.2 | 严重漂移 | 模型已失效，需要立即修复或替换 |
---
## 3. KS值变化（区分度稳定性）
### 原理
KS值衡量模型区分好坏用户的能力，KS越高区分度越好，KS<0.2说明模型基本失效。
### 判断标准
| KS变化幅度 | 含义 | 处理建议 |
|-----------|------|----------|
| 下降<5% | 正常波动 | 继续监控 |
| 下降5-15% | 轻度下降 | 分析原因，考虑模型迭代 |
| 下降>15% | 明显下降 | 模型效果已受影响，需要立即修复 |
| KS<0.2 | 基本失效 | 模型已无法区分好坏用户，必须立即替换 |
### 监控方法
```
每周计算KS值，和上周对比，和上线时对比，和历史平均对比
KS(当前) = 0.35，上线时KS = 0.42，下降幅度 = (0.35-0.42)/0.42 = -16.7% > 15%，触发报警
```

## 4. AUC变化（排序能力稳定性）
### 判断标准
| AUC变化幅度 | 含义 | 处理建议 |
|-----------|------|----------|
| 下降<2% | 正常波动 | 继续监控 |
| 下降2-5% | 轻度下降 | 分析原因，可能需要模型迭代 |
| 下降>5% | 明显下降 | 模型效果已受影响，需要立即修复 |
| AUC<0.7 | 效果较差 | 模型已不太适合生产环境 |

## 5. 特征级稳定性诊断
### 特征缺失率监控
| 缺失率变化 | 含义 | 处理建议 |
|-----------|------|----------|
| 变化<5% | 正常波动 | 继续监控 |
| 变化5-10% | 轻度变化 | 关注，可能是数据源正常波动 |
| 变化>10% | 明显变化 | 数据源可能有问题，需要立即排查 |
| 突然>50% | 严重问题 | 数据源可能中断，必须立即处理 |
### 特征分布监控
| 监控指标 | 阈值建议 | 说明 |
|----------|----------|------|
| 均值变化 | 变化>20% | 特征分布可能已漂移 |
| 分位数变化（25/50/75分位） | 变化>25% | 特征分布可能已漂移 |
| 标准差变化 | 变化>30% | 特征波动性可能已变化 |
| KL散度/JS散度 | >0.5 | 特征分布已发生显著变化 |
### 特征PSI分析
对每个特征单独计算PSI，识别漂移最严重的特征：
```
特征PSI排名（本周）：
1. 近7天交易金额PSI=0.38 ❌（严重漂移，需立即排查）
2. 信用卡使用率PSI=0.22 ⚠️（轻度漂移，关注）
3. 逾期次数PSI=0.15 ✅（稳定）
4. 收入负债比PSI=0.08 ✅（非常稳定）
```
**排查优先级**：PSI最高的特征优先排查，往往是模型不稳定的主要原因。

## 综合稳定性评估矩阵
| 稳定性等级 | PSI | KS变化 | AUC变化 | 特征PSI>0.25比例 | 建议 |
|------------|-----|--------|---------|------------------|------|
| 绿色（稳定） | <0.1 | 变化<5% | 变化<2% | <10% | 继续使用，每周一监控 |
| 黄色（关注） | 0.1-0.25 | 变化5-15% | 变化2-5% | 10-30% | 加强监控，每日报警，分析原因 |
| 橙色（预警） | 0.25-0.5 | 变化>15% | 变化>5% | >30% | 立即诊断，准备切换，可能需要降级或关闭模型 |
| 红色（告警） | >0.5 | KS<0.2 | AUC<0.7 | >50% | 立即下线旧模型，切换备用模型，紧急迭代新模型 |
