## 产品背景

### 为什么需要分析套件

```
业务痛点：
├── 数据不统一
│   └── "我和风控看的数对不上"
├── 看数周期长
│   └── "这个报表要等一天才能跑出来"
├── 口径不透明
│   └── "这个指标是怎么算的？"
└── 报表重复开发
    └── "同样的需求开发了3遍"

技术痛点：
├── 指标分散：每个系统都有自己的指标
├── 计算重复：同一个指标算了十几遍
├── 维护困难：改了底层数据结构，报表全挂
└── 性能差：SQL跑几分钟甚至几十分钟
```

### 消费金融的独特挑战

| 挑战 | 说明 |
|------|------|
| **监管报表** | 需定期向监管报送数据 |
| **指标一致性** | 各部门数据要能对上 |
| **实时性要求** | 业务决策需要实时数据 |

## 核心功能

### 1. 指标平台（统一指标定义）

**功能描述**：统一管理指标定义，避免口径不一致

**核心能力**：
- 指标定义：原子指标、派生指标、复合指标
- 指标分级：北极星指标、过程指标、基础指标
- 指标服务：API方式提供指标
- 指标血缘：从业务→数据→报表的链路

**指标类型**：
```
原子指标：
├── 放款金额
├── 放款笔数
└── 在贷余额

派生指标：
├── 日放款金额（时间周期限定）
├── 周放款笔数（时间周期限定）
└── 近30天逾期率（时间周期+业务限定）

复合指标：
├── 账龄分析（多指标组合）
├── 迁徙率（多时期对比）
└── 核销率（多指标计算）
```

### 2. 自助分析（拖拽式数据分析）

**功能描述**：让业务人员能自己分析数据

**核心能力**：
- 拖拽维度：拖拽字段生成图表
- 图表类型：折线图、柱状图、饼图、漏斗图
- 多维分析：下钻、上卷、旋转
- SQL能力：支持高级用户写SQL

**适用场景**：
```
运营分析：转化漏斗、留存分析
风控分析：逾期分布、欺诈模式
财务分析：收入分析、成本分析
```

### 3. 报表自动化

**功能描述**：自动化生成和分发报表

**核心能力**：
- 报表配置：拖拽配置报表样式
- 定时任务：自动刷新数据
- 订阅分发：自动发送到邮箱、企业微信
- 预警通知：指标异常自动告警

**报表类型**：
```
日报/周报/月报：
├── 核心指标看板
├── 业务进展报表
└── 财务汇总报表

专项报表：
├── 风控报表
├── 营销报表
└── 监管报表
```

### 4. 数据门户

**功能描述**：数据资产的统一入口

**核心能力**：
- 数据搜索：搜索表、指标、报表
- 数据血缘：查看数据流转链路
- 使用排行：最热门的表和指标
- 申请入口：数据权限申请

## 落地实践

### MVP版本（6个月）

```
Phase 1（0-2月）：
├── 指标平台MVP
│   ├── 指标定义
│   ├── 指标服务
│   └── 基础指标
└── 报表自动化
    └── 核心报表配置

Phase 2（3-4月）：
├── 自助分析基础版
│   ├── 拖拽图表
│   └── 预置报表模板
└── 数据门户基础版
    └── 搜索和导航

Phase 3（5-6月）：
├── 指标平台完善
│   ├── 指标血缘
│   └── 指标监控
└── 自助分析增强
    ├── SQL能力
    └── 高级图表
```
