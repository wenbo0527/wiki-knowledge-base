## 征信特征核心价值
征信数据是消费金融风控的"黄金数据"，是区分度最高的特征类型：
- 贡献超过30%的模型效果，单特征IV值普遍在0.3以上，最高可达1.0
- 是评估用户还款意愿和历史信用表现的最可靠依据
- 没有征信数据的风控体系基本是"裸奔"，坏账率会是有征信体系的2-3倍
```
征信特征价值排序：
人行征信 > 百行征信 > 持牌第三方征信数据 > 互联网公开数据 > 爬虫数据
```

## 1. 人行征信特征 ⭐（价值最高，必接）
人行征信报告是官方最权威的信用数据，所有持牌消费金融公司必须接入，是风控的核心。
### 报告核心模块
| 模块 | 核心信息 | 衍生特征价值 |
|------|----------|--------------|
| 个人基本信息 | 身份信息、婚姻状况、居住信息、职业信息 | 低-中，基础属性特征 |
| 信息概要 | 信贷账户数、逾期账户数、查询次数概要、负债概要 | 高，核心特征来源 |
| 信贷交易信息明细 | 每笔贷款/信用卡的开户时间、额度、使用情况、逾期记录、还款记录 | 极高，最强特征来源 |
| 查询记录 | 近2年的所有查询记录：查询时间、查询机构、查询原因 | 极高，多头借贷核心特征 |
| 公共信息 | 社保、公积金、欠税、行政处罚、失信被执行人记录 | 高，辅助风险特征 |
### 核心衍生特征（按价值排序）
#### 1.1 逾期类特征（IV值0.4-1.0，最强特征）
| 特征 | 计算逻辑 | 业务含义 | IV值范围 |
|------|----------|----------|----------|
| 历史逾期总次数 | 所有信贷账户的逾期总次数 | 历史信用履约情况，次数越多风险越高 | 0.5-1.0 |
| 近1年/2年逾期次数 | 近1/2年的逾期总次数 | 近期信用表现，越近的逾期风险越高 | 0.4-0.8 |
| 最长逾期天数 | 所有逾期记录中最长的逾期天数 | 逾期严重程度，超过90天的逾期属于严重不良 | 0.4-0.7 |
| 最近一次逾期距今天数 | 最近一次逾期发生时间到现在的天数 | 逾期的新鲜度，越近的逾期风险越高 | 0.3-0.6 |
| 逾期账户占比 | 逾期账户数 / 总信贷账户数 | 历史违约比例，占比越高风险越高 | 0.3-0.5 |
#### 1.2 负债类特征（IV值0.2-0.5）
| 特征 | 计算逻辑 | 业务含义 | IV值范围 |
|------|----------|----------|----------|
| 信用卡平均使用率 | 信用卡已使用额度 / 信用卡总授信额度 | 信用卡负债水平，使用率>80%属于高负债 | 0.25-0.4 |
| 贷款余额 / 年收入 | 所有贷款总余额 / 用户年收入 | 整体负债水平，比例>5倍属于高负债 | 0.2-0.35 |
| 近6个月新增贷款金额 | 近6个月新发放的贷款总金额 | 近期负债增长速度，增长越快风险越高 | 0.2-0.3 |
| 担保贷款金额 | 对外担保的总贷款金额 | 或有负债，担保金额越高风险越高 | 0.15-0.25 |
#### 1.3 查询类特征（IV值0.3-0.6，多头借贷核心）
| 特征 | 计算逻辑 | 业务含义 | IV值范围 |
|------|----------|----------|----------|
| 近1个月贷款审批查询次数 | 近1个月机构以"贷款审批"为原因的查询次数 | 近期申请贷款的频率，次数>5属于高危 | 0.3-0.6 |
| 近3个月贷款审批/信用卡审批查询次数 | 近3个月两类查询总次数 | 中期多头借贷情况，次数>10属于高危 | 0.25-0.5 |
| 查询机构集中度 | 近1个月查询机构数 / 总查询次数 | 是不是在集中申请多家机构贷款，集中度越高风险越高 | 0.2-0.3 |
| 不同类型查询占比 | 贷款审批查询占比 / 信用卡审批占比 / 贷后管理占比 | 贷款审批占比越高，用户近期越缺钱，风险越高 | 0.15-0.25 |
### 人行征信使用避坑指南
1. **查询次数不要过度使用**：不要因为用户查询次数多就直接拒贷，有些用户只是在比价，结合其他特征综合判断，避免误杀优质用户
2. **逾期区分恶意和非恶意**：偶尔的1-3天小额逾期可能是忘记还款，不是恶意违约，不要直接拒贷，要看逾期的频率、金额、时长
3. **注意报告更新延迟**：人行征信报告更新有T+1到T+7的延迟，最新的贷款和逾期记录可能还没更新，要结合其他数据交叉验证
4. **合规使用**：必须获得用户授权才能查询，不能超范围使用，符合《征信业管理条例》要求

## 2. 百行征信特征
百行征信是由互联网金融协会牵头，持牌的市场化征信机构，覆盖大量人行征信没有覆盖的互联网信贷、消费金融、小贷公司的信贷记录，是人行征信的重要补充。
### 核心特征
| 特征类别 | 典型特征 | 价值 |
|----------|----------|------|
| 互联网信贷逾期记录 | 互联网小贷、消费金融的逾期记录 | 高，很多用户人行征信没有逾期，但在互联网平台有大量逾期 |
| 互联网信贷多头借贷记录 | 近1/3/6个月在互联网平台的申请次数、获批次数 | 高，识别互联网端的多头借贷，人行征信查不到 |
| 共债信息 | 在多家消费金融公司的总负债、总授信 | 高，评估用户的整体负债水平 |
| 欺诈风险分 | 百行征信的欺诈评分、黑名单标识 | 高，识别互联网欺诈用户 |
### 适用场景
- 下沉用户人群：人行征信空白或者记录很少的用户，比如年轻人、蓝领、下沉市场用户
- 互联网端风险识别：识别互联网平台的多头借贷、欺诈风险
### 价值评估
- 百行征信特征的IV值普遍在0.2-0.4之间，低于人行征信，但高于其他第三方数据
- 对于人行征信空白的用户，百行征信可以大幅提升模型效果，降低坏账率20%以上

## 3. 第三方征信数据特征
持牌第三方征信机构提供的风险数据，作为人行和百行征信的补充。
### 主流第三方数据源
| 数据源类型 | 代表机构 | 核心特征 | 价值 |
|----------|----------|----------|------|
| 多头借贷数据 | 同盾、百融云创 | 全行业的贷款申请次数、获批次数、多头借贷标签 | 中高，覆盖很多非持牌机构的申请记录 |
| 反欺诈黑名单 | 同盾、腾讯安全、阿里安全 | 欺诈黑名单、羊毛党名单、中介名单 | 高，直接拒绝命中黑名单的用户 |
| 运营商数据 | 三大运营商、持牌数据公司 | 在网时长、话费缴纳情况、欠费记录、常用联系人风险 | 中，辅助评估用户稳定性和风险 |
| 电商消费数据 | 阿里、京东、持牌数据公司 | 消费层级、消费品类、收货地址稳定性、消费能力 | 中，评估用户消费能力和稳定性 |
### 第三方数据使用原则
1. **优先选择持牌机构**：确保数据合规，避免合规风险
2. **数据质量验证**：接入前先做小流量测试，验证数据的区分度和覆盖率，不要盲目接入
3. **性价比评估**：很多第三方数据价格很高，要算ROI，投入的成本要低于减少的坏账损失
4. **避免数据冗余**：不要接入多个数据源的同类型特征，比如同时接入3家的多头借贷数据，浪费成本

## 4. 多头借贷特征 ⭐（核心风险特征）
多头借贷是指用户在多家金融机构同时申请贷款，是高风险的强信号，多头借贷用户的坏账率是普通用户的3-5倍。
### 多头借贷分层规则（行业通用标准）
| 多头等级 | 近1个月贷款申请机构数 | 坏账率倍数 | 处理策略 |
|----------|------------------------|------------|----------|
| 正常 | <3 | 1x | 正常准入 |
| 轻度多头 | 3-5 | 1.5x | 谨慎准入，提高定价，降低额度 |
| 中度多头 | 5-10 | 3x | 严格审核，大部分直接拒绝 |
| 重度多头 | >10 | 5x+ | 直接拒绝 |
### 核心衍生特征
| 特征 | 计算逻辑 | 价值 |
|------|----------|------|
| 近1/3/6个月不同机构申请次数 | 不同的放款机构数量 | 高，比总申请次数更能反映多头情况 |
| 申请通过率 | 获批次数 / 总申请次数 | 中，通过率越低说明用户资质越差，被其他机构拒绝越多 |
| 不同类型机构申请占比 | 消费金融/小贷/网贷/银行的申请占比 | 中，小贷、网贷申请占比越高，风险越高 |
| 首次申请距今天数 | 第一次贷款申请到现在的时长 | 中，信贷历史越短的用户风险越高 |
### 避坑指南
1. **不要一刀切**：不要只要是多头就直接拒绝，有些优质用户只是短期资金周转，收入高负债低，还款能力强，可以谨慎准入
2. **区分申请和放款**：用户申请了不代表获批了，要重点看获批的机构数和负债水平
3. **关注申请集中度**：短时间内集中申请多家机构的用户，风险远高于分散申请的用户

## 5. 黑名单特征
黑名单是风险最高的用户群体，命中后直接拒绝，不需要进模型评估，坏账率接近100%。
### 黑名单类型
| 类型 | 内容来源 | 处理策略 |
|------|----------|----------|
| 官方黑名单 | 失信被执行人名单、限制高消费名单、涉毒涉赌涉恐名单、法院被执行人名单 | 100%直接拒绝 |
| 行业共享黑名单 | 消费金融行业共享的欺诈用户、逾期用户名单 | 直接拒绝 |
| 内部黑名单 | 本公司历史上的欺诈用户、严重逾期用户、套现用户 | 直接拒绝 |
| 第三方黑名单 | 第三方征信机构的欺诈黑名单、羊毛党名单、中介名单 | 核实后拒绝 |
### 黑名单使用避坑指南
1. **同名同姓问题**：要结合身份证号、手机号等唯一标识判断，不要因为同名同姓就误拒
2. **黑名单时效**：很多黑名单是有时效的，比如3年前的逾期黑名单，用户已经还款，现在信用良好，可以考虑解除黑名单
3. **误判申诉机制**：要有申诉通道，用户被误判为黑名单可以申请复核
4. **数据合规**：黑名单数据要来源合规，不能使用非法渠道获取的黑名单

## 征信特征落地最佳实践
### 接入优先级
1. 优先接入人行征信：必接，核心中的核心
2. 其次接入百行征信：补充人行覆盖不到的互联网信贷数据
3. 按需接入第三方数据：根据客群情况选择性价比高的数据源，不要盲目贪多
### 特征加工原则
1. **多时间窗口**：同一个指标做近1/3/6/12个月多个时间窗口的特征，覆盖短期和长期表现
2. **比率特征优先**：优先使用占比、比率类特征，比绝对值更稳定，比如逾期账户占比比逾期账户数更好
3. **时效性特征**：优先使用最近的特征，近1个月的特征比近1年的特征更有预测性
4. **交叉验证**：不同数据源的同类型特征交叉验证，比如人行的多头次数和百行的多头次数互相验证，避免数据错误
### 合规要求
1. **授权合规**：所有征信数据查询必须获得用户的明确书面授权，符合《个人信息保护法》《征信业管理条例》要求
2. **数据最小必要**：只采集必要的征信数据，不要超范围采集
3. **存储安全**：征信数据要加密存储，严格控制访问权限，避免数据泄露
4. **用户知情权**：用户可以查询自己的征信数据使用情况，有异议可以申诉
