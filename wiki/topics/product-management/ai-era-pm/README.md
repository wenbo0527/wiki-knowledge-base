## 🔗 与其他Topic的关系

### 1. 与 [[ai-native/ai-programming|Vibe Coding]] 的关系

**Vibe Coding 是PM的新编程语言**：
- PM不需要写代码，但要能"表达意图"
- AI理解自然语言需求，生成代码
- PM的角色：审核、迭代、定义边界

**PM在Vibe Coding中的职责**：
```
需求描述 → AI生成 → PM审核 → 迭代调整 → 交付
```

### 2. 与 [[ai-native/agent-engineering|Agent Engineering]] 的关系

**PM需要理解Agent的能力边界**：
- 什么任务适合Agent
- 什么任务需要人工
- 如何设计Human-in-the-loop

- 核心方法论不变：用户需求、优先级、商业价值
- 变化的是工具和执行方式
- AI放大PM的生产力

### 4. 与 [[product-management/pm-career-development|职业发展]] 的关系

## 核心洞察

### 1. PM的核心能力转变

> "AI时代PM的核心能力不是编码，而是**定义边界**的能力" — 盯人日报

**边界定义三要素**：
- **功能边界**：什么做，什么不做
- **能力边界**：AI能做什么，不能做什么
- **安全边界**：什么不能交给AI

### 2. AI PM技能重写

**传统PM技能** → **AI时代PM技能**：
| 传统 | AI时代 |
|------|--------|
| Axure/Figma原型 | Prompt + AI生成原型 |
| 需求文档 | AI增强PRD |
| 跨部门沟通 | 人机协作沟通 |
| 数据分析 | AI实时分析 |

### 3. OpenClaw龙虾与PM工作流

**[[ai-native/openclaw-practices|OpenClaw龙虾]] 为PM打开的新世界**：
- 11人Agent团队 = PM的虚拟执行团队
- 自动化日报、周报生成
- 自动化信息收集与分析
- PM从执行者变成审核者
