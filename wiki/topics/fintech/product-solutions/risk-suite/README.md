## 产品背景

### 为什么需要风控套件

```
业务痛点：
├── 审批时效慢
│   └── "客户等太久流失了"
├── 坏账率高
│   └── "风控模型不准，坏客户进来了"
├── 规则难调整
│   └── "规则改了要等开发排期"
└── 模型效果难追踪
    └── "不知道模型效果是变好还是变差"

技术痛点：
├── 特征重复开发：每个模型都要自己抽特征
├── 决策链路长：从进件到放款经过多个系统
└── 模型迭代慢：改一个模型要几天甚至几周
```

### 消费金融的独特挑战

| 挑战 | 说明 |
|------|------|
| **监管合规** | 决策必须可解释、可追溯 |
| **实时性要求** | 贷款审批、支付验证需毫秒级响应 |
| **欺诈升级** | 黑产使用AI生成、自动化工具 |

## 核心功能

### 1. 实时决策引擎

**功能描述**：提供毫秒级的风控决策能力

**核心能力**：
- 决策流编排：可视化配置决策流程
- 多模型融合：信用模型+反欺诈模型+收入模型
- 阈值管理：动态调整通过率
- 决策理由：输出可解释的决策原因

**性能指标**：
```
响应时间：<50ms（目标）
高并发：每秒数万笔
可用性：99.99%
```

### 2. 特征平台

**功能描述**：统一管理和共享特征，避免重复计算

**核心能力**：
- 特征注册：统一管理特征定义
- 特征计算：实时计算+离线计算
- 特征服务：API方式提供特征
- 特征监控：特征覆盖度、数据质量

**特征类型**：
```
实时特征（<100ms）：
├── 最近1小时交易次数
├── 最近登录状态
└── 当天申请次数

离线特征（T+1）：
├── 过去30天逾期次数
├── 过去90天消费金额
└── 信用评分

组合特征：
├── 实时特征 + 离线特征
└── 交叉特征
```

### 3. 模型管理

**功能描述**：管理模型的生命周期

**核心能力**：
- 模型注册：统一管理模型版本
- A/B测试：灰度发布、效果对比
- 效果追踪：监控模型表现
- 模型下线：按规则自动下线差模型

**模型类型**：
```
信用模型：
├── 评分卡模型
├── XGBoost模型
└── 深度学习模型

反欺诈模型：
├── 规则引擎
├── 图神经网络
└── 异常检测
```

### 4. 规则引擎

**功能描述**：让业务人员能自主配置规则

**核心能力**：
- 可视化配置：拖拽配置规则条件
- 规则组合：支持规则组、规则链
- 规则测试：模拟规则效果
- 规则版本：记录规则变更历史

## 落地实践

### MVP版本（6个月）

```
Phase 1（0-2月）：
├── 实时决策引擎MVP
│   ├── 基础决策流
│   ├── 规则配置
│   └── 决策理由输出
└── 特征平台基础版
    └── 特征注册和服务

Phase 2（3-4月）：
├── 模型管理基础版
│   ├── 模型注册
│   └── 效果追踪
└── 规则引擎
    └── 可视化配置

Phase 3（5-6月）：
├── 特征平台完善
│   ├── 特征计算
│   └── 特征监控
└── 模型管理增强
    ├── A/B测试
    └── 自动下线
```
