## 产品背景

### 为什么需要营销套件

```
业务痛点：
├── 流程配置复杂（70%反馈）
│   └── "一个活动要配置几十个节点，太麻烦了"
├── 人群圈选慢（50%反馈）
│   └── "我知道要精细化运营，但不知道怎么分群"
├── 合规风险（30%反馈）
│   └── "这个营销短信合规吗？监管查怎么办"
└── 效果难追踪（30%反馈）
    └── "发了100万条短信，怎么知道哪些人有效果"

技术痛点：
├── 人群圈选性能差：几千万用户，SQL跑不出来
├── 流程引擎慢：节点多了就超时
└── 触达渠道多：短信、APP、微信，每个渠道规则不一样
```

### 消费金融的独特挑战

| 挑战 | 说明 |
|------|------|
| **监管合规优先** | 营销内容要审核，不能随便发 |
| **人群圈选性能** | 几千万用户，要能快速跑出来 |
| **效果归因难** | 短信/APP/微信，哪个渠道有效？ |

## 核心功能

### 1. 营销自动化画布（拖拽配置）

**功能描述**：让运营人员通过拖拽方式配置营销流程

**核心能力**：
- 可视化画布：拖拽节点，连线配置流程
- 节点类型：触发节点、人群节点、动作节点、条件节点
- 流程模板：预置常见营销模板
- 版本管理：支持回滚、复制

**节点类型**：
```
触发节点：
├── 定时触发（每天10点）
├── 事件触发（注册、交易、逾期）
└── 手动触发

人群节点：
├── 标签人群（性别、年龄、资产等级）
├── 行为人群（最近30天有交易）
└── 规则人群（自定义SQL）

动作节点：
├── 短信触达
├── APP推送
├── 微信模板消息
└── 外呼

条件节点：
├── 分支条件（根据属性分流）
└── 等待节点（延时触达）
```

### 2. 人群圈选（预计算人群包）

**功能描述**：支持精细化的人群划分和圈选

**核心能力**：
- 标签体系：人口属性、资产属性、行为属性
- 预计算人群包：提前计算好，避免实时计算
- 人群包管理：创建、编辑、过期设置
- 组合人群：多个人群包交并差

**不做的原因**（资源约束）：
- 实时人群计算技术债务多
- 先做预计算，MVP优先

### 3. 合规审核节点（嵌入流程）

**功能描述**：在营销流程中嵌入合规检查

**核心能力**：
- 内容审核：敏感词、监管禁止词
- 渠道合规：时间段、频率限制
- 人群合规：监管要求的人群限制
- 审批流程：重要营销活动需审批

**嵌入方式**：
```
画布配置 → 合规审核节点 → 通过/不通过 → 继续/终止
```

### 4. 效果分析（触达统计）

**功能描述**：追踪营销活动的效果

**核心能力**：
- 触达统计：发了多少、触达多少
- 转化追踪：点击、注册、交易
- 渠道对比：短信 vs APP vs 微信
- ROI分析：投入产出比

## 落地实践

### MVP版本（6个月）

```
Phase 1（0-2月）：
├── 营销自动化画布MVP
│   ├── 基础节点（触发、人群、动作）
│   ├── 简单条件分支
│   └── 流程模板
└── 基础人群圈选
    └── 预计算人群包

Phase 2（3-4月）：
├── 合规审核节点
│   ├── 内容审核
│   └── 审批流程
└── 基础效果分析
    └── 触达统计

Phase 3（5-6月）：
├── 人群圈选增强
│   ├── 组合人群
│   └── 人群画像
└── 效果分析增强
    ├── 转化追踪
    └── ROI分析
```
