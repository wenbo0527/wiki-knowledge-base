# AI PM技能重写：从"定义功能"到"定义边界"

## 📌 核心洞察

### PM角色的根本转变

| 维度 | 传统PM | AI时代PM |
|------|--------|----------|
| **核心任务** | 定义功能 | 定义边界 |
| **能力重心** | 功能设计 | 需求边界 |
| **价值创造** | 输出PRD | 输出洞察 |
| **协作方式** | 跨部门沟通 | 人机协作 |

---

## 💡 边界定义的三个维度

### 1. 功能边界

**定义**：什么做，什么不做

```markdown
### 功能边界示例

✅ 做：
- 用户登录
- 数据展示

❌ 不做：
- 支付功能
- 第三方集成
```

### 2. 能力边界

**定义**：AI能做什么，不能做什么

| 能力 | AI适合 | AI不适合 |
|------|--------|----------|
| 信息收集 | ✅ | |
| 初步分析 | ✅ | |
| 创意生成 | ✅ | |
| 最终决策 | | ❌ |
| 用户沟通 | | ❌ |
| 风险判断 | | ❌ |

### 3. 安全边界

**定义**：什么不能交给AI

```markdown
### 安全边界

🔒 始终Human-in-the-loop：
- 涉及资金的决策
- 用户数据访问
- 权限变更操作
- 敏感信息披露
```

---

## 🔮 深层启示

### PM的新价值公式

```
AI时代PM价值 = 边界定义能力 × AI协作效率 × 洞察输出质量
```

### PM的核心竞争力

| 竞争力 | 说明 |
|--------|------|
| **需求洞察** | 发现真正的用户需求 |
| **边界设计** | 清晰定义功能边界 |
| **人机协作** | 高效与AI协作 |
| **质量把控** | 确保输出质量 |
