## 案例1：电商优惠券效果评估

### 场景
> 电商平台发放了100万张满减优惠券，需要评估优惠券对GMV的真实提升效果，排除"本来就会买的用户"的影响。

### 数据
| 字段 | 说明 |
|------|------|
| user_id | 用户ID |
| coupon_received | 是否收到优惠券（1=是，0=否） |
| purchase_amount | 购买金额 |
| age | 用户年龄 |
| gender | 性别 |
| historical_purchase | 历史3个月购买金额 |
| activity_level | 过去1个月活跃天数 |

### 分析步骤
1. **问题定义**：优惠券对用户购买金额的平均处理效应（ATE）是多少？
2. **因果图**：
   ```
   历史购买 → 是否收到优惠券 → 购买金额
   活跃天数 → 是否收到优惠券 → 购买金额
   年龄 → 购买金额
   性别 → 购买金额
   ```
3. **识别策略**：后门调整，控制历史购买、活跃天数、年龄、性别等混杂因素
4. **估计方法**：倾向得分匹配 + 线性回归
5. **结果**：
   - 原始差值：收到优惠券用户比没收到的多花120元
   - 调整后ATE：优惠券真实提升效果为75元
   - 投资回报率：(75-30)/30 = 150%

### 业务建议
- 优惠券效果显著，但有45元是"挤出效应"（用户本来就会买）
- 重点向历史购买金额中等的用户发放优惠券，效果最好
- 高价值用户发放优惠券效果不明显，建议用其他权益

## 案例2：风控策略效果评估

### 场景
> 消费金融公司新上线了一个风控策略，拒绝了部分高风险用户，需要评估该策略是否真的降低了坏账率。

### 挑战
- 无法观察到被拒绝的用户如果通过申请的坏账率
- 策略上线前的用户和上线后的用户群体可能有差异

### 方法：双重差分法 (Difference in Differences, DID)
```
对照组：不受策略影响的用户群体（比如信用分极高的用户，无论新旧策略都会通过）
处理组：受策略影响的用户群体（信用分中等的用户，旧策略通过，新策略部分拒绝）

效应 = (处理组上线后坏账率 - 处理组上线前坏账率) - (对照组上线后坏账率 - 对照组上线前坏账率)
```

### 结果
- 坏账率绝对下降1.2个百分点
- 按100亿放贷规模计算，每年减少损失1.2亿元
- 但通过率下降5个百分点，影响放款规模5亿元，需要权衡

## 案例3：Uber 溢价定价对司机接单率的影响

### 场景
> Uber在高峰时段实行溢价定价，需要评估溢价是否真的能提升司机接单率，还是只是把订单转移到了其他平台。

### 挑战
- 溢价通常发生在需求高的时段，这些时段本来接单率就高，存在反向因果
- 无法同时观察到同一时段有溢价和没有溢价的情况

### 方法：断点回归 (Regression Discontinuity Design, RDD)
- 政策：当需求超过阈值T时，自动开启溢价
- 比较阈值T附近很小范围内的订单：T-ε（无溢价） vs T+ε（有溢价）
- 这两部分订单的其他特征几乎相同，差异只能来自溢价

### 结果
- 溢价1.2x时，接单率提升20%
- 溢价超过1.5x后，提升效应递减，用户开始取消订单
- 最优溢价区间是1.2-1.5x

## 案例4：营销活动效果评估（异质性分析）

### 场景
> 互联网公司上线了一个拉新活动，给新用户发补贴，需要评估不同用户群体的效果差异，优化补贴发放策略。

### 方法：因果森林 + Uplift建模
```
目标：找到"补贴对转化率提升最大"的用户群体

Uplift = P(转化|补贴) - P(转化|无补贴)
```

### 结果
| 用户群体 | Uplift | ROI | 建议 |
|----------|--------|-----|------|
| 新注册1天内的年轻用户 | 15% | 300% | 优先发放 |
| 新注册7天以上的用户 | 3% | 50% | 减少发放 |
|  iOS用户 | 12% | 250% | 重点投放 |
|  Android用户 | 6% | 120% | 正常投放 |
| 三四线城市用户 | 18% | 400% | 加大投入 |

### 优化效果
- 预算不变的情况下，整体转化率提升25%
- ROI从150%提升到220%

## 案例5：产品功能迭代评估

### 场景
> 社交APP新上线了"附近的人"功能，需要评估该功能对用户留存的影响。

### 方法：A/B测试 + 中介分析
- A组：无新功能
- B组：有新功能
- 跟踪指标：7日留存、使用频率、好友添加数

### 结果
- 整体7日留存提升2.1个百分点
- 中介分析：
  - 直接效应：0.8个百分点（功能本身带来的留存提升）
  - 间接效应：1.3个百分点（通过添加更多好友带来的留存提升）
- 效应主要来自添加更多好友，建议后续迭代优化好友推荐算法

## 总结：不同场景适用方法

| 业务场景 | 首选方法 | 备选方法 |
|----------|----------|----------|
| 有条件做随机实验 | A/B测试 | 无 |
| 无法随机实验，有混杂变量 | 倾向得分匹配、回归调整 | 加权 |
| 有政策变化，有对照组 | DID | 合成控制 |
| 政策有明确的阈值 | RDD | 无 |
| 有外生冲击作为工具变量 | IV | 无 |
| 需要评估不同用户群体的差异 | 因果森林、Uplift模型 | 分层回归 |
