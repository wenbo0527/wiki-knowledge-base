# Superpowers 产品经理实战指南

> 专为产品经理定制的 Superpowers 工作流落地指南
> 适用场景：需求拆解、PRD生成、知识库整理、数据中台/营销工具设计

---

## 产品经理专属工作流

### 场景 1：需求拆解

#### 输入
- 业务方给的一句话需求
- 模糊的业务目标

#### Superpowers 工作流

```
Brainstorming（澄清&设计）
  ↓ 输出：需求澄清问题清单
Writing Plans（拆任务）
  ↓ 输出：PRD结构大纲
TDD（测试先行）
  ↓ 输出：验收标准
Code Review（审查）
  ↓ 输出：完整PRD
Finish（交付）
```

#### 专属模板

```markdown
## 需求拆解任务卡

### Brainstorming（30分钟）
**澄清问题**：
1. 这个需求的用户是谁？解决什么问题？
2. 成功的标准是什么？（可量化指标）
3. 时间约束？上线deadline？
4. 技术约束？必须用什么技术栈？

**方案对比**：
| 方案 | 优点 | 缺点 | 风险 |
|:---|:---|:---|:---|
| 方案A： | | | |
| 方案B： | | | |

**批准**：方案__已确认，进入Writing Plans

---

### Writing Plans（拆任务）

**PRD 结构大纲**：
```
1. 需求背景
2. 用户故事
3. 功能范围
4. 详细功能描述
   4.1 功能1：...
   4.2 功能2：...
5. 非功能需求
6. 验收标准
```

**任务拆分**（每个2-5分钟）：
1. 【2分钟】填充"需求背景"
2. 【3分钟】定义"功能范围"的IN/OUT清单
3. 【5分钟】写出第一个功能的详细描述
...

---

### TDD（验收标准）

**验收标准模板**：
```markdown
### [功能名] 验收标准

#### 功能1：[功能描述]

**Given** [前置条件]
**When** [用户操作]
**Then** [预期结果]

**验收Checklist**：
- [ ] 正常路径
- [ ] 边界条件
- [ ] 异常处理
- [ ] 性能指标
```

---

### Code Review（审查清单）

**PRD 审查Checklist**：
- [ ] 需求背景是否清晰？
- [ ] 用户故事是否符合INVEST原则？
- [ ] 功能边界是否明确？
- [ ] 验收标准是否可测试？
- [ ] 非功能需求是否完整？
- [ ] 风险评估是否考虑？

---

### Finish（交付物）

**最终交付**：
- [ ] PRD 文档（Markdown/PDF）
- [ ] 验收标准清单
- [ ] 风险评估报告
```

---

### 场景 2：PRD 生成

#### 输入
- 已确认的需求
- 验收标准

#### Superpowers 工作流

```
Brainstorming
  ↓ 确定PRD结构
Writing Plans
  ↓ 按模块拆任务
TDD
  ↓ 每个功能配验收标准
Code Review
  ↓ 检查完整性
Finish
  ↓ 输出标准PRD
```

## 极简指令（直接发给AI用）

### 场景 1：需求拆解

```
请使用 Superpowers 工作流完成需求拆解：
1. 先澄清需求，给出3个澄清问题
2. 给出2-3个方案对比，等我批准
3. 拆成2-5分钟的任务（PRD章节）
4. 每个功能配验收标准（Given/When/Then）
5. 输出标准PRD结构
```

### 场景 2：PRD生成

```
请使用 Superpowers 工作流完成PRD：
1. 先确认PRD类型（功能型/数据型/营销型/API型）
2. 按章节拆任务（2-5分钟/章）
3. 每章配验收标准
4. 审查完整性（检查清单）
5. 输出标准PRD文档
```
