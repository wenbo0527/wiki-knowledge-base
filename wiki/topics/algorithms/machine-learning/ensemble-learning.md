## 集成学习是什么？

**核心思想**：训练多个不同的模型，结合它们的预测结果，得到比单个模型更好的效果。

**两个关键条件**：
1. 基学习器之间要有差异（多样性）
2. 每个基学习器的效果至少要比随机猜测好（准确率>50%）

```
单个弱学习器 → 组合 → 强学习器
（效果一般）        （效果好，泛化能力强）
```

---

## 三大类别
| 类别 | 核心思想 | 代表算法 | 特点 |
|------|----------|----------|------|
| **Bagging** | 并行训练多个独立的基学习器，预测时投票/平均 | 随机森林 | 降低方差，抗过拟合 |
| **Boosting** | 串行训练基学习器，每个学习器学习前一个学习器的残差/错误 | GBDT、XGBoost、LightGBM | 降低偏差，效果好 |
| **Stacking** | 训练多层模型，第一层模型的预测结果作为第二层模型的输入 | 多模型融合 | 竞赛中常用，效果最好，复杂度高 |

## 1. Bagging (Bootstrap Aggregating)
### 核心流程
```
1. 有放回采样（Bootstrap采样）：每次从原始数据中采样N个样本（和原始数据大小相同，有重复），采样M次，得到M个数据集
2. 每个数据集训练一个基学习器，共M个独立的基学习器
3. 预测时：
   - 分类任务：M个模型投票，得票最多的类别为最终结果
   - 回归任务：M个模型的预测值取平均
```
### 核心优点
- 降低方差：多个模型平均减少了单模型的不稳定性，抗过拟合能力强
- 可以并行训练：基学习器之间独立，训练速度快
### 代表算法：随机森林 (Random Forest)
**改进点**：
- 基学习器用决策树
- 除了样本采样，还增加了特征采样：每次分裂节点时，随机选择k个特征，从中选最优分裂
- 进一步增加了基学习器的多样性，降低了树之间的相关性，提升了泛化能力
**参数调优**：
| 参数 | 作用 | 常用值 |
|------|------|--------|
| n_estimators | 树的数量 | 100-1000，越多效果越好，但训练越慢 |
| max_depth | 树的最大深度 | 不限/较大值，因为Bagging抗过拟合 |
| max_features | 每次分裂的特征采样数 | 分类：√n_features，回归：n_features/3 |
| min_samples_leaf | 叶子节点最小样本数 | 1-10 |

## 2. Boosting
### 核心流程
```
1. 初始化样本权重为1/N
2. 串行训练M个基学习器：
   a. 用当前样本权重训练第i个基学习器
   b. 计算基学习器的错误率，根据错误率给学习器分配权重（错误率越低，权重越高）
   c. 更新样本权重：预测错误的样本权重增加，预测正确的样本权重减少，让下一个学习器更关注错误样本
3. 预测时：加权平均所有基学习器的结果
```
### 核心优点
- 降低偏差：每个学习器都关注前一个学习器的错误，不断拟合残差，拟合能力强
- 效果比Bagging好，工业界首选
### 代表算法
#### 2.1 AdaBoost
- 核心：给错误样本增加权重，基学习器用决策树桩（单层决策树）
- 现在用得少，效果不如GBDT

#### 2.2 GBDT (Gradient Boosting Decision Tree)
- 核心：用梯度下降的思想，每个新的学习器拟合之前所有学习器的损失函数的负梯度（残差是平方损失的特例）
- 基学习器用CART回归树
- 优点：效果好，缺点：串行训练速度慢

#### 2.3 XGBoost/LightGBM/CatBoost
> 三大工业界主流GBDT实现，之前监督学习已经详细讲过，这里总结对比：
| 算法 | 核心优势 | 适用场景 |
|------|----------|----------|
| XGBoost | 功能完善，正则化强，效果稳定 | 通用场景，竞赛首选 |
| LightGBM | 速度快，内存占用小，支持类别特征 | 大数据集，工业界在线训练 |
| CatBoost | 自动处理类别特征，减少过拟合 | 类别特征多的数据集 |

### Bagging vs Boosting对比
| 维度 | Bagging | Boosting |
|------|---------|----------|
| 训练方式 | 并行 | 串行 |
| 关注点 | 降低方差，抗过拟合 | 降低偏差，提升拟合能力 |
| 基学习器要求 | 高方差、低偏差的复杂模型（如深度决策树） | 低方差、高偏差的简单模型（如浅决策树） |
| 对异常值敏感 | 不敏感 | 敏感 |
| 效果 | 略差 | 更好 |

## 3. Stacking
### 核心流程
```
两层模型结构：
1. 第一层（基模型层）：
   a. 训练集做K折交叉验证，每个基模型输出K份预测结果，拼接起来作为第二层的训练特征
   b. 每个基模型在全量训练集上训练，对测试集预测，结果取平均作为第二层的测试特征
2. 第二层（元模型层）：
   a. 用第一层的输出作为特征，训练元模型（通常用简单模型如线性回归、逻辑回归）
   b. 元模型的输出作为最终预测结果
```
### 核心要点
1. 第一层的基模型要尽量多样，差异越大越好：
   - 不同类型的模型：树模型、线性模型、SVM、神经网络
   - 不同参数的模型
   - 不同特征的模型
2. 第二层的元模型要简单，防止过拟合
3. 必须用K折交叉验证生成第二层的特征，不能直接用训练集的预测结果，否则会过拟合
### 适用场景
- 数据竞赛中提升最后0.1-0.5个百分点的效果
- 工业界中效果优先级远高于速度和可解释性的场景
### 缺点
- 复杂度高，训练速度慢
- 可解释性差
- 容易过拟合，调参难度大

## 4. 集成学习最佳实践
### 4.1 集成策略
#### 分类任务
| 策略 | 说明 | 适用场景 |
|------|------|----------|
| 硬投票 | 每个模型一票，得票最多的类别为结果 | 基模型效果差不多 |
| 软投票 | 每个模型输出类别概率，加权平均，概率最高的为结果 | 效果比硬投票好，基模型能输出概率 |
| 加权投票 | 效果好的模型权重高，效果差的权重低 | 基模型效果差异大 |

#### 回归任务
| 策略 | 说明 |
|------|------|
| 平均法 | 所有模型预测值取平均 |
| 加权平均 | 效果好的模型权重高 |
| 中位数法 | 取中位数，对异常值鲁棒 |

### 4.2 提升集成效果的方法
1. **增加基模型的多样性**：
   - 不同算法：树模型、线性模型、深度学习模型
   - 不同参数：同一算法不同参数
   - 不同特征：不同特征子集训练
   - 不同样本：不同采样的数据集
   - 不同随机种子：保证每次训练的模型有差异
2. **基模型的效果至少要比随机猜测好**：太差的模型加入会拉低整体效果
3. **优先用同质集成还是异质集成？**
   - 效果优先级：异质集成（不同类型模型）> 同质集成（同类型不同参数）
   - 速度优先级：同质集成 > 异质集成
   - 工业界：通常先尝试多个不同参数的LightGBM/XGBoost集成，效果提升明显，复杂度低
   - 竞赛：多种类型模型混合Stacking，效果最好

### 4.3 工业界落地建议
| 阶段 | 方案 |
|------|------|
| 基线阶段 | 单个LightGBM/XGBoost，调参到最优 |
| 提升阶段 | 多个不同参数的LightGBM/XGBoost软投票/平均，提升0.2-0.5个百分点，复杂度低，速度快 |
| 极致优化阶段 | 异质集成+Stacking，再提升0.1-0.3个百分点，复杂度高，适合效果优先的场景 |

### 4.4 常见坑
1. **基模型相关性太高**：集成效果提升不明显，甚至下降。保证基模型的多样性是关键。
2. **基模型太少或太多**：一般5-20个基模型性价比最高，太多了提升边际收益低，训练慢。
3. **Stacking不用交叉验证**：直接用训练集的预测结果训练第二层模型，严重过拟合，线上效果差。
4. **第二层模型太复杂**：容易过拟合，第二层用简单模型即可。

## 5. 经典案例
### 竞赛场景
Kaggle竞赛中，前几名的方案几乎都用了集成学习，通常是10+个不同类型的模型Stacking，比单个模型提升1-3个百分点。
### 工业界场景
- 风控：多个不同特征的XGBoost集成，提升模型稳定性，减少Bad Case
- 推荐：召回阶段多路召回，排序阶段多模型融合，提升推荐效果
- 广告：CTR预估多模型融合，提升点击率
