## 技术架构

### 金融LLM典型架构

```
┌─────────────────────────────────────────────────────────────┐
│  金融大模型 = 基础LLM + 金融知识库 + 领域微调 + 合规审核    │
└─────────────────────────────────────────────────────────────┘

基础层: LLM基座 (GPT-4/Claude/文心/混元)
      ↓
知识层: 金融知识图谱 + 监管规则库 + 产品数据库
      ↓
微调层: RLHF对齐 + 金融术语强化 + 安全过滤
      ↓
应用层: 智能客服 / 投顾 / 风控 / 运营
      ↓
合规层: 内容审核 + 监管报送 + 审计追溯
```

### 技术选型矩阵

| 场景 | 推荐模型 | 微调方式 | 知识增强 |
|------|---------|---------|---------|
| **智能客服** | GPT-4/Claude | SFT+RLHF | RAG检索 |
| **投顾建议** | 金融垂类模型 | LoRA微调 | 知识图谱 |
| **风控报告** | 文心/混元 | SFT | 规则引擎 |
| **内部知识库** | 开源模型(70B) | PEFT | 向量检索 |

## 落地挑战

### 核心挑战

| 挑战 | 描述 | 应对策略 |
|------|------|----------|
| **幻觉问题** | LLM生成不准确金融信息 | RAG增强+规则校验+人工复核 |
| **合规要求** | 金融产品推荐需持牌 | 答案分级+敏感词过滤 |
| **实时性** | 金融市场变化快 | 实时数据注入+定时更新 |
| **可解释性** | 监管要求决策可解释 | CoT推理+证据链展示 |
| **数据安全** | 金融数据高度敏感 | 私有化部署+数据脱敏 |
