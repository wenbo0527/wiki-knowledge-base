## 📌 核心洞察

> 用户根本不在乎AI多会"说话"，他们在乎的是AI能不能替他们把活干完。

### AI产品进化三阶段

| 阶段 | 产品 | 特点 |
|------|------|------|
| **第一代** | ChatGPT们 | 知而不行的"嘴" |
| **第二代** | Claude Skills/Manus | 租来的"工具箱" |
| **第三代** | OpenClaw | 知行合一的"数字武力" |

## 💡 OpenClaw = 主权代理

### 架构组成

| 组件 | 说明 |
|------|------|
| **前台** | 对接微信/飞书 |
| **大脑** | 调用各类大模型 |
| **双手** | 40+独立工具（浏览器、文件操作） |
| **档案柜** | 本地记忆 |

### 核心转变

```
过去：画界面，让用户用得顺手
未来：设计工作流，让用户"不用动手就能拿到结果"
```

## ⚠️ Agent落地四道门槛

### 1. 部署与稳定性

**问题**：环境配置灾难，面向大众产品必须开箱即用

### 2. 算力成本

**核心**：引入"模型路由"（Model Routing）
- 简单任务 → 开源小模型
- 关键决策 → 调用大模型

### 3. 权限让渡

**PM核心能力转变**：
```
从"定义功能" → "定义边界"
```

**必须设置**：
- AI能做什么/不能做什么
- 高危操作的人类确认节点

### 4. 真实高频场景

**策略**：
- 大厂争平台 → 我们争场景
- 垂直场景深耕（财税自动化、法律案卷）
