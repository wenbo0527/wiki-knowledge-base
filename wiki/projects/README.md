## 🎯 项目管理说明

本目录用于存放所有产品管理项目的文档，由**派蒙**作为总协调进行维护。

### 项目生命周期

```
概念阶段 → 设计阶段 → 开发阶段 → 上线阶段 → 运营阶段
   │           │           │           │           │
   ▼           ▼           ▼           ▼           ▼
 想法确认    PRD编写    Sprint规划   灰度发布   效果追踪
```

### 团队分工

| 角色 | 职责 |
|------|------|
| **派蒙** | 总协调：任务分解、进度跟踪、协调桥接、状态汇报 |
| **尼克·弗瑞** 🕵️ | 情报分析：市场调研、竞品分析、用户研究、需求输入 |
| **钟离** ⚔️ | 技术实现：架构设计、技术选型、开发实现 |
| **文博** | 最终决策：方向确认、关键决策、审批 |

## 🚀 创建新项目

当文博提出新项目想法时：

```
1. 派蒙创建项目目录
   └── wiki/projects/{project-name}/

2. 派蒙初始化README.md
   └── 使用项目页模板

3. 派蒙分配任务
   ├── 情报任务 → 尼克
   └── 技术任务 → 钟离

4. 尼克/钟离执行并更新文档

5. 派蒙跟踪进度并汇报
```
