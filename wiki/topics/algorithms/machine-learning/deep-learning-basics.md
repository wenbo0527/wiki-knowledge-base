## 深度学习是什么？
**核心思想**：用多层神经网络自动学习数据中的特征，不需要人工做特征工程。
```
传统机器学习：人工特征工程 → 模型训练
深度学习：原始数据 → 多层神经网络自动提取特征 → 输出结果
```
**核心优势**：
- 不需要人工特征工程，尤其适合非结构化数据（文本、图像、音频、视频）
- 数据量越大，效果越好，传统机器学习在数据量达到一定程度后效果会饱和，深度学习会持续提升
- 可以处理复杂的模式和任务（如大语言模型、图像生成、自动驾驶）

## 基础概念
### 1. 神经元 (Neuron)
神经网络的基本单元，模拟生物神经元：
```
输出 = 激活函数( 权重矩阵 × 输入 + 偏置 )
y = σ( W·X + b )
```
**激活函数**：引入非线性，让神经网络可以拟合任意复杂的函数
| 激活函数 | 表达式 | 适用场景 | 优点 | 缺点 |
|----------|--------|----------|------|------|
| Sigmoid | σ(x) = 1/(1+e^-x) | 二分类输出层 | 输出在[0,1]之间，可表示概率 | 梯度消失，计算慢，输出不是0均值 |
| Tanh | tanh(x) = 2σ(2x)-1 | 隐藏层 | 输出在[-1,1]之间，0均值 | 还是有梯度消失问题 |
| ReLU | f(x)=max(0,x) | 隐藏层，最常用 | 计算快，缓解梯度消失问题 | 神经元死亡问题（负数部分梯度为0，永远不更新） |
| Leaky ReLU | f(x)=max(αx, x), α很小 | 隐藏层 | 解决ReLU的神经元死亡问题 | |
| GELU | f(x)=x*Φ(x), Φ是正态分布累积函数 | Transformer、BERT、GPT | 效果比ReLU好，是目前主流 |
| Softmax | f(x_i)=e^x_i/Σe^x_j | 多分类输出层 | 输出是概率，和为1 | |

### 2. 神经网络结构
#### 2.1 前馈神经网络 (FNN/DNN)
最简单的神经网络，层与层之间全连接：
```
输入层 → 隐藏层1 → 隐藏层2 → ... → 输出层
```
**特点**：结构简单，适合结构化数据、特征已经处理好的场景

#### 2.2 卷积神经网络 (CNN)
适合处理网格结构的数据：图像（2D网格）、时序（1D网格）
**核心组件**：
- 卷积层：提取局部特征，参数共享，减少参数量
- 池化层：下采样，减少特征维度，提高鲁棒性
- 全连接层：输出最终结果
**适用场景**：图像分类、目标检测、图像分割、时序分类

#### 2.3 循环神经网络 (RNN/LSTM/GRU)
适合处理序列数据，有记忆能力，能处理变长序列
**问题**：RNN有梯度消失问题，长序列记忆能力差
**改进**：LSTM/GRU引入门控机制，解决长序列记忆问题
**适用场景**：文本分类、机器翻译、语音识别（现在已经被Transformer替代）

#### 2.4 Transformer
目前主流的深度学习架构，大语言模型的基础
**核心组件**：
- 自注意力机制：让模型可以关注输入序列中不同位置的相关信息
- 多头注意力：多个独立的注意力头，学习不同的模式
- 位置编码：给序列加入位置信息
- 残差连接+层归一化：训练更深的网络
**架构分类**：
| 类型 | 代表 | 适用场景 |
|------|------|----------|
| 编码器-only | BERT | 自然语言理解（分类、匹配、抽取） |
| 解码器-only | GPT系列 | 自然语言生成、大语言模型 |
| 编码器-解码器 | T5、Transformer翻译模型 | 序列到序列任务（翻译、摘要） |
**优势**：
- 并行计算能力强，比RNN训练速度快
- 长距离依赖建模能力强
- 通用性强，文本、图像、音频都可以用（多模态模型）

## 深度学习训练要点
### 1. 损失函数
| 任务类型 | 损失函数 |
|----------|----------|
| 二分类 | 二元交叉熵 (Binary Cross Entropy) |
| 多分类 | 多分类交叉熵 (Categorical Cross Entropy) |
| 回归 | MSE、MAE |
| 序列生成 | 交叉熵、CTC损失 |

### 2. 优化器
| 优化器 | 特点 | 适用场景 |
|------|------|----------|
| SGD | 收敛慢，需要调学习率，泛化能力可能更好 | 数据量大，有足够调参时间 |
| Adam | 自适应学习率，收敛快，最常用 | 大部分场景，默认选择 |
| AdamW | Adam+权重衰减，效果更好，泛化能力强 | Transformer、大模型训练首选 |
| RMSProp | 适合RNN | |

### 3. 正则化方法
防止过拟合：
| 方法 | 说明 |
|------|------|
| L1/L2正则 | 损失函数加权重的惩罚项 |
| Dropout | 训练时随机让一部分神经元失活，减少神经元之间的依赖 |
| 早停 | 验证集效果不再提升时停止训练 |
| 数据增强 | 对输入数据做变换，扩充训练集（如图像翻转、裁剪；文本同义词替换、掩码） |
| 权重衰减 | 限制权重的大小 |
| 层归一化/批量归一化 | 稳定训练，加快收敛，减少过拟合 |

### 4. 训练流程
```
1. 数据准备 → 数据清洗、标注、划分训练/验证/测试集
2. 模型结构设计 → 根据任务选择合适的结构（DNN/CNN/Transformer）
3. 损失函数/优化器选择 → 根据任务类型选择
4. 超参数设置 → 学习率、批大小、迭代次数、Dropout比例等
5. 训练 → 监控训练集/验证集损失、指标，用早停防止过拟合
6. 评估 → 在测试集评估，和基线对比
7. 上线部署 → 模型压缩、量化，部署为API服务
```

## 工业界最佳实践
### 1. 什么时候用深度学习？
✅ **优先用深度学习的场景**：
- 非结构化数据：文本、图像、音频、视频
- 数据量很大：百万/千万级以上样本
- 任务复杂：传统机器学习效果不好
- 特征工程成本高：需要大量人工提取特征的场景
❌ **不要用深度学习的场景**：
- 结构化数据，小数据集：传统树模型（XGBoost/LightGBM）效果更好，速度更快，可解释性更强
- 对可解释性要求极高：如风控、医疗，深度学习黑盒难以解释
- 计算资源不足：深度学习需要GPU，训练和推理成本高
- 需要快速迭代：深度学习训练慢，树模型可以快速迭代

### 2. 落地建议
**结构化数据场景**：
- 优先用LightGBM/XGBoost，90%的场景下效果和深度学习差不多，速度快10倍，可解释性强
- 数据量极大（千万级以上），特征维度高的场景下，可以尝试DNN，可能会有1-3个百分点的提升

**非结构化数据场景**：
- 优先用预训练模型微调，不要从零训练
- 文本用BERT/GPT类预训练模型
- 图像用ResNet/ViT类预训练模型
- 微调成本低，效果好，比从零训练节省90%的时间和数据

**部署优化**：
- 模型压缩：剪枝、蒸馏、量化，减少模型大小，加快推理速度
- 推理引擎：用TensorRT、ONNX Runtime等优化推理引擎，速度提升几倍到几十倍
- 服务化：用FastAPI、Triton等部署为API服务，支持高并发

### 3. 调参要点
**最重要的参数**：
1. 学习率 (Learning Rate)：最影响效果，常用1e-5到1e-3，Transformer类预训练模型微调用1e-5到5e-5
2. 批大小 (Batch Size)：越大训练越稳定，受GPU显存限制
3. 迭代次数 (Epoch)：配合早停使用，不需要手动设置
4. Dropout比例：0.1-0.5，防止过拟合
**调参技巧**：
- 用学习率搜索找到最优初始学习率
- 用学习率衰减策略，训练过程中逐步降低学习率
- 小批量数据集上先试跑，找到合适的参数范围再全量训练

## 大模型时代的深度学习
2023年之后，大语言模型（LLM）成为主流，深度学习落地范式发生变化：
```
传统范式：针对每个任务，采集数据，标注，训练小模型
新范式：基于通用大模型，用少量数据微调/提示工程/Agent开发，适配具体任务
```
**优势**：
- 不需要大量标注数据，少量样本即可得到很好的效果
- 通用能力强，一个模型可以适配多个任务
- 理解和生成能力远超过传统小模型
**适用场景**：
- 自然语言理解、生成类任务
- 多模态任务
- 通用智能场景
