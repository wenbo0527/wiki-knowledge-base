## OpenClaw 龙虾实战

### 11人Agent阵容（PM配置）

| Agent角色 | 职责 | PM使用场景 |
|-----------|------|------------|
| 📝 文档撰写 | PRD、周报 | 自动化文档生成 |
| 🔍 信息收集 | 竞品分析、行业研究 | 市场调研 |
| 📊 数据分析 | 数据整理、报告 | 数据驱动决策 |
| 🎨 原型助手 | UI描述、交互设计 | 快速原型 |
| 💬 对外沟通 | 邮件、报告 | 客户沟通 |
| 🔧 技术评审 | 代码审核、技术方案 | 技术可行性 |
| 📅 日程管理 | 会议安排、提醒 | 时间管理 |
| 📚 知识管理 | 文档整理、归档 | 知识库维护 |
| 🔮 预测分析 | 趋势预测、风险评估 | 战略规划 |
| ✍️ 内容创作 | 文案、方案 | 营销物料 |
| 🎯 质量把控 | 审核、检查 | 质量保证 |

### PM的3步上手指南

#### Step 1: 定义角色
```markdown
你是文博的产品经理助手，专注于[领域]，
擅长[技能]，工作风格是[风格描述]。
```

#### Step 2: 建立工作流
```markdown
任务类型 → 触发Agent → 审核输出 → 交付
```

#### Step 3: 持续优化
- 记录有效的Prompt模板
- 优化Agent协作流程
- 建立质量检查清单

## Claude Code 实践

### PM用例

| 场景 | 使用方式 | 效果 |
|------|----------|------|
| 快速验证想法 | 用自然语言描述功能 | 快速看到原型 |
| 技术可行性评估 | 让AI实现技术方案 | 节省沟通时间 |
| 竞品功能分析 | 让AI分析竞品代码 | 深入理解实现 |

### PM注意事项

| ✅ 适合 | ❌ 不适合 |
|--------|----------|
| 快速原型 | 生产级代码 |
| 技术调研 | 核心业务逻辑 |
| 自动化脚本 | 复杂系统 |
