## 推荐工具栈


### 方案2：进阶型

```
移动端：Get笔记

同步：GitHub
```

### 方案3：团队协作型

```

```

## 工具选择原则

### 1. 数据可迁移

| 好的选择 | 不好的选择 |
|----------|------------|
| Obsidian（纯Markdown） | Notion（专有格式） |
| Logseq（纯Markdown） | 印象笔记（封闭格式） |
| GitHub（开放） | 语雀（封闭） |

### 2. 本地优先

- 优先选择本地存储的工具
- 云同步只是备份，不是主存储
- 离线可用性很重要

### 3. 链接优先

- 选择支持双向链接的工具
- 链接是知识网络的基础

### 4. 简约够用

- 不要被工具绑架
- 用最少的工具做最多的事
- 定期清理不用的工具
