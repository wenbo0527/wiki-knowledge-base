## 三大方法论对比

| 方法 | 核心理念 | 优点 | 缺点 | 适合人群 |
|------|----------|------|------|----------|
| **Zettelkasten** | 原子化+链接 | 知识自然涌现 | 上手慢 | 学术研究者 |
| **PARA** | 项目导向 | 实用性强 | 分类压力大 | 知识工作者 |
| **LYT** | 流动性+框架 | 灵活度高 | 依赖每日坚持 | 笔记爱好者 |

## Zettelkasten（卡片盒）

### 核心原则

```
1. Atomic（原子化）
   └── 每个笔记只包含一个想法

2. Linked（链接）
   └── 笔记之间通过双向链接连接

3. Emergence（涌现）
   └── 笔记多了会自动形成知识网络
```

### 笔记类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **Fleeting** | 临时想法 | 闪念笔记 |
| **Literature** | 阅读笔记 | 书/文章摘录 |
| **Permanent** | 永久笔记 | 自己的思考 |

### 工作流

```
阅读/学习
    │
    ▼
Literature Notes（原文摘录）
    │
    ▼
理解后用自己话重写
    │
    ▼
Permanent Notes（永久笔记）
    │
    ▼
建立链接 → 形成网络
```

## PARA方法

### 四类内容

| 类型 | 定义 | 时间跨度 |
|------|------|----------|
| **Projects** | 正在做的有明确目标的事 | <3个月 |
| **Areas** | 长期负责的领域 | 持续进行 |
| **Resources** | 感兴趣但暂时用不到 | 随时 |
| **Archives** | 完成后不再活跃的 | 已结束 |

### 组织原则

```
文件夹结构
├── 01_Projects/
│     ├── 产品规划/
│     └── 技术方案/
├── 02_Areas/
│     ├── 技术架构/
│     └── 职业发展/
├── 03_Resources/
│     ├── AI研究/
│     └── 心理学/
└── 04_Archives/
      └── 旧项目/
```

### 核心思想

**"项目优先"** — 一切都围绕当前项目组织

## LYT框架

### 核心组件

| 组件 | 说明 |
|------|------|
| **Daily Notes** | 每日入口，捕获当天想法 |
| **Fleeting Notes** | 临时存储，未经整理 |
| **Literature Notes** | 阅读时做的笔记 |
| **Permanent Notes** | 整理后的永久笔记 |

### 特性

```
LYT的独特之处

1. 流动性
   └── Daily Notes是一切入口

2. 框架页面
   └── 创建MOC（Map of Content）聚合相关笔记

3. 渐进式
   └── Fleeting → Literature → Permanent
```

## 工具支持

| 方法 | 最佳工具 | 原因 |
|------|----------|------|
| **Zettelkasten** | Obsidian | 双向链接+图谱 |
| **PARA** | Notion | 文件夹+数据库 |
| **LYT** | Logseq | 大纲+双向链接 |
| **通用** | Roam Research | 块级引用 |

## 如何选择？

### 选择指南

| 如果你... | 选择... |
|-----------|---------|
| 是学术研究者 | Zettelkasten |
| 注重项目产出 | PARA |
| 喜欢灵活组织 | LYT |
| 需要协作 | Notion |
| 喜欢本地优先 | Obsidian |
| 喜欢大纲写作 | Logseq |

### 组合使用

```
实际推荐

PARA作为顶层分类（文件夹）
    │
    └── Zettelkasten作为笔记方法（链接）

例子：
├── 01_Projects/产品规划/
│     ├── 竞品分析.md（链接到03_Resources/AI研究/）
│     └── 用户访谈.md（链接到02_Areas/职业发展/）
```
