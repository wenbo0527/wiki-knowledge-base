## 关联网络核心价值
传统的风控特征都是基于用户个体的，而欺诈很多都是团伙作案，关联网络可以挖掘用户之间的隐藏关系，识别团伙欺诈，是反欺诈的核心武器：
- 识别团伙欺诈：传统特征识别不了的团伙作案，关联网络识别率超过80%
- 挖掘隐藏风险：用户本身资质良好，但关联了很多高风险用户，未来逾期风险是普通用户的3倍以上
- 提升反欺诈效果：关联网络特征可以将反欺诈模型的KS值提升0.1以上，减少欺诈损失30%以上
- 降低误杀率：结合关联关系可以区分正常用户和欺诈用户，降低误杀，提升用户体验
> **真实案例**：某消费金融公司通过关联网络识别出一个200多人的欺诈团伙，该团伙伪造资料批量申请贷款，传统风控模型只识别出30%的成员，通过关联网络识别出90%以上的成员，避免损失超过5000万。

## 关联网络核心架构
### 实体类型（节点）
消费金融场景下的核心实体：
| 实体类型 | 唯一标识 | 说明 |
|----------|----------|------|
| 用户 | 身份证号、用户ID | 核心实体 |
| 设备 | 设备ID、IMEI、MAC地址 | 手机、电脑等设备 |
| 手机号 | 手机号 | 本人手机号、紧急联系人手机号 |
| 银行卡 | 银行卡号 | 放款、还款银行卡 |
| 地址 | 收货地址、居住地址、工作地址 | 标准化后的地址 |
| IP地址 | IP地址 | 登录、交易IP |
| 联系人 | 通讯录联系人、紧急联系人 | 社交关系 |
| 商户 | 交易商户 | 消费交易的商户 |
### 关系类型（边）
实体之间的关联关系：
| 关系类型 | 说明 | 权重 |
|----------|------|------|
| 设备关联 | 用户使用过同一设备 | 高 |
| 手机号关联 | 用户使用同一手机号、或紧急联系人相同 | 高 |
| 银行卡关联 | 用户使用同一张银行卡 | 极高 |
| 地址关联 | 用户的居住/收货/工作地址相同 | 中 |
| IP关联 | 用户使用同一IP地址登录/交易 | 低 |
| 联系人关联 | 用户的通讯录有共同好友、或紧急联系人相同 | 中 |
| 交易关联 | 用户之间有转账交易、或有共同交易对手 | 中 |
| 社交关联 | 用户在社交平台是好友 | 低 |
### 网络构建
```
实体 + 关系 = 关联网络知识图谱
每个实体是节点，关系是边，边有权重，代表关联的紧密程度
```

## 核心关联特征体系
### 1. 基础关联特征（IV值0.2-0.4，强特征）
基于直接关联的统计特征，计算简单，效果好。
| 特征类别 | 典型特征 | 业务含义 | 风险阈值 |
|----------|----------|----------|----------|
| 关联风险用户数 | 1度关联的黑名单用户数、逾期用户数、欺诈用户数 | 直接关联的高风险用户数量，越多风险越高 | >2个直接关联高风险用户，风险是普通用户的3倍以上，>5个直接拒绝 |
| 关联关系数量 | 同一设备关联的用户数、同一手机号关联的用户数、同一地址关联的用户数 | 关联的用户越多，越可能是中介、团伙 | 同一设备关联>3个用户，直接拒绝；同一地址关联>5个用户，高风险 |
| 关联密度 | 1度关联用户中高风险用户占比 | 关联的用户中风险用户的比例，越高越危险 | 占比>30%，高风险，谨慎准入 |
| 关联关系类型 | 是否有关联高风险的强关系（银行卡、设备、手机号） | 强关系关联高风险用户的风险远高于弱关系 | 银行卡/设备关联高风险用户，直接拒绝 |
### 2. 图算法特征（IV值0.25-0.5，强特征，识别团伙）
基于图算法计算的高阶特征，识别团伙、社区等复杂关系。
| 算法类型 | 典型特征 | 业务含义 | 应用场景 |
|----------|----------|----------|----------|
| 社区发现 | Louvain算法计算社区、社区大小、社区内高风险用户占比、社区内用户平均风险分 | 识别关联紧密的用户群体，社区就是团伙 | 社区内高风险用户占比>20%，整个社区标记为高风险团伙，全部拒绝 |
| 中心性算法 | PageRank值、度中心性、中介中心性、接近中心性 | 识别团伙的核心人物、中介 | 中心性高的用户如果是高风险，其关联用户风险都很高 |
| 路径算法 | 最短路径长度、N度关联内是否有高风险用户、关联路径的类型 | 识别和高风险用户的间接关联 | 2度关联内有黑名单用户，风险提升2倍 |
| 标签传播 | 基于已知高风险用户的标签传播给关联用户，计算风险评分 | 扩散风险标签，发现更多隐藏的高风险用户 | 传播后的风险评分>阈值，标记为高风险 |
### 3. 团伙特征
团伙欺诈的典型特征：
| 团伙类型 | 典型网络特征 | 坏账率 |
|----------|----------|--------|
| 中介造假团伙 | 大量用户共享同一设备、同一IP、同一地址、同一联系人，资料相似 | >60% |
| 骗贷团伙 | 用户之间互相转账、互为紧急联系人，申请资料高度相似，集中申请 | >50% |
| 套现团伙 | 大量用户和同一商户交易，交易时间集中，交易金额相似 | >30% |
| 羊毛党团伙 | 大量新用户，同一设备/IP注册，参加同一优惠活动，后续无交易 | >20% |
#### 典型团伙识别规则
✅ 同一设备1个月内注册>5个账号 → 中介团伙，所有关联用户拒绝  
✅ 社区大小>20人，社区内高风险用户占比>30% → 欺诈团伙，全部拒绝  
✅ 10个以上用户互相为紧急联系人 → 骗贷团伙，全部拒绝

## 关联网络落地流程
### 1. 数据准备
- 收集所有需要的实体和关系数据，标准化实体的唯一标识，比如地址标准化、设备ID去重
- 数据清洗：去除异常数据、脏数据，比如无效的设备ID、格式错误的手机号
### 2. 网络构建
- 选择图数据库：Neo4j（小数据量，适合查询）、Nebula Graph/JanusGraph（大数据量，分布式）
- 导入实体和关系，构建知识图谱，建立索引，提升查询效率
### 3. 特征计算
- 离线特征：T+1计算批量关联特征，用于贷前模型、规则
- 实时特征：实时计算新注册、新申请用户的关联特征，用于实时反欺诈决策
### 4. 规则和模型开发
- 开发关联关系规则，比如同一设备关联超过3个用户直接拒绝
- 关联特征加入反欺诈模型，提升模型效果
- 开发团伙识别算法，自动识别欺诈团伙
### 5. 上线运营
- 灰度上线，验证规则和模型的准确率、误杀率
- 定期更新网络数据，迭代算法，识别新的欺诈模式
- 团伙识别后要人工审核，确认后再处置，避免误杀
> **落地成本参考**：中小公司基于Neo4j搭建关联网络，1个算法工程师+1个开发工程师2-3个月即可完成，成本低，效果明显。

## 最佳实践
### 1. 优先强关系特征
关联关系权重越高，风险相关性越强，优先使用：
强关系（权重高）：银行卡关联 > 设备关联 > 手机号关联 > 地址关联 > IP关联（权重低）
强关系关联高风险用户直接拒绝，弱关系关联可以作为辅助特征，不要直接拒贷，避免误杀。
### 2. 避免误杀正常用户
很多正常用户也会有合理的关联关系，比如一家人共用一个设备、一个地址，不要一刀切：
- 关联特征要和其他特征结合使用，不要只凭关联关系就拒贷
- 建立申诉机制，被误拒的用户可以申诉，人工审核
- 对于合理的关联（比如直系亲属），降低权重或者排除
### 3. 实时计算优先
欺诈团伙都是集中申请，实时关联特征可以在申请时就识别团伙，拦截欺诈，事后识别损失已经造成了，所以优先做实时关联计算。
### 4. 数据合规
关联网络使用的用户数据（通讯录、地址、设备信息等）必须获得用户的明确授权，符合《个人信息保护法》要求，不能超范围使用，不能泄露用户隐私。
### 5. 不断迭代欺诈模式
欺诈团伙的模式不断变化，要持续监控新的欺诈模式，更新关联规则和算法，比如现在很多团伙会用设备改码工具修改设备ID，要增加设备指纹、行为模式等辅助识别。

## 行业应用案例
### 案例1：反中介批量造假
**场景**：黑中介批量收集用户资料，用同一设备、同一IP批量注册申请贷款，包装资料骗贷。
**识别方法**：同一设备1个月内注册>3个账号，同一IP1天内注册>10个账号，关联的用户资料相似度高 → 标记为中介团伙，所有关联用户直接拒绝。
**效果**：中介欺诈识别率提升80%，减少损失2000万/年。
### 案例2：骗贷团伙识别
**场景**：团伙成员互相为紧急联系人，伪造收入证明、工作证明，集中申请贷款，到手后失联。
**识别方法**：用Louvain社区发现算法识别出关联紧密的社区，社区内用户的紧急联系人互相重叠，社区内已有逾期用户占比>30% → 标记为骗贷团伙，全部拒绝。
**效果**：识别出10+个骗贷团伙，每个团伙20-100人，避免损失超过5000万。
### 案例3：失联修复
**场景**：用户逾期后失联，找不到人。
**识别方法**：通过关联网络找到用户的联系人、共同设备、共同地址的其他用户，联系这些用户转告还款，或者找到用户的最新联系方式。
**效果**：失联用户回款率提升20%以上。
