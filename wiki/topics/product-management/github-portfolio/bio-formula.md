# Bio写作公式

> 让HR/面试官3秒内判断你是否是目标候选人
> 更新: 2026-04-27

---

## Bio的重要性

**HR看GitHub的时间：** ~3秒
**决定因素：** Bio + pinned repos

```
3秒内要传达的信息：
✓ 我是谁
✓ 我做什么
✓ 我擅长什么
```

---

## Bio公式

### 公式A: 基础版

```
[职称] | [垂直领域] | [核心技能]
```

**示例：**
```
数据中台产品经理 | CDP·MA·归因分析 | 金融科技
```

### 公式B: 进阶版

```
[职称] | [垂直领域] | [核心技能] | [差异化]
```

**示例：**
```
金融科技PM | 数据中台·营销套件 | CDP·MA·归因 | ex-头部互金
```

### 公式C: 完整版

```
[职称] + [经验年限] + [垂直领域] + [核心能力] + [数据亮点]
```

**示例：**
```
Lead AI PM | 8年金融科技 | 数据平台·CDP·Agent | $10M+项目交付
```

## 不要这样写

| ❌ 错误写法 | 问题 | ✅ 修改建议 |
|------------|------|------------|
| `PM` | 太泛 | `数据中台PM` |
| `Interested in AI and Data` | 没聚焦 | `AI数据PM` |
| `Product Manager at xxx` | 只有公司 | 加技能和领域 |
| `AI/ML/LLM/Python/SQL...` | 技术堆砌 | 精选3个关键词 |
| 超过160字符 | 太长 | 精简到80字内 |
