> 将新信息消化并写入Wiki的标准流程

## 🎯 流程说明

Ingest 是将外部信息（RSS文章、最佳实践、网页内容等）转化为 Wiki 知识的过程。

---

## 🔄 Ingest 流程图

```
┌─────────────────────────────────────────────────────────────┐
│                      新信息到达                              │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 判断信息类型                                         │
│  ├── 洞察类 → 是有价值的见解？→ insights/                     │
│  ├── 实体类 → 公司/人物/市场？→ entities/                     │
│  ├── 主题类 → 属于现有Topic？→ topics/                       │
│  └── 项目类 → 需要派蒙协调？→ projects/                      │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 2: 评估价值                                            │
│  ⭐⭐⭐⭐⭐ 卓越见解 → 立即写入insights/                        │
│  ⭐⭐⭐⭐ 高价值 → 补充现有体系                                │
│  ⭐⭐⭐ 有价值 → 补充现有专题                                  │
│  ⭐⭐ 待观察 → 暂存sources/待定                               │
│  ⭐ 无价值 → 归档不引入                                       │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 3: 写入文档                                            │
│  ├── 选择对应模板                                            │
│  ├── 使用洞察页模板 → insights/                              │
│  ├── 使用实体页模板 → entities/                              │
│  └── 使用主题页模板 → topics/                                │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 4: 关联与链接                                          │
│  ├── 添加双向链接                                            │
│  ├── 更新关联实体的关联内容                                   │
│  └── 在相关主题页添加引用                                      │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 5: 记录与更新                                          │
│  ├── 更新文档头部的"最后更新时间"                             │
│  ├── 在log.md中记录本次更新                                  │
│  └── 标注信息来源                                             │
└─────────────────────────────────────────────────────────────┘
```

## 📊 价值评估标准

| 评级 | 说明 | 处理方式 |
|------|------|---------|
| ⭐⭐⭐⭐⭐ | 卓越见解，开创新领域 | 立即写入insights/，通知派蒙 |
| ⭐⭐⭐⭐ | 高价值，扩展现有体系 | 写入insights/，补充相关Topic |
| ⭐⭐⭐ | 有价值，补充现有专题 | 更新相关Topic的内容 |
| ⭐⭐ | 待观察，补充有限 | 暂存sources/待定，定期回顾 |
| ⭐ | 无价值，不引入 | 归档到sources/rejected/ |

## 📝 洞察页 Ingest 清单

### 必填项
- [ ] 标题（一句话概括）
- [ ] 来源（RSS/最佳实践/手动）
- [ ] 收集时间
- [ ] 维护者
- [ ] 评级

### 内容项
- [ ] 核心要点（3-5条）
- [ ] 背景分析
- [ ] 关键发现
- [ ] 影响评估
- [ ] 洞察与启示

### 关联项
- [ ] 至少2个双向链接
- [ ] 来源引用标注
- [ ] 更新记录

## 📁 来源管理

### 来源类型标记

| 标记 | 类型 | 说明 |
|------|------|------|
| `[RSS]` | RSS简报 | 来自RSS情报系统 |
| `[BP]` | 最佳实践 | 来自最佳实践收集 |
| `[Manual]` | 手动输入 | 用户直接提供 |
| `[Ref]` | 参考资料 | 外部参考资料 |

### 来源引用格式

```markdown
## 📝 原始资料

- [RSS] [文章标题](URL) - 简要说明
- [BP] [最佳实践标题](URL) - 简要说明
```

## ⚠️ 不引入记录

对于评星较低的来源，记录但不引入 Wiki：

**路径**: `sources/rejected/{date}.md`

```markdown
# 不引入来源记录 - YYYY-MM-DD

## 文章1
- **标题**: {标题}
- **来源**: {来源}
- **评级**: ⭐
- **不引入原因**: {原因}

## 文章2
- ...
```
