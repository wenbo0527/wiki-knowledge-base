## 核心摘要

本文系统性分析了 **Manus、Claude Code、OpenClaw、Cursor、LangChain** 五家AI产品的上下文压缩方法论，核心观点：

> **上下文压缩的本质是注意力管理**，不是数据压缩，而是在有限注意力预算内让模型看到最关键信息。

## 关键洞察

### 1. 压缩必须可恢复

| 产品 | 方案 | 核心思路 |
|------|------|----------|
| **Manus** | Compact双版本 | 完整版+紧凑版，压缩后保留引用 |
| **Claude Code** | Tool Result Clearing | 清除旧结果但保留文件路径 |
| **OpenClaw** | 占位符替换 | 用引用替代全文 |
| **Cursor** | Dynamic Context Discovery | 按需加载，大输出写文件 |

> **关键原则**：压缩 ≠ 删除，只是暂时移出视野

### 2. KV Cache 是严重低估的优化维度

| 问题 | 影响 |
|------|------|
| 系统提示放时间戳 | KV Cache命中率直接归零 |
| 动态增删工具 | KV Cache全部失效 |
| JSON序列化键顺序不确定 | 缓存全部失效 |

> **Anthropic API**：缓存命中/未命中价格差 **10倍**

### 3. 管理优先级框架

```
优先级从轻到重：
┌─────────────────────────────────────────────────────────────┐
│  1. 管入口 ─── 少放不必要信息 (Cursor按需加载、Manus小工具集)  │
│  2. 管工具 ─── 工具结果最大token源 (截断、compact、清除)       │
│  3. 对话压缩 ─── Compaction / Summarization                 │
│  4. 拆Agent ─── 上下文隔离                                  │
│  5. 文件系统 ─── 外部记忆兜底                               │
└─────────────────────────────────────────────────────────────┘
能用简单手段解决就不上复杂方案
```

### 4. Manus 三板斧：Reduce / Offload / Isolate

#### Reduce（缩减）

**两阶段逐级加码**：

| 阶段 | 方法 | 说明 |
|------|------|------|
| **第一阶段** | Compaction | 完整版→紧凑版，保留引用 |
| **第二阶段** | Summarization | 预定义schema约束摘要格式 |

> 关键设计：所有压缩都是可恢复的

#### Offload（卸载）

- 把文件系统当"外挂大脑"
- 信息用完写到文件，需要时读回
- 工具只有不到20个原子函数，其他通过CLI暴露
- 核心思路：**像一本手册——从目录开始，然后是特定章节，最后是附录**

#### Isolate（隔离）

- 子Agent目标不是按角色分工，而是**隔离上下文**
- 三层架构：Planner分配任务、Knowledge Manager决定持久化、Executor执行
- 两种委派模式：简单任务传指令，复杂任务共享上下文

### 5. Claude Code 三层防线

| 层级 | 手段 | 说明 |
|------|------|------|
| **第一层** | Tool Result Clearing | 轻量，清除老工具结果，几乎无信息损失 |
| **第二层** | Compaction | 结构化摘要，保留任务进度+当前状态+已做决策 |
| **第三层** | 子Agent | 探索几万token返回1-2K摘要，10-20倍压缩比 |

### 6. OpenClaw 工程细节

| 参数 | 值 | 说明 |
|------|-----|------|
| 单工具结果上限 | 上下文窗口30% | 硬性限制 |
| 安全边距 | 1.2 (20%) | 补偿中文等多字节字符 |
| 摘要触发 | 体量180K左右 | 预留20K给下一轮 |
| 最近对话 | 保留3轮不压缩 | 保持连续性 |
| 大消息截断 | head+tail策略 | 保留首尾，中间省略 |
| 标识符保护 | 强制保留UUID/路径 | 摘要指令要求原文保留 |

#### ContextEngine 可插拔接口

```
ingest（摄入）→ assemble（组装）→ compact（压缩）
```

### 7. LangChain 自主压缩

| 触发时机 | 说明 |
|----------|------|
| 子任务刚完成 | 中间状态已转化为结果 |
| 大量数据提取完毕 | 原始数据不再需要 |
| 准备消费大量新信息前 | 腾出空间 |

> **核心假设**：Agent比固定规则更懂"现在是不是个好时机"

### 8. 其他工程坑

| 坑 | 说明 |
|-----|------|
| 别在系统提示开头放时间戳 | 精确到秒会把KV Cache命中率干到地板 |
| 保留错误，别擦除 | 移除证据=移除学习机会 |
| 小心few-shot陷阱 | 重复处理类似任务会让模型陷入"节奏" |
| 引入结构化随机性 | 不同序列化模板、替代措辞、格式变化 |

## 核心认知

> **上下文压缩的本质是注意力管理**
> 不是在做数据压缩，是在帮模型"聚焦"——确保模型当前决策所需要的信息在它的注意力范围内。

> **Karpathy定义**：
> "context engineering is the delicate art and science of filling the context window"
> 即"填充上下文窗口的精妙艺术和科学"
