## 2. 功能结构（按模块归档）

### 2.1 FEATURE-DEX_SHARE_MANAGE（分享管理）

#### 功能入口 & 作用

**入口位置**：数据分析 → 分享中心 / 报表详情 → 分享

**核心作用**：
- 多种分享方式（链接、邀请、团队）
- 分享权限精细配置
- 分享链接管理（有效期、密码）
- 分享记录和统计

#### 核心逻辑

```
分享配置流程：

Step 1: 选择分享内容
  - 选择分享对象（报表、图表、数据集）
  - 确认分享内容版本
  - 选择分享范围（个人/团队/公开）

Step 2: 配置分享方式
  - 链接分享（生成链接、设置密码）
  - 邀请分享（指定用户/部门）
  - 团队分享（加入团队后自动获得权限）

Step 3: 设置权限
  - 查看权限（只读、可导出）
  - 编辑权限（可修改内容）
  - 管理权限（可转让、删除）

Step 4: 配置高级选项
  - 分享有效期
  - 访问次数限制
  - 是否允许评论
  - 是否允许下载
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---|
| `share.default_validity` | 默认链接有效期（天）| `7` |
| `share.max_expiry_days` | 最大有效期（天）| `365` |
| `share.allow_password` | 是否允许密码保护 | `true` |
| `share.track_enabled` | 是否记录访问统计 | `true` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **分享链接泄露** | 链接被未授权人员访问 | 设置密码、限制IP、定期失效 |
| **权限过宽** | 分享权限过大导致数据泄露 | 默认最小权限、支持细粒度配置 |
| **分享过期** | 分享链接过期后无法访问 | 提醒续期、自动续期选项 |
| **权限回收难** | 分享后难以撤销权限 | 支持实时撤销、权限列表管理 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 自助分析 | 上游 | 分享的报表内容 |
| 权限中心 | 依赖 | 权限校验 |
| 消息中心 | 依赖 | 分享通知 |

### 2.2 FEATURE-DEX_COMMENT_COLLAB（评论协作）

#### 功能入口 & 作用

**入口位置**：报表详情页 → 评论区 / 图表详情 → 评论区

**核心作用**：
- 对分析内容进行评论讨论
- @提及相关人员
- 评论通知和回复
- 评论管理（置顶、删除）

#### 核心逻辑

```
评论协作流程：

Step 1: 发表评论
  - 选择评论内容（报表、图表、数据点）
  - 编写评论内容
  - @提及相关人员
  - 提交评论

Step 2: 评论通知
  - 被@用户收到通知
  - 作者收到回复通知
  - 支持邮件/消息多渠道

Step 3: 评论互动
  - 对评论进行回复
  - 评论点赞
  - 评论收藏

Step 4: 评论管理
  - 评论置顶
  - 评论删除
  - 评论审核（敏感词过滤）
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---:|
| `comment.max_length` | 最大评论长度 | `1000` |
| `comment.allow_mention` | 是否允许@提及 | `true` |
| `comment.moderation_enabled` | 是否启用审核 | `true` |
| `comment.notification_enabled` | 是否发送通知 | `true` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **评论通知泛滥** | 大量评论导致通知轰炸 | 合并通知、免打扰模式 |
| **敏感信息泄露** | 评论中包含敏感信息 | 内容审核、敏感词过滤 |
| **评论难以追溯** | 评论与数据点关联不清 | 支持选择具体数据点评论 |
| **评论质量差** | 无意义评论干扰讨论 | 评分机制、差评折叠 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 自助分析 | 上游 | 评论附着的内容 |
| 消息中心 | 依赖 | 评论通知 |
| 内容审核 | 依赖 | 敏感信息过滤 |

### 2.3 FEATURE-DEX_KNOWLEDGE_BASE（知识沉淀）

#### 功能入口 & 作用

**入口位置**：数据分析 → 知识中心

**核心作用**：
- 优秀分析案例展示
- 分析模板共享
- 经验知识沉淀
- 知识搜索和发现

#### 核心逻辑

```
知识沉淀流程：

Step 1: 内容贡献
  - 将优秀分析收录为知识
  - 上传分析模板
  - 发布经验总结
  - 提交案例说明

Step 2: 知识审核
  - 管理员审核内容质量
  - 分类和标签标注
  - 质量评分

Step 3: 知识发布
  - 审核通过后发布
  - 通知关注用户
  - 更新知识索引

Step 4: 知识消费
  - 搜索和浏览知识
  - 收藏和订阅
  - 基于知识创建分析
  - 反馈和改进
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---:|
| `knowledge.approval_required` | 是否需要审批 | `true` |
| `knowledge.max_size` | 最大知识内容(MB) | `50` |
| `knowledge.retention_days` | 知识保留天数 | `forever` |
| `knowledge.rating_threshold` | 推荐阈值 | `4.0` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **知识质量参差** | 收录内容质量不一 | 建立审核机制、评分机制 |
| **知识难以发现** | 知识太多难以找到需要的 | 优化搜索、智能推荐 |
| **贡献激励不足** | 用户缺乏贡献动力 | 积分和排行榜、荣誉激励 |
| **知识陈旧** | 知识过时但未及时更新 | 定期审查、版本更新提醒 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 自助分析 | 上游 | 知识内容来源 |
| 搜索服务 | 依赖 | 知识检索 |
| 用户中心 | 依赖 | 用户激励信息 |

### 5.2 敏感信息处理规则

```
评论/分享内容审核规则：

1. 敏感词过滤

   - 禁止外链
   - 限制文件类型

3. 数据脱敏
   - 分享时自动脱敏

4. 内容举报
   - 用户可举报违规内容
   - 管理员审核处理
```

## 7. 问题库 & FAQ

### 7.1 常见问题排查

#### Q: 分享链接打不开怎么办？

**排查步骤**：
1. 确认链接是否过期
2. 检查访问权限是否正确
3. 确认链接是否被删除

**解决方案**：
- 联系分享人重新生成分享链接
- 申请访问权限
- 确认内容是否还存在

#### Q: 评论通知没有收到怎么办？

**排查步骤**：
1. 检查通知设置是否正确
2. 查看是否被拦截
3. 确认是否被@

**解决方案**：
- 检查消息接收设置
- 查看垃圾箱
- 手动刷新通知

#### Q: 知识收录被拒绝怎么办？

**排查步骤**：
1. 查看拒绝原因
2. 确认是否包含敏感信息
3. 改进内容质量

**解决方案**：
- 按照审核意见修改
- 移除敏感内容
- 提升内容质量

## 8. 方法论沉淀

### 8.1 协作设计原则

```
协作最佳实践：

1. 权限最小化
   - 默认只给必要权限
   - 渐进式授权

2. 透明可追溯
   - 所有操作有记录
   - 版本历史完整

3. 及时反馈
   - 操作结果及时通知
   - @提及确保触达

4. 知识沉淀
   - 优秀案例及时沉淀
   - 失败经验总结分享
```

### 8.2 知识运营策略

```
知识沉淀流程：

1. 征集阶段
   - 定期征集优秀案例
   - 激励机制促进贡献

2. 审核阶段
   - 专家评审内容质量
   - 分类和标签标注

3. 发布阶段
   - 精选推荐给用户
   - 通知关注用户

4. 迭代阶段
   - 用户反馈持续优化
   - 版本更新维护
```

### 8.3 工具流程

**协作工具链**：
```
分享 → 协作 → 沉淀 → 复用

工具：
- 分享：分享服务、权限服务
- 协作：评论服务、通知服务
- 沉淀：知识中心、审核服务
- 复用：搜索服务、模板库
```
