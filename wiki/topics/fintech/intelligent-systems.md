## 三、支付系统

### 架构分层

```
┌─────────────────────────────────────────┐
│  接入层 - API Gateway / 负载均衡         │
├─────────────────────────────────────────┤
│  业务层 - 支付核心 / 订单 / 账户 / 清结算 │
├─────────────────────────────────────────┤
│  数据层 - 交易记录 / 对账 / 风控 / 报表   │
├─────────────────────────────────────────┤
│  基础层 - 数据库 / 缓存 / 消息队列        │
└─────────────────────────────────────────┘
```

### 高可用指标

| 指标 | 目标 | 代表 |
|------|------|------|
| **SLA** | 99.99% | 支付宝 |
| **RTO** | <5分钟 | 行业标准 |
| **RPO** | <1分钟 | 行业标准 |
| **峰值** | 54.4万笔/秒 | 支付宝双十一 |

### 核心能力

| 能力 | 说明 |
|------|------|
| **聚合支付** | 微信/支付宝/银联统一接入 |
| **幂等控制** | 防止重复支付 |
| **分布式事务** | TCC/Saga模式 |
| **对账系统** | 日终对账，差错处理 |
