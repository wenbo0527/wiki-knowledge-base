## 1. 产品总览

### 1.1 一句话定位
**企业内部数字社区的统一门户入口，提供个性化工作台、便捷导航和内容聚合，提升员工协作效率和获取信息的速度。**

### 1.2 产品定位 & 核心价值

| 维度 | 描述 |
|:---|:---|
| **目标用户** | 全体员工、管理员、运营人员 |
| **核心痛点** | 信息分散在各个系统、缺少统一入口、找信息效率低 |
| **解决方案** | 统一门户 + 个性化工作台 + 智能推荐 + 便捷导航 |
| **核心价值** | 信息获取效率提升 50%，协作响应速度提升 30% |

## 2. 功能结构（按模块归档）

### 2.1 FEATURE-COM_PORTAL_CONFIG（门户配置管理）

#### 功能入口 & 作用

**入口位置**：系统管理 → 门户管理 → 页面配置

**核心作用**：
- 提供可视化的页面编辑能力，支持拖拽式布局
- 配置门户的主题风格、品牌元素
- 管理门户组件的显示、隐藏、排序
- 支持多语言、多端适配配置

#### 核心逻辑

```
门户配置流程：

Step 1: 选择配置对象
  - 全局门户（企业级）
  - 部门门户（部门级）
  - 个人门户（用户级）

Step 2: 布局配置
  - 选择布局模板（左右、上下、网格）
  - 拖拽调整区域大小
  - 设置响应式断点（PC/Pad/手机）

Step 3: 组件配置
  - 添加组件（从组件库选择）
  - 配置组件参数（数据源、显示数量）
  - 设置组件样式（宽度、高度、背景）
  - 调整组件位置和顺序

Step 4: 主题配置
  - 选择主题模板或自定义
  - 配置品牌色（主色、辅色）
  - 上传 Logo 和 Banner
  - 设置字体和图标风格

Step 5: 预览和发布
  - 多端预览效果
  - 保存草稿或直接发布
  - 设置生效时间（如节假日主题）
```

#### 关键配置项

| 配置项 | 说明 | 示例 |
|:---|:---|:---|
| `portal.layout.type` | 布局类型 | `GRID` |
| `portal.layout.columns` | 栅格列数 | `12` |
| `portal.theme.mode` | 主题模式 | `LIGHT` |
| `portal.theme.primary_color` | 主色调 | `#1890ff` |
| `portal.components` | 组件列表 | `[{type: todo, order: 1}, {type: notice, order: 2}]` |
| `portal.data_source` | 数据源配置 | `{todo: feishu_api, notice: internal_system}` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **组件依赖冲突** | 不同组件依赖同一数据源，配置复杂 | 建立组件隔离机制，独立配置数据源 |
| **多端显示不一致** | PC 配置在移动端显示异常 | 提供多端独立配置能力 |
| **主题切换影响可用性** | 主题变更导致页面错乱 | 建立主题版本管理，支持快速回滚 |
| **配置保存冲突** | 多人同时编辑配置 | 引入配置锁机制，记录操作日志 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 组件库 | 依赖 | 提供可配置的组件 |
| 主题引擎 | 依赖 | 主题切换和渲染 |
| 权限系统 | 调用 | 组件权限校验 |
| 内容管理系统 | 下游 | 门户内容数据来源 |

### 2.2 FEATURE-COM_PORTAL_AGGREGATION（门户内容聚合）

#### 功能入口 & 作用

**入口位置**：门户首页 / 个人工作台

**核心作用**：
- 聚合来自不同数据源的内容（通知、待办、新闻等）
- 智能排序和推荐，提升信息获取效率
- 支持用户个性化定制（收藏、置顶、隐藏）
- 提供统一的消息提醒和红点提示

#### 核心逻辑

```
内容聚合流程：

Step 1: 数据源接入
  - 注册数据源（飞书、企微、内部系统）
  - 配置数据接口（REST API、Webhook）
  - 设置数据更新策略（实时、定时、触发）

Step 2: 内容获取
  - 并行拉取多数据源内容
  - 数据格式统一转换
  - 权限过滤（只获取有权限的内容）
  - 去重和合并

Step 3: 内容排序
  - 时效性权重（新 > 旧）
  - 相关性权重（用户标签匹配）
  - 热度权重（阅读量、互动量）
  - 个性化权重（历史行为、收藏）

Step 4: 内容展示
  - 分页加载
  - 无限滚动
  - 按类型分组展示
  - 红点数字显示
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---:|
| `aggregation.datasource_timeout` | 数据源超时时间 | `3000ms` |
| `aggregation.max_items` | 最大展示数量 | `50` |
| `aggregation.refresh_interval` | 刷新间隔 | `5min` |
| `aggregation.sort_strategy` | 排序策略 | `TIME_DESC` |
| `aggregation.personalization_enabled` | 个性化开关 | `true` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **数据源超时阻塞** | 某个数据源响应慢导致整体超时 | 设置单独超时、使用异步加载、优雅降级 |
| **内容重复展示** | 多数据源内容重复 | 基于内容 ID 去重、智能合并 |
| **隐私信息泄露** | 用户看到不该看到的内容 | 权限前置过滤、数据脱敏 |
| **更新不及时** | 内容已更新但页面未刷新 | 建立推送机制（WebSocket/SSE） |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 数据源接入服务 | 依赖 | 各系统的数据拉取 |
| 排序服务 | 依赖 | 内容智能排序 |
| 权限服务 | 上游 | 获取用户权限信息 |
| 消息推送服务 | 下游 | 通知提醒 |

### 2.3 FEATURE-COM_PORTAL_ACCESS（门户访问管理）

#### 功能入口 & 作用

**入口位置**：系统管理 → 门户管理 → 访问控制

**核心作用**：
- 统一身份认证（支持多种登录方式）
- 访问权限控制（基于角色、部门、数据）
- 访问行为审计（记录访问日志）
- 安全防护（防爬虫、异常检测）

#### 核心逻辑

```
访问控制流程：

Step 1: 身份认证
  - 用户发起访问请求
  - SSO 单点登录验证
  - Token 生成和校验
  - Session 管理

Step 2: 权限校验
  - 获取用户角色和部门
  - 查询门户访问权限
  - 组件级权限校验
  - 数据级权限校验

Step 3: 访问控制
  - 有权限 → 正常访问
  - 无权限 → 友好提示 + 申请入口
  - 异常检测 → 安全拦截

Step 4: 审计记录
  - 记录访问时间、用户、设备
  - 记录访问的页面和组件
  - 记录异常访问行为
  - 定期生成审计报告
```

#### 关键配置项

| 配置项 | 说明 | 示例 |
|:---|:---|:---|
| `access.auth_type` | 认证方式 | `SSO` |
| `access.session_timeout` | 会话超时时间 | `30min` |
| `access.max_concurrent` | 最大并发会话数 | `5` |
| `access.risk_detection_enabled` | 风险检测开关 | `true` |
| `access.audit_retention_days` | 审计日志保留天数 | `180` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **SSO 登录失败** | SSO 服务异常导致无法登录 | 提供备用登录方式、降级为本地认证 |
| **权限配置遗漏** | 新增功能忘记配置权限 | 建立权限配置检查清单 |
| **审计日志量大** | 访问量大的情况下日志存储压力大 | 分级存储、热冷分离、定期归档 |
| **Token 泄露风险** | Token 被窃取导致冒充访问 | Token 加密传输、短期失效、异常检测 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| SSO 服务 | 依赖 | 单点登录 |
| 权限服务 | 依赖 | 权限校验 |
| 审计服务 | 依赖 | 日志记录 |
| 风控服务 | 依赖 | 异常检测 |

## 4. 数据相关

### 4.1 核心指标口径

| 指标 | 定义 | 计算逻辑 | 目标值 |
|:---|:---|:---|:---:|
| **DAU** | 日活跃用户数 | `count(DISTINCT user_id WHERE date = today)` | > 5000 |
| **日均 PV** | 页面日均浏览量 | `count(page_views WHERE date = today)` | > 20000 |
| **人均浏览深度** | 用户平均浏览的组件数 | `avg(components_viewed_per_user)` | > 8 |
| **首页加载时间** | 门户首页加载完成的平均时间 | `avg(page_load_time_ms)` | < 1000ms |
| **组件渲染成功率** | 组件成功渲染比例 | `success_render / total_render * 100%` | > 99.5% |
| **个性化覆盖率** | 开启个性化用户占比 | `personalized_users / total_users * 100%` | > 60% |

## 7. 问题库 & FAQ

### 7.1 常见问题排查

#### Q: 门户页面加载很慢怎么办？

**排查步骤**：
1. 检查网络状况（DNS、CDN）
2. 查看浏览器控制台性能面板
3. 检查组件加载时间
4. 查看后端接口响应时间

**解决方案**：
- 开启浏览器缓存
- 清理浏览器缓存后重试
- 联系管理员检查服务状态

#### Q: 某些组件显示空白怎么办？

**排查步骤**：
1. 检查组件配置是否正确
2. 检查数据源是否可用
3. 查看组件权限是否足够
4. 检查 Token 是否过期

**解决方案**：
- 刷新页面重试
- 重新登录
- 联系管理员检查组件配置

#### Q: 主题样式显示异常怎么办？

**排查步骤**：
1. 清除浏览器缓存
2. 检查是否有多个浏览器标签页
3. 确认主题配置是否正确

**解决方案**：
- Ctrl+Shift+R 强制刷新
- 换一个浏览器尝试
- 联系管理员检查主题配置

## 8. 方法论沉淀

### 8.1 门户设计方法论

**门户价值公式**：
```
门户价值 = 信息获取效率 × 内容质量 × 个性化程度 / 访问成本
```

**门户体验原则**：
1. **三秒原则**：用户 3 秒内找不到想要的内容就会离开
2. **一目了然**：首页信息密度要适度，关键信息一眼可见
3. **渐进披露**：复杂信息分层展示，避免信息过载
4. **操作闭环**：每个操作都要有明确的反馈和结果

### 8.2 组件设计最佳实践

**组件设计原则**：
1. **单一职责**：每个组件只做一件事
2. **配置驱动**：行为由配置决定，而非硬编码
3. **错误隔离**：组件错误不影响其他组件
4. **性能优先**：渲染性能要达标（首屏 < 100ms）

**组件分类体系**：
| 类别 | 说明 | 示例 |
|:---|:---|:---|
| **信息类** | 展示信息的组件 | 公告、新闻、日程 |
| **待办类** | 处理任务的组件 | 待办、审批、流程 |
| **快捷入口** | 快速跳转的组件 | 应用入口、收藏 |
| **数据类** | 展示数据的组件 | 统计、图表、排名 |
| **互动类** | 社交互动的组件 | 投票、评论、问答 |

### 8.3 行业对比

| 对比维度 | 传统门户 | 现代协作门户（飞书/钉钉）| 数字社区门户（我们）|
|:---|:---|:---|:---|
| **核心定位** | 信息展示 | 协作平台 | 智能化工作台 |
| **个性化** | 弱 | 中 | 强（AI推荐）|
| **内容聚合** | 静态 | 半动态 | 智能动态 |
| **用户体验** | 较差 | 良好 | 优秀 |
| **移动端** | 弱适配 | 良好 | 优秀 |

### 8.4 工具流程

**门户配置工具链**：
- **页面编辑**：拖拽编辑器（如 grapejs）
- **组件市场**：内部组件市场（私有化部署）
- **主题工具**：在线主题编辑器
- **性能监控**：前端性能监控（ARMS/秦皇）

**运营工具链**：

- **数据分析**：埋点 + ClickHouse + Grafana
