## 🎯 流程说明

Lint 是对 Wiki 进行健康检查的过程，发现问题并修复，确保 Wiki 的质量和可用性。

## ✅ 检查清单

### 1. 孤立页面检查

- [ ] 没有任何双向链接引用的页面
- [ ] 没有被任何 Topic 关联的 Insights
- [ ] 没有被任何 Insights 引用的 Entities

**处理方式**：
- 如果有价值但遗漏链接 → 添加链接
- 如果确实无价值 → 考虑归档或删除

### 2. 过时内容检查

- [ ] Insights 最后更新超过3个月
- [ ] Topics 最后更新超过6个月
- [ ] Entities 数据明显过时

**处理方式**：
- 标记为 `【待更新】`
- 评估是否值得更新
- 必要时发起 Ingest 流程

### 3. 矛盾信息检查

- [ ] 同一实体在不同页面的描述是否一致
- [ ] 相关联的信息是否有逻辑矛盾
- [ ] 数据是否与来源一致

**处理方式**：
- 发现矛盾 → 立即标记 `【矛盾】`
- 协调相关维护者解决
- 必要时回滚或覆盖

### 4. 链接完整性检查

- [ ] 双向链接 `[[xxx]]` 是否指向存在的页面
- [ ] 外部链接是否可访问
- [ ] 是否有死链或空链接

**处理方式**：
- 失效链接 → 修复或删除
- 空链接 → 补充内容或移除

| - | 无矛盾 | - |
