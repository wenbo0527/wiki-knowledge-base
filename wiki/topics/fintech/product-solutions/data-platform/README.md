## 产品背景

### 为什么需要数据中台

```
业务痛点：
├── 找不到数据（70%反馈）
│   └── "我知道数据库里有，但不知道在哪"
├── 不知道数据质量（50%反馈）
│   └── "这个数据准不准，能不能用于风控模型"
├── 不知道数据从哪来（30%反馈）
│   └── "为什么这个数和那个数对不上"
└── 数据资产难计量
    └── "我的数据值多少钱？能入表吗？"

数据团队困境：
├── 3人能做的事有限
├── 技术债务较多
└── 需要优先解决最痛的
```

### 消费金融的独特需求

| 需求 | 说明 |
|------|------|
| **数据资源登记** | 监管要求数据资产入表 |
| **数据确权** | 明确数据归属和责任 |
| **数据资产计量** | 财务报表要体现数据价值 |
| **合规可追溯** | 数据使用要可审计 |

## 核心功能

### 1. 数据目录（搜索 + 分类 + 标签）

**功能描述**：让业务方能快速找到需要的数据

**核心能力**：
- 全文搜索：按表名、字段名、描述搜索
- 分类导航：按业务域、数据类型搜索
- 标签体系：业务标签、技术标签、质量标签
- 预览功能：数据样本、变更历史

**用户路径**：
```
搜索"用户" → 看到相关表列表 → 查看表详情 → 申请数据权限 → 使用数据
```

### 2. 数据血缘（表级血缘）

**功能描述**：展示数据的上下游关系

**核心能力**：
- 表级血缘：表的输入输出关系
- 字段级血缘（按需）：关键字段的来源
- 可视化展示：树状图、流程图
- 影响分析：改一个表，影响哪些下游

**不做的原因**（资源约束）：
- 字段级血缘太重，维护成本高
- 技术债务多，短期内无法实现

### 3. 数据资产计量

**功能描述**：量化数据的业务价值和财务价值

**核心能力**：
- 数据资源登记：登记企业拥有的原始数据
- 资产等级评估：按完整性、准确性、时效性评分
- 入表支持：生成数据资产财务报表
- 使用监控：谁在用、用多少、效果如何

**三个概念**：
```
数据资源：企业拥有或控制的原始数据（量大、散乱）
数据资产：预期带来经济利益的数据（已治理、已确权）
数据要素：可交易流通的数据产品（合规、安全）
```

### 4. 数据质量评分

**功能描述**：评估数据的可用性

**核心能力**：
- 质量评分：完整性、准确性、一致性、时效性
- 问题追踪：质量问题记录和跟进
- 预警通知：质量问题及时通知

## 落地实践

### MVP版本（6个月）

```
Phase 1（0-2月）：
├── 数据目录MVP
│   ├── 搜索功能
│   ├── 基础分类
│   └── 表信息展示
└── 血缘基础版
    └── 表级血缘

Phase 2（3-4月）：
├── 数据目录增强
│   ├── 标签体系
│   └── 预览功能
├── 数据质量评分
└── 基础资产计量

Phase 3（5-6月）：
├── 资产计量完善
│   ├── 入表支持
│   └── 报表生成
└── 高级功能
    └── 影响分析（按需）
```
