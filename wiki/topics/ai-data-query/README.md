## 核心场景

**业务人员自助问数** - 不涉及多表，但需要准确，数据结果支持交互式分析

### 核心需求

| 需求 | 说明 | 优先级 |
|------|------|--------|
| 准确优先 | 生成的SQL必须100%准确 | 🔴 核心 |
| 单表场景 | 不涉及复杂多表关联 | ✅ 简化 |
| 交互分析 | 下钻/筛选/对比/导出 | 🔴 核心 |
| 业务友好 | 业务术语自动映射 | 🔴 核心 |
| 快速响应 | 秒级返回结果 | 🟠 重要 |

## 关键设计决策

| 决策点 | 推荐方案 | 理由 |
|--------|----------|------|
| **SQL生成方式** | 模板+参数填充 > 纯LLM生成 | 准确率优先，单表场景模板足够 |
| **多轮对话** | 简单上下文窗口 | 业务人员问数通常单轮 |
| **缓存策略** | 查询结果缓存30分钟 | 减少数据库压力 |
| **图表选择** | AI推荐+用户可调 | 智能默认值，用户有控制感 |
| **错误处理** | 友好的错误提示 | "数据不存在"比"SQL Error"更好 |
