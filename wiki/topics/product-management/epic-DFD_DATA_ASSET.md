# 1. 产品总览

## 1.1 一句话定位

**统一的数据资产元数据管理中心，实现数据资产的"可见、可管、可用、可运营"，让企业数据资产一目了然、触手可及。**

## 1.2 产品定位 & 核心价值

| 维度 | 描述 |
|:---|:---|
| **目标用户** | 数据管理员、数据分析师、开发人员、业务人员 |
| **核心痛点** | 数据资产分散、元信息缺失、血缘关系不清、资产利用率低 |
| **解决方案** | 统一的元数据管理、完善的资产分类体系、清晰的血缘关系追踪、便捷的资产发现机制 |
| **核心价值** | 让数据资产可见（发现）、可管（治理）、可用（消费）、可运营（增值） |

## 2.2 FEATURE-DFD_ASSET_METADATA_MGMT - 资产元数据管理

### 2.2.1 功能入口 & 作用

**功能入口**：
- 管理后台 → 元数据管理 → 资产元数据
- 资产详情页 → 元数据标签页
- 数据采集任务 → 元数据采集配置

**核心作用**：
- 统一采集和存储数据资产的全量元数据
- 建立技术、业务、管理三位一体的元数据体系
- 为资产目录、血缘追踪、质量评估提供数据基础

### 2.2.2 核心逻辑

**元数据分类体系**：

| 元数据类型 | 定义 | 包含内容 | 示例 |
|:---|:---|:---|:---|
| **技术元数据** | 描述数据的技术属性和存储结构 | 表结构、字段类型、存储位置、更新频率、数据格式 | 表名：user_order，字段：order_id (bigint)，存储：Hive |
| **业务元数据** | 描述数据的业务含义和使用场景 | 业务定义、业务规则、使用场景、关联业务、责任人 | 业务域：电商交易，业务含义：用户订单信息，owner：张三 |
| **管理元数据** | 描述数据的管理属性和生命周期 | 创建时间、更新时间、版本信息、安全等级、生命周期状态 | 创建时间：2023-01-01，版本：v2.1，安全等级：L3 |

**元数据采集流程**：

```
┌─────────────────────────────────────────────────────────────────┐

│  - 采集范围设定（库、表、字段级过滤）                              │
└─────────────────────────────────────────────────────────────────┘
                              │

│  2. 元数据采集任务                                               │

│  - 增量采集：定期的变更元数据捕获（基于时间戳/版本对比）             │

└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
```

### 2.2.3 关键配置项

| 配置项 | 配置值 | 说明 |
|:---|:---|:---|
| **数据源类型** | MySQL、PostgreSQL、Oracle、SQL Server、Hive、ClickHouse、MongoDB、Elasticsearch 等 | 支持的数据源类型 |
| **采集模式** | 全量采集、增量采集、实时监听 | 元数据采集的三种模式 |
| **采集频率** | 实时、每小时、每天、每周 | 增量采集的调度频率 |
| **存储引擎** | Neo4j 图数据库 | 元数据的存储引擎 |
| **缓存策略** | Redis 缓存热点元数据 | 提升查询性能 |
| **索引配置** | 资产名称、标签、分类、owner 等字段索引 | 加速检索 |

### 2.2.4 常见坑 & 注意点

| 坑点 | 影响 | 解决方案 | 预防措施 |
|:---|:---|:---|:---|
| **多数据源元数据冲突** | 同名表/字段被覆盖，信息丢失 | 数据源命名空间隔离、全局唯一ID生成 | 接入时强制指定数据源标识 |
| **大规模元数据采集性能差** | 首次全量采集耗时极长，系统卡顿 | 分库分表并行采集、流式写入、异步处理 | 合理设置采集批次大小，避免内存溢出 |
| **元数据与实际数据不一致** | 元数据过时，用户获取错误信息 | 增量采集+实时监听双保险、定时全量校验 | 设置数据变更触发器，确保元数据及时更新 |
| **元数据查询响应慢** |
