## 方法论

### Vibe Coding 流程

```
┌─────────────────────────────────────────────────────────────┐
│  Vibe Coding 完整流程                                       │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1️⃣ 需求描述                                               │
│     自然语言描述想要什么功能                                │
│                                                              │
│  2️⃣ AI生成 (V0/Cursor)                                   │
│     生成初始代码/组件                                       │
│                                                              │
│  3️⃣ 快速验证                                               │
│     立即运行看效果                                          │
│                                                              │
│  4️⃣ 迭代优化                                               │
│     "这里不对，改一下..."                                   │
│                                                              │
│  5️⃣ 代码整合                                               │
│     合并到主项目                                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### Prompt Engineering for Coding

| Prompt类型 | 模板 | 适用场景 |
|------------|------|----------|
| **生成型** | "你是XX语言专家，帮我写一个..." | 新功能开发 |
| **优化型** | "Review这段代码，找出性能问题" | 代码审查 |
| **解释型** | "解释这段代码的业务逻辑" | 代码理解 |
| **转换型** | "把Python转成TypeScript" | 代码迁移 |

### 实战技巧

1. **角色设定**: "你是资深Python工程师，熟悉FastAPI..."
2. **输出格式**: "用TypeScript，输出包含类型定义"
3. **约束条件**: "不依赖外部API，使用本地存储"
4. **示例驱动**: "参考这个模式，写一个类似的..."

## 工具链

### AI编程工具对比

| 工具 | 定位 | 优势 | 劣势 | 推荐场景 |
|------|------|------|------|----------|
| **Cursor** | AI IDE | 深度代码理解 | 收费 | 专业开发 |
| **V0 (Vercel)** | UI生成 | 快速原型 | 只限前端 | 前端组件 |
| **Copilot** | 代码补全 | 上下文感知 | 需要大量代码 | 常规开发 |
| **Trae** | AI IDE | 中文友好 | 生态较弱 | 快速上手 |
| **Windsurf** | AI IDE | 流程图生成 | 新工具 | 探索项目 |

### 工具选型建议

```
入门首选: Trae / Copilot (免费)
快速原型: V0 / Cursor
专业开发: Cursor (付费)
复杂项目: Cursor + Copilot 组合
```

## 实战案例

### 案例1: Instreet RSS爬虫

| 维度 | 内容 |
|------|------|
| **工具** | Cursor + Claude |
| **需求** | 50个RSS源每日抓取 |
| **耗时** | 2小时 (vs 传统1-2天) |
| **方法** | Vibe Coding: 描述需求 → AI生成 → 迭代调试 |

### 案例2: Superpowers工作流 (143k star)

| 维度 | 内容 |
|------|------|
| **工具** | Claude Code/Cursor/Codex |
| **核心理念** | 给AI编程代理装上「方法论」 |
| **特点** | 强制工作流，不是建议 |
| **方法** | brainstorming → writing-plans → 子代理驱动 → TDD → review |

**核心洞察**: AI只有「能力」没有「方法论」——Superpowers解决的是让AI会规划、会停下来思考、会在bug没修好时拒绝继续。

### 案例3: Spec Coding实战 (2754次调用)

| 维度 | 内容 |
|------|------|
| **工具** | Claude Code + openspec |
| **数据** | 10天、2.5万行代码、提效36% |
| **核心方法** | Proposal→Design→Specs→Tasks 规格驱动 |
| **三层规范** | CLAUDE.md + Rules + Specs |
| **MCP应用** | 接口文档直连、飞书云文档直读 |

**核心洞察**: AI失效的三种模式——本地无法复现、多层根因掩盖、隐性依赖无文档。

## 适用场景评估

### 适合用Vibe Coding的场景 ✅

- 快速原型验证
- 单个小工具/脚本
- 代码片段生成
- 学习新技术练手
- 简单CRUD页面

### 不适合的场景 ⚠️

- 复杂业务逻辑系统
- 高并发/高性能要求
- 安全敏感代码（支付、认证）
- 需要深度架构设计的系统
- 团队协作的大型项目

## 最佳实践

### 1. Prompt模板库

```markdown
# 功能开发模板
你是[语言]专家，熟悉[框架]。
帮我写一个[功能描述]，
要求：
1. [约束1]
2. [约束2]
3. 包含单元测试

# 代码审查模板
Review这段代码：
[代码]
重点检查：
- 性能问题
- 安全漏洞
- 可维护性
```

### 2. 迭代技巧

| 技巧 | 说明 |
|------|------|
| **小步快跑** | 每次让AI生成10-20行，而非整个模块 |
| **即时验证** | 生成后立即运行，不要等到最后 |
| **明确错误** | 把报错信息直接发给AI |
| **上下文补充** | 提供相关代码文件让AI理解 |

### 3. 代码质量控制

1. **AI生成后必须Review**: 不要直接用
2. **关键逻辑人工写**: 核心算法不要全交给AI
3. **测试必须跑**: AI生成的代码要跑通测试
4. **安全检查**: 防止AI生成有安全漏洞的代码
