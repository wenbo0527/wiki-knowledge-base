## 核心原则

### 1. 简单优先

| 原则 | 说明 | 反例 |
|------|------|------|
| **少层级** | 最多3层目录深度 | 5层以上的嵌套目录 |
| **宽分类** | 横向扩展多于纵向 | 小类目、大纵深 |
| **统一命名** | 命名规范一致 | 中英混杂、无规律 |

### 2. 用户导向

| 原则 | 说明 |
|------|------|
| **按人组织** | 用户视角的导航 |
| **按场景组织** | 任务导向的页面 |
| **搜索优先** | 搜索比导航更快 |

### 3. 可维护性

| 原则 | 说明 |
|------|------|
| **模板化** | 减少重复、统一规范 |
| **自动化** | 减少手动维护 |
| **轻量化** | 避免过度工程化 |

## 空间划分模式

### 模式对比

| 模式 | 说明 | 适用场景 | 代表产品 |
|------|------|----------|----------|
| **按团队** | 每个团队一个空间 | 团队自治型组织 | Confluence |
| **按项目** | 每个项目一个空间 | 项目驱动型组织 | Notion |
| **按主题** | 按知识主题分类 | 内容驱动型组织 | 飞书Wiki |
| **混合** | 团队+主题双维度 | 大型复杂组织 | - |

### 推荐：混合模式

```
企业Wiki
├── 公共空间
│     ├── 规章制度
│     ├── 公司介绍
│     └── 公共知识库
│
├── 团队空间
│     ├── 产品团队
│     ├── 技术团队
│     └── 市场团队
│
└── 项目空间
      ├── 项目A
      ├── 项目B
      └── 项目C
```

## 搜索优化

### 结构化搜索

| 技术 | 说明 |
|------|------|
| **元数据** | 添加tags、日期、作者 |
| **双向链接** | 建立页面关联 |
| **别名** | 支持同义词搜索 |
| **热门内容** | 高频内容优先 |

| 要素 | 说明 |
|------|------|
| **即时搜索** | 输入即显示结果 |

## 最佳实践

### DO

1. **从用户场景出发** — 考虑谁会来搜索什么
2. **保持简单** — 避免过度分类
3. **定期清理** — 删除/归档过时内容
4. **内容为王** — 先有内容，再谈架构
5. **持续迭代** — 架构不是一次成型的

### DON'T

1. **不要追求完美架构** — 先用起来
2. **不要过度分类** — 找起来比看起来重要
3. **不要忽视搜索** — 导航是辅助，搜索是主力
4. **不要孤立内容** — 链接比堆砌好
5. **不要忽视移动端** — 移动访问越来越重要
