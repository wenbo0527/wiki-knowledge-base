### 项目结构

```

clean_business_data.csv        # 清洗后数据
data_preparation.ipynb         # Python清洗notebook
advanced_queries.sql           # 高级SQL（CTE+归因）
marketing_performance_dashboard.twbx  # Tableau看板
stakeholders_presentation.pdf  # 商业呈现PPT
```

### KPI矩阵

| 指标 | 定义 | 计算 |
|------|------|------|
| Conversions | 首单数 | COUNT(DISTINCT user_id) |
| CAC | 获客成本 | Spend / New Customers |
| LTV | 生命周期价值 | Avg Revenue × Lifespan |
| M+1 Retention | 月留存 | Day30复购率 |

### 归因模型实现

```sql
-- First-order, month-strict attribution
-- 用户首单归因到当月的首次触达渠道
WITH first_touch AS (
  SELECT 
    user_id,
    MIN(channel) AS first_channel,
    MIN(event_date) AS first_date
  FROM marketing_events
  WHERE event_type = 'first_touch'
  GROUP BY user_id
)
SELECT 
  first_channel,
  COUNT(DISTINCT user_id) AS conversions,
  SUM(spend) AS total_spend,
  SUM(spend) / COUNT(DISTINCT user_id) AS cac
FROM first_touch
GROUP BY first_channel;
```

## 对文博GitHub改造的启示

### 当前状态 vs 目标

| 维度 | 当前 | 目标 |
|------|------|------|
| **Profile** | 空白 | Name + Bio + 定位 |
| **仓库** | 零散（6个） | 聚焦（3个核心） |
| **内容** | 代码为主 | 问题导向 |
| **量化** | 无 | 有数字支撑 |

### 建议的仓库结构

```
wenbo0527/
├── fintech-cdp-analytics/         # CDP/分群/画像
├── fintech-attribution-dashboard/ # 归因分析
├── fintech-data-platform/          # 找数看数用数
└── README.md                      # Profile入口
```
