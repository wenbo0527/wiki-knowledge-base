# 软件开发项目日志模板

> 用于记录每日项目进展、问题追踪、风险管理的标准化模板。
> 
> *模板设计：尼克·弗瑞 | 基于行业最佳实践*

---

## 一、模板说明

### 使用场景

| 场景 | 说明 | 适用人群 |
|------|------|----------|
| **每日站会准备** | 记录昨日进展、今日计划、阻碍 | 敏捷团队 |
| **周报素材** | 汇总一周工作、问题、风险 | 项目经理 |
| **知识传承** | 记录上下文，便于团队交接 | 离职交接、新人入职 |
| **进度追踪** | 可视化项目真实进度 | 项目负责人 |
| **问题复盘** | 记录问题根因、解决方案 | 故障复盘 |

### 模板特点

| 特点 | 说明 |
|------|------|
| **结构化** | 固定字段，便于检索和分析 |
| **可量化** | 量化的进展描述，拒绝"做了很多" |
| **问题导向** | 明确识别阻碍和风险 |
| **可追溯** | 链接具体任务、需求、代码 |

## 五、最佳实践指南

### 记录时机

| 时机 | 建议 |
|------|------|
| **每日结束** | 花5-10分钟填写，保持最新 |
| **遇到问题** | 立即记录，避免遗忘 |
| **任务完成** | 及时更新状态和产出 |
| **会议之后** | 记录决策和待办 |

### 写作原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **具体** | 描述具体动作和结果 | "修复了登录bug" ✅ |
| **可量化** | 包含数字和百分比 | "完成80%的API开发" ✅ |
| **可追溯** | 链接到具体任务 | "关联TASK-123" ✅ |
| **问题导向** | 明确识别阻塞 | "缺少API文档" ✅ |
| **简洁** | 避免冗余 | 删掉废话 ✅ |

### 常见问题

| 问题 | 改进 |
|------|------|
| 写成流水账 | 聚焦关键产出和决策 |
| 缺少问题 | 主动识别阻塞和风险 |
| 无法量化 | 添加进度百分比、工时估算 |
| 没有关联 | 链接具体工单、需求、PR |
| 过于简略 | 补充上下文，便于他人理解 |
