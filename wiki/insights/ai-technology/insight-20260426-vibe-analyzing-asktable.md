## 🔄 范式转变：从被动呈现到主动共创

### 传统BI的瓶颈

| 问题 | 说明 |
|------|------|
| 固定看板 | 预建模、固定看板，流程长 |
| 响应迟滞 | 需要IT排期，灵活性和响应速度差 |
| 被动呈现 | 数据消费方式僵化 |

### 新范式：Vibe Analyzing

> **核心目标**：只要有表，用户就能安全、即时地查到所需数据

## 🔍 问数引擎：让业务人员秒级取数

### 核心能力

| 模块 | 说明 |
|------|------|
| **输入端** | MySQL、Doris、数据仓库、Excel、API |
| **处理层** | Table Agent Builder，自然语言→SQL |
| **输出端** | 网页、App、企业微信、飞书、钉钉 |

### 核心洞察

> **"原始表→即时查询"：让业务人员用自然语言秒级取数，无需依赖IT排期**

### 权限挑战

动态生成的结果无法沿用传统报表级权限控制。**权限必须前移至数据源层**，基于原始表进行行列级管控。

## 🎨 分析画布：让分析师沉浸式探索

### NoChatBI理念

> **"对话式交互本质是沟通方式，无法承载人类对数据的非线性思考。"**

### 传统ChatBI局限

- 仅能处理简单查询
- 难以支撑分析师在探索中不断迭代、关联验证的思维链条

### 分析画布核心设计

| 角色 | 说明 |
|------|------|
| **Builder** | 自然语言→SQL→Python代码→可视化渲染 |
| **Viewer** | 查看AI生成的思维链、总结与洞察 |

### 三大核心功能

1. **自然语言查数**：输入一句话自动生成SQL，结果以可固化节点呈现
2. **智能生图可视化**：AI自动推荐图表，支持交互式调整
3. **跨源数据处理**：Python代码实现字段计算、清洗及多源融合

### 能力矩阵

| 功能 | 说明 |
|------|------|
| AI报告 | 自动生成分析报告 |
| AI看板 | 交互式数据看板 |
| AI解读 | 数据解读与洞察 |
| AI预测 | 预测性分析 |
| AI异常洞察 | 异常检测与预警 |

## 💡 关键洞察

### 对话式交互的边界

| 适用 | 不适用 |
|------|--------|
| 简单问答 | 深度探索 |
| 线性任务 | 非线性思考 |
| 单轮查询 | 多轮迭代 |

### 分析师价值曲线

1. **共识效率** - 主数据管理、指标治理、统一口径
2. **认知增量** - 主动识别异常、揭示趋势
3. **决策支撑** - 从运营到战略的高质量决策
