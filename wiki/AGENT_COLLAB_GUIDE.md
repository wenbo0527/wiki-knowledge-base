## 📝 命名规范

### 文件命名

| 类型 | 格式 | 示例 |
|------|------|------|
| 洞察页 | `insight-YYYYMMDD-{简短标题}.md` | `insight-20260413-ai-agent.md` |
| 主题页 | `kebab-case.md` | `llm-agent.md`、`data-platform.md` |
| 实体页 | `{type}/{kebab-case}.md` | `companies/openai.md`、`people/karpathy.md` |
| 概念页 | `kebab-case.md` | `rag.md`、`ai-llm.md` |
| 项目页 | `{project-name}/README.md` | `smart-credit/README.md` |
| 洞察页内子目录 | `_insights/`（可选） | `fintech/_insights/` |

### 目录命名

- 使用 **kebab-case**：`fintech`、`ai-native`、`analysis-frameworks`
- 专题内子目录：使用 `_` 前缀表示可选或临时（如 `_insights/`、`_drafts/`）

## 🚨 升级机制

### 派蒙职责升级

当遇到以下情况，派蒙必须立即汇报文博：

| 情况 | 描述 |
|------|------|
| 方向分歧 | 尼克和钟离对方案有重大分歧 |
| 范围蔓延 | 需求明显超出原定计划 |
| 里程碑延误 | 预计无法按时完成关键节点 |
| 资源不足 | 时间/人力/预算不够 |
| 技术风险 | 关键技术方案不可行 |

### 尼克/钟离职责升级

当遇到以下情况，应通知派蒙协调：

| 情况 | 处理 |
|------|------|
| 需要跨角色协作 | 发消息给派蒙 |
| 缺少关键信息 | 发消息给派蒙，请求支援 |
| 发现新风险 | 通知派蒙，记录到项目 |
