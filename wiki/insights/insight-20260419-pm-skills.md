### 3.2 AI时代产品经理的新技能

**新增核心技能**：

1. **AI工具链选型与集成**
   - 能力: 评估、选择、整合多种AI工具
   - 工具: Claude, GPT-4, Midjourney, Runway, etc.
   - 产出: 高效的AI工具工作流

2. **Prompt Engineering**
   - 能力: 设计高效的AI提示词
   - 技术: Chain-of-thought, Few-shot, Role-playing
   - 产出: 可复用的Prompt模板库

3. **AI辅助决策**
   - 能力: 利用AI进行数据分析和情景模拟
   - 方法: 假设检验、风险评估、预测模型
   - 产出: 数据驱动的决策建议

4. **人机协作设计**
   - 能力: 设计高效的人机协作流程
   - 原则: Human-in-the-loop, Human-on-the-loop
   - 产出: 人机协作的最佳实践

5. **AI伦理与合规**
   - 能力: 识别和规避AI使用的伦理风险
   - 关注: 隐私、偏见、安全、可解释性
   - 产出: AI使用的合规框架

## 4. 实践应用：如何将PM Skills应用于AI产品开发

### 4.1 AI产品开发的特殊性

AI产品开发与传统软件开发的主要区别：

| 维度 | 传统软件 | AI产品 |
|:---|:---|:---|
| **需求确定性** | 高，功能明确 | 低，需要探索边界 |
| **开发周期** | 明确，可预测 | 不确定，需要迭代 |
| **成功率** | 较高 | 较低，需要快速试错 |
| **评估标准** | 功能完整性 | 效果指标（准确率等） |
| **团队构成** | 工程为主 | 工程+数据+算法 |
| **技术债务** | 代码债务 | 模型债务+数据债务 |

### 4.2 针对AI产品的PM Skills应用

**模块1: 发现与用户研究 → 问题-解决方案匹配**

```
传统做法:
用户访谈 → 需求提取 → 功能设计 → 开发

AI产品做法:
问题识别 → 数据可行性评估 → AI解决方案探索 → POC验证 → 规模化

关键差异:
- 不是用户说要什么就做什么，而是探索AI能解决什么
- 需要评估数据可用性和AI技术成熟度
- 快速POC验证比详细需求文档更重要
```

**关键Skills应用**:
- `identify-assumptions-new`: 识别AI解决方案的核心假设
- `brainstorm-experiments-new`: 设计AI POC实验
- `user-personas`: 定义AI产品的目标用户群体

**模块2: 假设与优先级 → 不确定性管理**

```
传统做法:
功能优先级 = 价值 × 可行性

AI产品做法:
