## 1. 产品总览

### 1.1 一句话定位
**企业数字社区的统一权限管控中心，实现细粒度的角色、权限配置和全链路的访问审计，确保"正确的用户在正确的场景下访问正确的数据"。**

### 1.2 产品定位 & 核心价值

| 维度 | 描述 |
|:---|:---|
| **目标用户** | 系统管理员、安全管理员、审计人员、业务负责人 |
| **核心痛点** | 权限管理分散、权限变更繁琐、权限审计困难、安全风险高 |
| **解决方案** | 统一权限中心 + 角色化管理 + 审批流程 + 全链路审计 |
| **核心价值** | 权限管理效率提升 70%，安全审计覆盖率 100%，合规风险降低 90% |

### 1.5 关键约束

| 约束类型 | 具体约束 | 应对策略 |
|:---|:---|:---|
| **最小权限原则** | 用户只能拥有完成工作所需的最小权限 | 默认无权限，需申请授权 |
| **权限制约** | 敏感权限需二次确认或多人审批 | 敏感权限标记、强制审批流程 |
| **定期复审** | 高权限用户需定期复审权限合理性 | 建立权限复审机制，定时提醒 |
| **及时回收** | 员工离职/调岗时权限需及时清理 | 与 HR 系统同步，离职自动回收 |
| **审计追溯** | 所有权限变更必须可追溯 | 完整日志记录，防篡改 |
| **性能要求** | 权限校验接口响应 < 50ms | 权限数据本地缓存、本地校验 |

| 术语 | 英文 | 定义 | 示例 |

| **访问审计** | Access Audit | 对用户访问行为进行记录和分析 | 访问日志、异常检测 |

## 2. 功能结构（按模块归档）

### 2.1 FEATURE-COM_ROLE_MANAGE（角色管理）

#### 功能入口 & 作用

**入口位置**：系统管理 → 权限管理 → 角色管理

**核心作用**：
- 维护系统角色体系，支持角色的增删改查
- 配置角色的权限集合
- 管理角色与用户/部门的关联关系
- 支持角色继承和互斥配置

#### 核心逻辑

```
角色管理流程：

Step 1: 角色规划
  - 分析业务需求，确定角色体系
  - 梳理角色层级关系（继承结构）
  - 定义角色互斥关系
  - 编写角色说明书

Step 2: 角色创建
  - 填写角色基本信息（名称、描述、状态）
  - 选择角色类型（系统角色/业务角色）
  - 配置权限集合（从权限列表选择）
  - 设置角色继承关系（可选）
  - 配置权限互斥（可选）

Step 3: 角色分配
  - 选择分配对象（用户/部门）
  - 设置生效时间（立即/定时）
  - 填写分配原因
  - 审批（如需）

Step 4: 角色变更
  - 调整角色权限
  - 变更分配关系
  - 角色禁用/启用
  - 角色删除（仅允许未分配的角色）
```

#### 关键配置项

| 配置项 | 说明 | 示例 |
|:---|:---|:---|
| `role.name` | 角色名称 | `CONTENT_ADMIN` |
| `role.type` | 角色类型 | `SYSTEM` / `BUSINESS` |
| `role.parent` | 父角色 | `CONTENT_VIEWER` |
| `role.exclusions` | 互斥角色列表 | `["ROLE_A", "ROLE_B"]` |
| `role.permissions` | 权限列表 | `["news:view", "news:edit"]` |
| `role.status` | 角色状态 | `ACTIVE` / `INACTIVE` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **角色过于细分** | 角色数量过多导致管理复杂 | 按层级组织角色，减少平行角色数量 |
| **权限继承混乱** | 多层继承导致权限难以追踪 | 限制继承层级（最多 3 层），可视化权限链路 |
| **角色分配冲突** | 用户被分配互斥角色 | 系统自动校验，禁止分配 |
| **删除已分配角色** | 误删正在使用的角色 | 禁止删除已分配角色，强制先转移 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 权限配置模块 | 下游 | 角色拥有的权限来源 |
| 用户管理模块 | 下游 | 角色分配的对象 |
| 组织架构 | 依赖 | 获取部门信息 |
| 审批流程 | 依赖 | 角色变更的审批 |

### 2.2 FEATURE-COM_PERM_CONFIG（权限配置）

#### 功能入口 & 作用

**入口位置**：系统管理 → 权限管理 → 权限配置

**核心作用**：
- 定义系统权限清单（菜单、按钮、数据、字段）
- 配置权限与角色的对应关系
- 管理权限的层级结构
- 支持权限的增删改查和版本管理

#### 核心逻辑

```
权限配置流程：

Step 1: 权限规划
  - 梳理系统功能清单
  - 按模块分类权限
  - 定义权限编码规范
  - 编写权限说明文档

Step 2: 权限定义
  - 创建权限节点
  - 选择权限类型（MENU/BUTTON/DATA/FIELD）
  - 配置权限属性（编码、名称、描述、所属模块）
  - 设置上级权限（构建权限树）

Step 3: 权限分配
  - 选择要分配的角色
  - 勾选要分配的权限
  - 批量分配（支持导入导出）
  - 验证权限分配结果

Step 4: 权限变更
  - 新增功能时补充权限
  - 废弃功能时禁用权限
  - 历史权限版本保留
```

#### 关键配置项

| 配置项 | 说明 | 示例 |
|:---|:---|:---|
| `permission.code` | 权限编码 | `news:view` |
| `permission.type` | 权限类型 | `MENU` / `BUTTON` / `DATA` / `FIELD` |
| `permission.module` | 所属模块 | `news` |
| `permission.parent` | 上级权限 | `content:manage` |
| `permission.sensitive` | 是否敏感 | `true` |
| `permission.approval_required` | 是否需要审批 | `true` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **权限编码不统一** | 不同模块权限编码风格不一致 | 制定权限编码规范（模块:操作） |
| **权限遗漏** | 新增功能忘记配置权限 | 建立功能上线权限检查流程 |
| **权限过于粗放** | 一个权限包含太多功能 | 细分权限粒度，按钮级控制 |
| **权限版本混乱** | 权限变更后无法回溯 | 权限版本化管理，支持历史记录查看 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 角色管理模块 | 上游 | 权限分配的对象 |
| 菜单管理 | 依赖 | 菜单权限的数据来源 |
| 数据权限 | 下游 | 数据权限的实际执行 |
| 审计服务 | 下游 | 权限变更日志记录 |

### 2.3 FEATURE-COM_PERM_AUDIT（权限审计）

#### 功能入口 & 作用

**入口位置**：系统管理 → 权限管理 → 权限审计

**核心作用**：
- 记录和分析用户权限变更历史
- 监控异常访问行为和安全风险
- 生成合规审计报告
- 支持权限变更的追溯和回滚

#### 核心逻辑

```
权限审计流程：

Step 1: 日志采集
  - 权限变更日志（授权、回收、调整）
  - 访问日志（登录、登出、功能访问）
  - 操作日志（关键业务操作）
  - 异常日志（失败尝试、越权访问）

Step 2: 日志分析
  - 变更趋势分析
  - 异常模式识别（暴力破解、越权访问）
  - 权限覆盖分析（是否超出职责范围）
  - 离职风险分析（权限未及时清理）

Step 3: 风险预警
  - 高危权限使用告警
  - 异常访问行为告警
  - 权限过期未续期告警
  - 权限与岗位不匹配告警

Step 4: 报告生成
  - 合规报告（满足监管要求）
  - 权限分析报告（权限使用情况）
  - 安全报告（风险点和处置建议）
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---:|
| `audit.retention_days` | 日志保留天数 | `180` |
| `audit.risk_threshold` | 风险告警阈值 | `10` |
| `audit.report_schedule` | 报告生成周期 | `weekly` |
| `audit.anomaly_detection_enabled` | 异常检测开关 | `true` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **日志丢失** | 服务重启或故障导致日志缺失 | 日志双写、异步队列、持久化存储 |
| **性能影响** | 大量审计日志影响系统性能 | 异步写入、批量处理、索引优化 |
| **日志被篡改** | 恶意删除或修改日志 | 日志签名、定期备份、写入后不可更改 |
| **分析结果滞后** | 风险已经发生才知道 | 实时流处理，秒级告警 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 权限变更服务 | 上游 | 权限变更事件来源 |
| 访问日志服务 | 上游 | 访问日志数据来源 |
| 风控服务 | 依赖 | 异常行为识别 |

| 规则类型 | 规则内容 |
|:---|:---|
| **角色互斥** | 互斥角色不能同时赋予同一用户 |
| **角色继承** | 子角色自动继承父角色的所有权限 |
| **权限叠加** | 用户可以拥有多个角色，权限取并集 |

    │
    ▼ (敏感权限)
┌─────────────────┐

    │

    ▼

## 7. 问题库 & FAQ

### 7.1 常见问题排查

#### Q: 用户无法访问某个功能怎么办？

**排查步骤**：
1. 确认用户角色是否正确分配
2. 检查角色是否包含所需权限
3. 确认权限是否已生效（有时效配置）
4. 检查是否有数据权限限制

**解决方案**：
- 联系管理员检查角色分配
- 提交权限申请
- 等待权限生效（如有时效）

#### Q: 权限校验很慢怎么办？

**排查步骤**：
1. 检查网络延迟
2. 查看权限缓存命中率
3. 检查是否有大量权限校验请求

**解决方案**：
- 清除本地缓存重试
- 联系管理员检查服务状态
- 扩容权限校验服务

#### Q: 权限申请被拒绝了怎么办？

**排查步骤**：
1. 查看拒绝原因
2. 确认申请理由是否充分
3. 联系审批人沟通

**解决方案**：
- 补充更详细的申请理由
- 联系业务负责人说明情况
- 申请更合适的替代权限

## 8. 方法论沉淀

### 8.1 权限管理方法论

**RBAC 模型设计原则**：
1. **角色独立性**：角色之间相互独立，避免职责重叠
2. **权限最小化**：只授予完成工作所需的最小权限
3. **层级清晰**：角色层级不宜过深，建议最多 3 层
4. **职责分离**：互斥权限不能赋予同一角色

**权限变更管理原则**：
```
变更申请 → 变更审批 → 变更执行 → 变更验证 → 变更记录
```

### 8.2 权限设计最佳实践

**权限编码规范**：
```
格式：{模块}:{操作}

示例：
- news:view      # 查看新闻
- news:create     # 创建新闻
- news:edit      # 编辑新闻
- news:delete    # 删除新闻
- news:publish   # 发布新闻
```

**角色设计模式**：
| 模式 | 说明 | 适用场景 |
|:---|:---|:---|
| **岗位角色** | 按岗位命名角色 | 通用场景 |
| **职能角色** | 按职能命名角色 | 专业场景 |
| **项目角色** | 按项目命名角色 | 临时权限 |

### 8.3 行业对比

| 对比维度 | 传统 RBAC | 现代化权限（CASBIN）| 我们的方案 |
|:---|:---|:---|:---|
| **模型复杂度** | 简单 | 中等 | 中等（RBAC+ABAC）|
| **细粒度控制** | 粗粒度 | 细粒度 | 细粒度（行+列级）|
| **性能** | 快 | 中等 | 快（本地缓存）|
| **适用场景** | 简单权限 | 复杂权限 | 企业级复杂权限 |

### 8.4 工具流程

**权限管理工具链**：
- **权限模型设计**：UML 建模工具
- **权限配置管理**：内部权限管理平台
- **权限变更审批**：OA 系统集成
- **审计分析**：ELK + 自定义报表

**自动化工具**：
- **权限同步**：定时从 HR 系统同步组织和人员
- **权限清理**：离职自动回收权限
- **权限复审**：定期复审高权限用户
- **异常检测**：ML 模型识别异常行为
