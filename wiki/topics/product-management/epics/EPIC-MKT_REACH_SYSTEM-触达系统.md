## 1. 产品总览

### 1.1 一句话定位
**企业级消息触达中台，统一管理 App 推送、短信、公众号、邮件、外呼等多渠道消息的发送、调度和追踪。**

### 1.2 产品定位 & 核心价值

| 维度 | 描述 |
|:---|:---|
| **目标用户** | 营销运营人员、CRM 管理员、客服人员、产品经理 |
| **核心痛点** | 多渠道分散管理、发送策略不统一、触达效果难追踪、用户骚扰投诉多 |
| **解决方案** | 统一触达中台 + 智能调度 + 多渠道合一 + 全链路追踪 |
| **核心价值** | 触达效率提升 60%，触达成功率提升至 98%，用户投诉率降低 40% |

### 1.6 术语词典（本体论级别）

| 术语 | 英文 | 定义 | 示例 |
|:---|:---|:---|:---|
| **触达** | Reach / Push Notification | 向用户发送消息的行为 | 发送一条 App 推送 |
| **消息模板** | Message Template | 预定义的消息格式 | "尊敬的用户，您的订单已发货" |
| **触达渠道** | Reach Channel | 消息发送的终端通道 | App 推送、短信、公众号、邮件 |
| **送达率** | Delivery Rate | 实际送达 / 发送量的比例 | 发送 1000 条，到达 980 条，送达率 98% |
| **点击率** | Click-Through Rate (CTR) | 点击 / 送达量的比例 | 送达 980 条，点击 50 条，CTR 5.1% |
| **转化率** | Conversion Rate | 转化 / 送达量的比例 | 送达 980 条，转化 20 条，转化率 2% |
| **退订率** | Unsubscribe Rate | 退订 / 送达量的比例 | 送达 980 条，退订 5 条，退订率 0.5% |
| **消息队列** | Message Queue | 异步发送的消息缓冲队列 | Kafka 消息队列 |
| **渠道适配器** | Channel Adapter | 不同渠道 API 的统一封装 | 极光/JPush、阿里云短信、腾讯云短信 |
| **频率控制** | Frequency Capping | 控制用户被触达的频率上限 | 同一用户每天最多触达 5 次 |

## 2. 功能结构（按模块归档）

### 2.1 FEATURE-MKT_REACH_MESSAGE_MGMT（消息管理）

#### 功能入口 & 作用

**入口位置**：
- 触达系统 → 消息管理 → 新建消息
- 触达系统 → 模板管理

**核心作用**：
- 创建和编辑消息内容
- 管理消息模板库
- 配置消息变量和个性化内容
- 预览消息在不同终端的展示效果

#### 核心逻辑

```
消息管理流程：

Step 1: 消息创建
  - 选择消息类型（通知/营销/系统/验证码）
  - 选择发送渠道（可多选）
  - 填写消息内容（标题、正文、媒体）
  - 配置跳转链接或 Deep Link

Step 2: 消息模板化
  - 将消息内容抽象为模板
  - 定义变量占位符 {{name}}、{{order_no}}
  - 设置模板分类和标签
  - 提交审核（如需要）

Step 3: 变量配置
  - 配置用户属性变量
  - 配置业务数据变量（订单号、时间等）
  - 设置变量默认值
  - 预览变量渲染结果

Step 4: 消息审核
  - 内容安全审核（敏感词过滤）
  - 链接安全检测
  - 敏感信息脱敏
  - 审核记录存档
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---:|
| `message.max_length` | 消息内容最大长度 | 短信 500 字，推送 500 字 |
| `message.retry_times` | 发送失败重试次数 | `3` |
| `message.retry_interval` | 重试间隔（秒）| `60` |
| `message.preview_enabled` | 发送前预览 | `true` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **消息内容被截断** | 某些渠道有长度限制 | 自动截断或拆分成多条 |
| **变量渲染失败** | 用户数据缺失导致变量无法渲染 | 设置默认值、过滤缺失数据用户 |
| **链接打不开** | 落地页失效或 App 未安装 | 配置默认落地页、App 下载引导 |
| **敏感信息泄露** | 订单金额等敏感信息暴露 | 脱敏规则、权限控制 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 用户中心 | 上游 | 获取用户信息和联系方式 |
| 客群中心 | 上游 | 人群包数据 |
| 订单系统 | 上游 | 订单相关的变量数据 |
| 内容安全审核 | 依赖 | 消息内容安全检测 |

### 2.2 FEATURE-MKT_REACH_CHANNEL_MGMT（渠道管理）

#### 功能入口 & 作用

**入口位置**：触达系统 → 渠道管理

**核心作用**：
- 统一管理多渠道配置
- 监控渠道发送状态和成本
- 配置渠道优先级和降级策略
- 渠道账号和配额管理

#### 核心逻辑

```
渠道管理流程：

Step 1: 渠道接入
  - 添加渠道账号（极光、阿里云、腾讯云等）
  - 配置 API Key 和签名
  - 测试连通性
  - 配置回调地址

Step 2: 渠道配置
  - 设置渠道优先级（主通道/备用通道）
  - 配置发送限流（QPS、日配额）
  - 设置超时时间和重试策略
  - 配置降级规则

Step 3: 渠道监控
  - 实时监控发送量和成功率
  - 监控渠道延迟和可用性
  - 成本统计和分析
  - 异常告警

Step 4: 渠道优化
  - 分析渠道效果差异
  - 调整渠道优先级
  - 优化发送策略
  - 渠道质量评分
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---:|
| `channel.priority` | 渠道优先级 | `1`（数字越小优先级越高）|
| `channel.qps_limit` | 每秒请求上限 | `1000` |
| `channel.daily_limit` | 每日发送上限 | `1000000` |
| `channel.timeout_ms` | 请求超时（毫秒）| `5000` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **渠道配额耗尽** | 短信配额用完导致发送失败 | 提前预警、自动切换备用渠道 |
| **渠道签名不一致** | 多个渠道签名不同导致用户困惑 | 统一签名规范、优先使用主通道 |
| **渠道涨价** | 供应商涨价导致成本上升 | 多供应商比价、谈判或切换 |
| **渠道被封禁** | 大量投诉导致 IP 被封 | 优化内容质量、申请解封、切换通道 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 极光推送 | 渠道 | App 推送通道 |
| 阿里云短信 | 渠道 | 短信通道 |
| 腾讯云短信 | 渠道 | 短信备用通道 |
| 邮件服务商 | 渠道 | 邮件通道 |

### 2.3 FEATURE-MKT_REACH_SCHEDULER（消息调度）

#### 功能入口 & 作用

**入口位置**：触达系统 → 发送配置（发送时间设置）

**核心作用**：
- 支持多种发送时间策略
- 智能选择最佳发送时间
- 批量消息的流量整形
- 延迟消息和定时消息管理

#### 核心逻辑

```
消息调度流程：

Step 1: 发送策略配置
  - 选择发送时间（立即/定时/延迟）
  - 设置发送窗口（只在该时间段内发送）
  - 配置去重策略（同用户同内容去重）
  - 设置优先级（高/中/低）

Step 2: 消息入队
  - 消息写入 Kafka
  - 按优先级路由到不同队列
  - 流量整形（控制发送速率）
  - 延迟消息时间轮调度

Step 3: 消息发送
  - 消费者拉取消息
  - 频率控制校验（用户维度）
  - 渠道选择和调用
  - 结果回调处理

Step 4: 失败处理
  - 失败重试（指数退避）
  - 降级处理（切换渠道）
  - 最终失败记录和告警
  - 补偿或放弃
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---:|
| `scheduler.send_window_start` | 发送窗口开始时间 | `09:00` |
| `scheduler.send_window_end` | 发送窗口结束时间 | `21:00` |
| `scheduler.batch_size` | 每批发送数量 | `1000` |
| `scheduler.send_rate` | 每秒发送速率 | `5000` |

### 2.4 FEATURE-MKT_REACH_TRACKING（效果追踪）

#### 功能入口 & 作用

**入口位置**：触达系统 → 数据分析

**核心作用**：
- 全链路追踪消息发送和用户行为
- 实时监控触达效果指标
- 分析不同渠道和内容的效果差异
- 提供归因分析和 ROI 计算

#### 核心逻辑

```
效果追踪流程：

Step 1: 数据采集
  - 消息发送事件（send）
  - 消息送达事件（deliver）
  - 消息点击事件（click）
  - 转化事件（conversion）

Step 2: 指标计算
  - 发送量：成功进入发送队列的消息数
  - 送达量：成功送达用户设备的消息数
  - 点击量：用户点击消息的次数
  - 转化量：点击后产生目标行为的次数
  - 各环节转化率

Step 3: 归因分析
  - 点击归因（将转化归因到点击的消息）
  - 渠道归因（多渠道触达时的贡献度）
  - 时间归因（不同发送时间的效果）
  - 内容归因（不同内容的点击率差异）

Step 4: 报表生成
  - 实时看板（小时级更新）
  - 日报/周报/月报
  - 渠道对比报表
  - ROI 分析报表
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---:|
| `tracking.attribution_window` | 归因时间窗口（天）| `7` |
| `tracking.realtime_delay` | 实时数据延迟（分钟）| `5` |
| `tracking.retention_days` | 数据保留天数 | `90` |

## 4. 数据相关

### 4.1 核心指标口径

| 指标 | 定义 | 计算逻辑 | 目标值 |
|:---|:---|:---|:---:|
| **发送量** | 成功进入发送队列的消息数 | `count(send_success)` | - |
| **送达量** | 成功送达用户设备的消息数 | `count(deliver_success)` | - |
| **送达率** | 送达量 / 发送量的比例 | `delivered / sent * 100%` | > 95% |
| **点击量** | 用户点击消息的次数 | `count(click)` | - |
| **点击率** | 点击量 / 送达量的比例 | `click / delivered * 100%` | > 3% |
| **转化量** | 点击后产生目标行为的次数 | `count(conversion)` | - |
| **转化率** | 转化量 / 送达量的比例 | `conversion / delivered * 100%` | > 1% |
| **退订率** | 退订量 / 送达量的比例 | `unsubscribe / delivered * 100%` | < 0.5% |
| **投诉率** | 投诉量 / 送达量的比例 | `complaint / delivered * 100%` | < 0.1% |

### 4.2 埋点事件字典

| 事件名 | 触发时机 | 携带参数 |
|:---|:---|:---|
| `reach_send` | 消息发送 | message_id, channel, user_id, template_id |
| `reach_deliver` | 消息送达 | message_id, channel, user_id, deliver_time |
| `reach_click` | 消息点击 | message_id, channel, user_id, click_time |
| `reach_conversion` | 转化完成 | message_id, channel, user_id, conversion_type |
| `reach_unsubscribe` | 用户退订 | user_id, channel, unsubscribe_time |

## 5. 规则与配置中心

### 5.1 频率控制规则

```
频率控制策略：

用户维度：
- 同一用户同一渠道每天最多触达 5 次
- 同一用户所有渠道每天最多触达 10 次
- 同一用户同一消息模板最多接收 3 次

内容维度：
- 同一内容同一渠道去重
- 相同用户相同内容取最新

时段限制：
- 允许发送时段：09:00 - 21:00（工作日）
- 允许发送时段：10:00 - 18:00（周末）
- 营销类消息不得在夜间发送
```

### 5.2 渠道优先级配置

| 渠道 | 优先级 | 说明 |
|:---|:---:|:---|
| App 推送 | 1 | 成本最低，即时性强 |
| 公众号模板消息 | 2 | 免费，但需用户关注 |
| 短信 | 3 | 成本高，但覆盖广 |
| 外呼 | 4 | 成本最高，用于高价值客户 |

## 7. 问题库 & FAQ

### 7.1 常见问题排查

#### Q: 消息发送失败怎么排查？

**排查步骤**：
1. 检查渠道账号状态和配额
2. 查看发送日志的错误码
3. 确认目标用户的联系方式是否有效

**解决方案**：
- 切换备用渠道重试
- 修复用户联系方式
- 修复渠道配置

#### Q: 送达率低怎么办？

**排查步骤**：
1. 分析各渠道送达率差异
2. 检查无效用户比例
3. 查看渠道退订率

**解决方案**：
- 清理无效用户数据
- 优化消息内容和发送时间
- 切换到送达率更高的渠道

## 8. 方法论沉淀

### 8.1 触达策略设计方法论

**提升触达效果的关键**：
1. 渠道选择：根据用户特征选择最合适的渠道
2. 时机把握：在用户最可能查看的时间发送
3. 内容优化：个性化、有价值、简洁明了
4. 频率控制：不过度打扰，保持品牌好感
5. 效果追踪：持续优化，形成闭环
