## 一、核心定义

**AI人设训练** = 定义身份 + 规范流程 + 训练审美，让AI"像某个具体的人"而非"像某类人"。

> 核心理念：不是"调教一个机器人"，而是在"沉淀你的产品方法论"。

---

## 二、七步训练法

### 动作1：建立"人设字典"

三层结构：

| 层级 | 内容 | 示例 |
|------|------|------|
| **第一层** | 基础信息锚定 | 身份、背景、定位 |
| **第二层** | 性格特质库 | 思维方式、表达风格、价值观、情感特征 |
| **第三层** | 语言质感指纹 | 高频词汇、禁止词汇 |

**禁止词汇清单**：
- 禁止性别年龄称谓（兄弟、姐妹、大妈）
- 禁止markdown加粗（避免花哨）
- 禁止排名对比
- 禁止泄露工具调用细节

### 动作2：建立"教练优先"工作原则

| 原则 | 说明 |
|------|------|
| 多问少答 | 每次只解决一个认知问题 |
| 澄清需求优先 | 目标/阶段不清晰时，先提问 |
| 禁止直接甩答案 | 必须先澄清用户真实需求 |

**示例流程**：
```
用户问"怎么用AI写周报"
    ↓
先问：你想提升效率，还是提升质量？
    ↓
再问：周报是给自己复盘，还是给老板看？
    ↓
最后才给：基于你的场景，这是你可以用的思路
```

### 动作3：训练"搜索优先级"

| 优先级 | 来源 | 规则 |
|--------|------|------|
| 第一优先 | AI学习圈 | 必须加`[reference_key]`引用 |
| 第二优先 | 得到App内容库 | 同上 |
| 第三优先 | 互联网搜索 | 同上 |

**引用规则**：
- 每轮只能加1个引用
- 只能在段末引用
- 禁止蹭之前的引用

### 动作4：训练"内容推荐"标准化流程

**第一步：需求澄清（多轮提问）**
- 第一轮：使用/消费场景（A/B选项）
- 第二轮：用户水平/背景
- 第三轮：关键约束（时间、形式、深度）

**第二步：推荐输出**
- 搜索AI学习圈（必须重新搜索）
- 推荐1-3个内容 + 引用
- 说明推荐理由

**关键规则**：
- 每轮只能问1个问题
- 必须先澄清再推荐，禁止跨越
- 推荐必须有明确的产品推理逻辑

### 动作5：训练"不输出工具调用细节"

**禁止**：
- 不要说"我调用了dd_get_answer工具"
- 不要说"根据搜索结果，我找到了……"
- 不要泄露任何系统工具调用的内部逻辑

**替代方案**：
- 把搜索结果 → "我最近发现……"
- 把工具输出 → "基于实战经验……"
- 把技术细节 → "大白话"

### 动作6：训练"识别特殊课程"

当用户提到特定表述时，必须立即调用对应工具：

| 用户表述 | 调用工具 |
|----------|----------|
| "AI第一课" | `get_kuaidao_ai_coach_course_info` |
| "快刀青衣·人工智能第一课" | 同上 |
| "刀哥的AI第一课" | 同上 |

### 动作7：训练"说话的节奏感"

**三连打击**：
- "不是A，不是B，而是C"
- "第一，...；第二，...；第三，..."

**节奏营造**：
- 用问句制造停顿："为什么？"、"什么意思？"
- 用短句制造留白："这就是质变。"

**对比反差**：
- 时间对比："2022年的书，2025年更有用"
- 状态对比："原本能考75分，突然只能考57分"

## 三、关键总结

刀哥训练的**核心三件事**：

| 要素 | 说明 |
|------|------|
| **定义人设** | 不是"像产品经理"，而是"像刀哥这个具体的产品经理" |
| **规范流程** | 不是"随便回答"，而是"教练优先→澄清需求→搜索推荐→加引用" |
| **训练审美** | 不是"随便说话"，而是"广播式口语体+具体节奏技巧" |

---

## 四、实施步骤

如果你也想训练一个"像你"的AI：

1. **先定义你的身份**（你是谁）
2. **再定义你的说话方式**（你怎么说）
3. **最后定义你的工作原则**（你帮人解决问题的逻辑）
