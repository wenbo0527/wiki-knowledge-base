## 消费金融可解释性核心要求
消费金融场景对模型可解释性有特殊要求，和其他行业不同：
1. **高并发实时解释**：贷前审批、反欺诈需要在<100ms内返回解释，解释必须提前预计算
2. **自然语言友好**：解释面向普通用户，不能是SHAP值、系数等技术术语
3. **国内监管强制要求**：必须明确告知拒贷理由，不能模糊表述，理由必须留存5年
4. **公平性要求高**：必须证明模型没有歧视性，解释中不能出现禁止类特征
---
## 典型场景1：贷前准入模型（最高要求）
### 业务场景
用户申请消费贷款，系统在<500ms内做出审批决策，同时生成可向用户展示的拒贷理由。
### 合规要求
- 拒贷理由必须包含具体的失分项，比如"近3个月贷款查询次数超过5次"，不能仅说"综合评分不足"
- 理由必须和模型决策逻辑一致，不能出现模型判断通过但理由说拒绝的自相矛盾
- 理由中不能提及性别、民族、宗教信仰、年龄（40岁以上）等禁止类特征
### 落地方案
```
架构：模型预测 → SHAP实时解释 → Top3特征提取 → 理由模板映射 → 自然语言生成
延迟：模型预测30ms + SHAP预计算<50ms + 理由生成<20ms = 总延迟<100ms
存储：用户ID + 时间 + 决策结果 + 模型版本 + 理由内容 → 保存5年
```
### 拒贷理由模板示例
| 场景 | 理由生成示例 |
|------|-------------|
| 征信多头 | "您的近3个月贷款申请查询次数为6次，超过了平台安全阈值（5次），表明您近期在其他机构的贷款申请较为频繁，还款能力存在不确定性，这是本次申请被拒的主要原因。" |
| 收入负债比高 | "您的当前负债总额与月收入比例为8倍，高于平台安全标准（5倍），还款压力较大，为保障您的财务安全，本次申请暂无法通过。" |
| 历史逾期 | "您在人行征信记录中显示近2年内有3次逾期记录，虽然已全部还清，但历史信用表现是平台评估还款意愿的重要依据，建议您保持良好的信用记录后再尝试申请。" |
| 关联风险 | "系统检测到您的关联用户（如紧急联系人）中有逾期记录，根据平台风控规则，关联风险较高的用户需要更严格的审核，本次暂无法通过。" |

## 典型场景2：反欺诈实时拦截（次高要求）
### 业务场景
用户在交易过程中触发了反欺诈规则，系统实时拦截交易，同时生成解释用于客服申诉。
### 合规要求
- 解释必须能支撑客服向用户说明拦截原因，用户申诉时能作为判断依据
- 不能暴露具体的反欺诈规则逻辑，避免被攻击者利用
- 不能出现歧视性表述
### 落地方案
```
架构：交易实时特征 → 模型预测 → 全局规则判断 → 风险等级评估 → 拦截原因映射
拦截等级：P0强制拦截（直接拒绝）、P1人工审核（需客服跟进）、P2预警（观察）
解释内容：仅说明"您的交易存在异常，为保护您的资金安全，系统已启动安全验证流程"
```
### 拦截原因模板示例
| 拦截类型 | 对客解释 | 内部记录 |
|----------|----------|----------|
| 异地登录 | "您的账号在非常用地区发起交易，为保护您的账户安全，已启动身份验证流程" | 登录IP与常用城市距离>500km，触发异地登录规则 |
| 大额异常 | "您的本次交易金额较大，系统已启动大额交易安全审核" | 交易金额超过历史最高值的3倍，触发大额异常规则 |
| 频繁转账 | "您的账户存在频繁转账行为，为防止诈骗风险，已启动延时到账机制" | 24小时内转账次数超过10次，触发频繁转账规则 |

## 典型场景3：贷中额度调整（中要求）
### 业务场景
用户使用贷款过程中，系统根据用户表现调整额度，额度的升、降、冻结都需要可解释。
### 合规要求
- 额度调整必须向用户说明调整原因，用户不同意可以申诉
- 升额可以模糊描述，降额、冻结必须具体说明触发的原因
### 落地方案
```
升额理由：可使用"综合信用表现良好，还款记录稳定，系统已为您提升额度"
降额理由：必须具体，如"近3个月信用卡使用率持续超过80%，负债水平较高"
冻结理由：必须具体，如"您的贷款已逾期15天，根据合同约定，系统已启动冻结流程"
```

## 典型场景4：催收策略差异化（中要求）
### 业务场景
根据用户的风险分、还款意愿、还款能力，制定差异化催收策略，不同催收强度对应不同用户。
### 合规要求
- 催收策略必须符合《催收自律公约》，不能暴力、骚扰、恐吓
- 解释仅用于内部管理，不需要向债务人公开
### 催收分群示例
| 用户分群 | 特征表现 | 催收策略 | 对外口径 |
|----------|----------|----------|----------|
| 高意愿高能力 | 历史还款及时、收入稳定、主动联系 | 短信提醒即可 | "您的本期账单已逾期，请及时还款以免影响信用" |
| 高意愿低能力 | 历史还款及时、收入下降、主动说明情况 | 协商还款方案 | "我们理解您目前的情况，建议您协商分期还款" |
| 低意愿高能力 | 历史多次逾期、收入高但故意不还 | 法律途径威慑 | "您的欠款已多次逾期，我们将采取进一步法律催收措施" |
| 失联用户 | 联系不上、地址变化、紧急联系人联系不上 | 外访、上门 | 仅内部记录，不对外口径 |

## 合规落地检查清单
✅ 所有拒贷理由已在模型上线前通过合规部门审核  
✅ 理由内容不包含性别、民族、地域、宗教、年龄等禁止类特征  
✅ 理由和模型决策逻辑一致，不存在矛盾  
✅ 理由已按监管要求存储至少5年，可查询、可追溯  
✅ 客服团队已培训理由口径，能向用户正确解释  
✅ 每季度做一次理由准确性抽检，确保理由和模型逻辑一致
