## 🔧 CLAUDE.md最佳实践

### 标准结构

```markdown
# CLAUDE.md - 项目配置

## 项目概述
简要描述项目是什么、做什么

## 技术栈
- Framework: ...
- 语言: ...
- 关键依赖: ...

## 代码规范
- 命名规则
- 提交规范
- 测试要求

## 工作流
1. 开发流程
2. 提交流程
3. 部署流程

## 注意事项
- 已知问题
- 特别约束
```

### 核心原则

| 原则 | 说明 |
|------|------|
| **简洁** | 60行以内最佳 |
| **具体** | 用数字而非模糊描述 |
| **可执行** | 每条规则都应可被Agent理解和执行 |
| **渐进** | 从简单开始，逐步完善 |
