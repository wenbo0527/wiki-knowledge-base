## Vibe Coding 是什么？

### 核心定义

| 维度 | 说明 |
|------|------|
| **本质** | 用自然语言描述需求，AI生成代码 |
| **关键** | 上下文固定 = 不频繁修改 |
| **角色** | PM是"审核者"，不是"执行者" |

### Vibe Coding vs 传统开发

| 维度 | 传统开发 | Vibe Coding |
|------|----------|-------------|
| 需求表达 | PRD文档 | 自然语言 |
| 实现方式 | 代码 | AI生成 |
| 迭代方式 | 改代码 | 改Prompt |
| PM角色 | 写文档 | 描述+审核 |
| QA角色 | 测试 | 验证+反馈 |

## PM如何描述需求？

### 好的需求描述

```markdown
## 需求描述

### 背景
用户需要在移动端快速查看数据报表，当前操作路径太长。

### 功能要求
1. 首页增加"数据看板"入口
2. 支持3种图表类型：折线图、柱状图、饼图
3. 支持下拉刷新

### 约束条件
- 页面加载时间 < 2秒
- 图表数据来自现有API
- 兼容iOS 14+ 和 Android 8+

### 验收标准
- [ ] 用户可在3步内看到数据看板
- [ ] 图表渲染正确
- [ ] 无明显卡顿
```

### PM需要避免的坑

| 坑 | 说明 | 解决方法 |
|----|------|----------|
| 上下文丢失 | 频繁开新对话 | 保持单一对话上下文 |
| 需求模糊 | "做一个好看的界面" | 明确具体指标 |
| 过度干预 | 频繁改Prompt | 先让AI完成再迭代 |

## 🛠️ PM的AI设计工具

### SuperDesign ⭐ 6.3k

**第一个开源的AI设计Agent，直接集成在IDE中**

```bash
安装: Cursor/VS Code Marketplace 搜索 SuperDesign
```

**功能**：
| 功能 | 说明 |
|------|------|
| Product Mock | 一句话生成完整UI界面 |
| UI Components | 生成可复用的UI组件 |
| Wireframes | 低保真线框图快速探索 |
| Fork & Iterate | 复制并迭代设计 |

**PM使用场景**：
```
描述需求 → AI生成10种设计方案 → 选择最优 → 微调 → 交给开发
```

**优势**：
> "Why design one option when you can explore ten?"
