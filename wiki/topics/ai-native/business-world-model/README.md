## 核心定义

### 什么是商业世界模型？

**本质**：面向商业经营决策的仿真模拟器

**核心价值**：
1. 为经营Agent提供虚拟训练环境
2. 降低真实环境试错成本
3. 提升迭代速度

### 为什么商业领域需要世界模型？

| 原因 | 说明 |
|------|------|
| **反馈周期长** | 定价调整后需数周甚至数月才能看到最终效果 |
| **试错成本高** | 不成熟的决策直接上线会带来很高经营损失 |
| **复杂决策** | 多策略叠加的复杂序列决策难以在真实环境测试 |

### 经典案例参考

| 领域 | 应用 | 效果 |
|------|------|------|
| AlphaGo | 虚拟环境中自我对弈迭代 | 超越人类棋手 |
| LLM编程 | 代码执行环境的快速反馈 | 编程能力大幅提升 |
| 自动驾驶 | 仿真路况做决策训练 | 降低实车测试风险 |
| 电商定价 | 商业世界模型仿真 | 精细化定价决策 |

## 核心框架

### Judea Pearl的AI能力三层框架

| 层级 | 能力 | 评估标准 |
|------|------|----------|
| **Seeing（预测）** | 基于历史数据预测未来结果 | 预测误差 |
| **Doing（决策）** | 推演不同决策的不同结果 | 无法直接观测 |
| **Imaging（反事实）** | 反思"如果当时采取另一决策会如何" | 反思能力 |

### 核心误区

> ⚠️ **预测准确不代表决策准确**

以价格-销量关系为例：
- 符合常识的因果模型（价格↑销量↓）预测误差反而更大
- 错误的关联模型（价格↑销量↑）反而在单点预测中误差更小
- 基于错误模型做定价决策，会得出完全错误的结论

**结论**：构建商业世界模型必须满足因果逻辑约束，不能仅以预测准确率作为建模目标

## 因果推断四步框架

### Step 1: 构建因果图

引入行业专家经验，明确：
- **决策变量**：可干预的因素（如价格）
- **结果变量**：要优化的指标（如销量）
- **混淆变量**：同时影响决策和结果的外部因素

**典型混淆变量示例**：
```
情人节 → 鲜花价格↑
情人节 → 鲜花销量↑
错误结论：涨价能提升销量（混淆变量导致虚假相关）
```

### Step 2: 数据准备

| 方法 | 说明 | 局限性 |
|------|------|--------|
| 随机实验法 | 随机选择部分商品做价格调整对比 | 成本高、适用范围有限 |
| 观测数据纠偏法 | 倾向得分匹配等方法消除偏差 | 仅能处理简单单变量决策 |

### Step 3: 模型训练

**DML（双重机器学习）**：
- 对数据做正交化分解
- 分离混淆变量的影响
- 输出不同干预下的结果估计

**输入**：
- $X$：商品属性信息
- $T$：干预决策（如是否变价）
- $Y$：结果输出（如销量）

### Step 4: 效果评估

**Uplift排序验证法**：
```
① 基于随机实验，一半商品做干预，一半不做
② 用因果模型预估每个商品的干预提升量，按大小排序
③ 分组计算真实提升量
④ 验证：模型效果好 → 真实提升量逐步下降，累计曲线先升后降
⑤ 随机排序的累计提升为直线
```

## 京东定价实践

### 场景背景

| 挑战 | 说明 |
|------|------|
| 规模大 | 京东自营万亿GMV，大量商品需要定价调整 |
| 竞争激烈 | 要求精细化快速响应市场变化 |
| 多目标 | 兼顾销售目标与供应链库存调节 |
| 核心变量 | 价格是体验三要素（产品、价格、服务）中最易调整的 |

### 问题定义

**价格不是固定数值，而是动态价格策略序列**：
- 日常价
- 周末促销
- 大促降价
- 满减
- 区域优惠
- 人群优惠券
- 多种策略可叠加

### 解决方案

**商业世界模型输入**：价格策略序列
**商业世界模型输出**：对应时段的销量序列
