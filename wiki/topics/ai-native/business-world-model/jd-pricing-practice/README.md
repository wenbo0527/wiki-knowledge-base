## 场景背景

### 京东定价面临的挑战

| 挑战 | 说明 |
|------|------|
| **规模大** | 京东自营万亿GMV，大量商品需要定价调整 |
| **竞争激烈** | 电商价格战，要求精细化快速响应 |
| **多目标** | 兼顾销售目标与供应链库存调节 |
| **可调整** | 价格是体验三要素中最易快速调整的变量 |

### 定价需要平衡的多目标

| 目标 | 策略 |
|------|------|
| **销售最大化** | 低价获客 |
| **利润最大化** | 高价保利润 |
| **库存平衡** | 积压库存降价去化，缺货商品稳价控量 |
| **用户体验** | 保持价格竞争力 |

## 问题定义

### 传统方法的局限性

**单变量因果建模无法应对真实商业场景**

| 传统方法 | 现实场景 |
|----------|----------|
| 单次单变量（降价/不降价） | 动态价格策略序列 |
| 固定价格 | 日常价、周末促销、大促、满减、区域优惠、人群券 |

### 价格不是固定数值，而是策略序列

```
┌─────────────────────────────────────────────────────────────┐
│  动态价格策略序列                                           │
├─────────────────────────────────────────────────────────────┤
│  日常价 → 周末促销 → 大促降价 → 满减 → 区域优惠 → 人群券   │
│  ↑                                                         │
│  多种策略可叠加                                             │
└─────────────────────────────────────────────────────────────┘
```

### 复杂定价策略下的销量响应建模

![复杂定价策略销量响应建模](../sources/images/business-world-model/jd-pricing-strategy-sequence.webp)

**核心问题**：预测高度组合定价策略的因果效应

**价格层级（Price Layer）**：

| 层级 | 策略 | 说明 |
|------|------|------|
| 1 | Base Price | 基础定价 |
| 2 | Bulk Discounts | 批量折扣 |
| 3 | Periodic Promos | 周期性促销 |
| 4 | Major Sales Events | 大促活动 |

**因果关系**：
```
价格策略序列（多层叠加） → 因果效应 → 销量序列
     ↑_______________________________↑
         同时考虑交互影响
```

## 解决方案

### 商业世界模型架构

```
输入层 → 价格策略序列（日常价、周末促、大促、满减、区域优惠、人群券）
    ↓
仿真引擎 → 因果推断模型（DML + 因果图）
    ↓
输出层 → 销量预测序列 + 策略优化建议
```

### 价格序列分解（Price Strategy Series Decomposition）

京东将复杂价格序列分解为**4个可解释因子**：

| 因子 | 含义 | 示例 |
|------|------|------|
| **T (Trend)** | 整体市场趋势 | 长期价格走势 |
| **S (Periodic/Seasonal)** | 周期性/季节性 | 周末/节假日模式 |
| **C (Calendar Effect)** | 日历效应 | 双11、618等大促 |
| **P (Promotion/Price War)** | 促销/价格战 | 活动优惠、竞品调价 |

**分解公式**：
```
Price(t) = T(t) + S(t) + C(t) + P(t) + ε
```

![价格序列分解与因果效应计算](../sources/images/business-world-model/price-sequence-decomposition.webp)

### 因果效应计算（Counterfactual Reasoning）

核心思想是**反事实推断**：

```
t+1时刻：
- 实际价格 Price(t+1)
- 反事实 Counterfactual(t+1)（如果没有干预会怎样）

因果效应 = Price(t+1) - Counterfactual(t+1)
```

这正是**Pearl的Do-Calculus**核心：
> $P(Y=y | do(X=x))$ vs $P(Y=y | X=x)$
> 
> "在干预X=x的条件下Y=y的概率" vs "在观测到X=x的条件下Y=y的概率"

### 核心技术

| 技术 | 作用 |
|------|------|
| **因果图构建** | 明确定价决策的影响因素 |
| **DML双重机器学习** | 分离混淆变量，输出因果效应 |
| **序列决策模型** | 处理多策略叠加的复杂场景 |
| **强化学习** | 策略优化与迭代 |

## 效果验证

### 动态因果响应模型实验结果

![动态因果响应模型实验结果](../sources/images/business-world-model/jd-causal-model-results.webp)

| 方法 | 平均GMV提升 | 曲线形态 |
|------|------------|----------|
| **因果模型** | 峰值 ~1.4 | 平滑、一致 |
| **传统方法** | 峰值 ~0.5-0.6 | 不稳定 |
| **随机方法** | ~0 | 直线 |

**关键观察**：
1. 因果模型显著优于传统方法，提升幅度相差2-3倍
2. 10个日期表现一致，曲线形态稳定（鲁棒性强）
3. 累计提升曲线呈典型"先升后降"形态（符合Uplift验证标准）

## 三维评估框架

### 价格决策仿真三维评估标准

![三维评估框架](../sources/images/business-world-model/three-dimensional-evaluation-framework.webp)

| 维度 | 评估内容 | 说明 |
|------|----------|------|
| **Sales Forecast Accuracy** | 预测误差最小化 | 传统评估维度 |
| **Causal Magnitude Consistency** | 因果效应幅度一致性 | 预测因果效应与实际效果大小匹配 |
| **Causal Logical Rationality** | 因果逻辑合理性 | 结果必须符合因果逻辑，而非虚假相关 |

### 框架解读

```
                    因果逻辑合理性
                           ↑
                          /
                         /
                        /
                       /
                      /
   因果效应幅度一致性 ←----------------→ 预测精度
                      \
                       \
                        \
                         \
                          ↓
```

**核心洞察**：
- 传统方法仅评估**单一维度**（预测精度）
- 优秀的模型需要在**三个维度都表现良好**
- "Better Model" = 三维平衡的最优点

## 关键洞察

### 1. 商业世界模型 vs 传统预测

| 维度 | 传统预测模型 | 商业世界模型 |
|------|-------------|--------------|
| **目标** | 预测准确 | 决策优化 |
| **方法** | 相关性 | 因果性 |
| **输入** | 历史数据 | 策略序列 |
| **输出** | 销量预测 | 最优策略 |

### 2. 为什么需要仿真？

| 原因 | 说明 |
|------|------|
| **反馈周期长** | 定价效果需数周甚至数月才能观测 |
| **试错成本高** | 不成熟的定价决策可能造成巨大损失 |
| **复杂交互** | 多策略叠加效果难以预测 |

### 3. 仿真环境的关键要素

| 要素 | 说明 |
|------|------|
| **真实性** | 仿真环境要尽可能接近真实市场 |
| **多样性** | 支持多种价格策略组合 |
| **高效性** | 仿真速度要快，支持快速迭代 |
| **可解释性** | 输出结果要能解释决策原因 |

### 4. 核心结论

> ⚠️ **预测准确 ≠ 决策准确**
> 
> 因果推断模型在预测误差上可能更大，但在**决策效果**（GMV提升）上显著优于传统预测模型
