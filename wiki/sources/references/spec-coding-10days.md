## Spec Coding工作流

### 核心理念
**在写代码之前，先写规格文档**

通过 `openspec` 工具，每个功能变更经历：
```
Proposal → Design → Specs → Tasks
```

### 工作流价值
1. **减少返工**: 在proposal阶段明确为什么以及怎么做
2. **适合复杂功能**: tasks分组让AI聚焦当前步骤
3. **可审计**: 完整决策链留有记录，方便回溯

## 四阶段开发流程

### 阶段一：设计阶段
- 产品方向确认 + UI设计稿 + PRD输出
- 使用Cursor + 设计规范Rules
- 从概念沟通到高保真HTML稿
- 产出：可直接用于开发对齐的视觉参考

### 阶段二：项目搭建（2工作日，20条指令）
- 以问答式交互为主
- AI回答架构问题，人决策后执行
- 搭建项目结构、配置开发环境
- 完成第一个核心列表页面

### 阶段三：功能开发（4工作日，89条指令）
- 引入Spec Coding工作流
- 约80%功能代码在此阶段完成
- 先与AI共同定义清晰的功能规格（Specification）
- AI基于"蓝图"自主编码

### 阶段四：细节打磨与生产部署（4工作日，108条指令）
- 功能迭代、系统重构
- 项目首页深度代码重构

```

├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Layer 1: CLAUDE.md                                        │
│  - 项目全局规范                                             │
│  - 技术栈约束                                               │
│                                                             │
│  Layer 2: Rules (设计规范)                                 │

│  - 组件规范                                                │

│  Layer 3: Specs (功能规格)                                 │

│  - 验收标准                                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 典型案例

#### 案例一：AI驱动产品设计
没有产品经理、没有UI设计师，一个工程师如何用AI独立完成全流程。

方法：
1. 让AI扮演产品经理：植入「首席产品专家」Persona提示词
2. 让AI扮演UI设计师：对话式生成高保真HTML
3. 让AI生成研发可读的PRD：基于HTML设计稿生成

#### 案例二：SDD驱动前端功能研发
增量交付「定时任务管理」完整模块，对接6个后端接口。

**MCP联调路径**：
```
接口URL → MCP直连文档 → 一次性获取字段/枚举/必填项 → 接口文件一次生成 → 联调一次通过
```

结果：6个接口零联调返工，同日额外交付2个完整模块

#### 案例三：SDD驱动系统重构
重构与新功能的根本差异：在活体系统上动手术

**知识问答首页重构**：
- 架构债务：业务组件与公共组件混放、useChat导出20+方法、ChatInterface接收17个props
- TASKS执行：9组34个子任务
- 每步有明确输入和验收标准
- 结果：34个任务全部完成，人工干预<5条指令

#### 案例四：复杂问题排障
**AI失效的特征**：
- 本地无法复现的问题（需等待CI，约10分钟/轮）
- 多根因互相掩盖（解决一层才暴露下一层）
- 隐性行为无文档（根因藏在依赖源码内部）

## AI能力边界

### AI适合的场景 ✅
- 从无到有的新功能开发
- 有明确规格的增量交付
- 代码生成、代码审查
- 有参考案例的常见问题

### AI失效的场景 ❌
- 本地无法复现的问题
- 多层嵌套的隐性依赖
- 需要等待长时间反馈的排查
- 无文档的框架/库内部问题
