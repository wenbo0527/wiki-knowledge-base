## 机器学习是什么？

**定义**：从数据中自动学习模式，用于预测或决策的算法体系。

```
传统编程：规则 + 数据 → 结果
机器学习：数据 + 结果 → 规则
```

---

## 三大类别

| 类别 | 定义 | 典型任务 | 常用算法 |
|------|------|----------|----------|
| **监督学习** | 学习输入→输出的映射，有标签数据 | 分类、回归、排序 | 逻辑回归、SVM、决策树、随机森林、XGBoost/LightGBM |
| **无监督学习** | 学习数据的内在结构，无标签数据 | 聚类、降维、异常检测 | K-Means、PCA、GMM、DBSCAN、AutoEncoder |
| **强化学习** | 通过与环境交互学习最优策略，延迟奖励 | 游戏、机器人、推荐、自动驾驶 | Q-Learning、DQN、PPO、A2C |

## 基础概念

### 1. 数据集划分
```
┌─────────────────────────────────────────────────────┐
│                     完整数据集                       │
├─────────────────┬─────────────────┬─────────────────┤
│   训练集(60%)   │  验证集(20%)    │  测试集(20%)    │
│ 训练模型参数    │ 调参、模型选择  │ 最终效果评估    │
└─────────────────┴─────────────────┴─────────────────┘
```

### 2. 评价指标
#### 分类任务
| 指标 | 公式 | 适用场景 |
|------|------|----------|
| **准确率 (Accuracy)** | 预测正确数 / 总数 | 平衡数据集 |
| **精确率 (Precision)** | TP / (TP+FP) | 重视误判成本的场景（如风控） |
| **召回率 (Recall)** | TP / (TP+FN) | 重视漏判成本的场景（如癌症筛查） |
| **F1 Score** | 2*P*R/(P+R) | P和R的调和平均，均衡场景 |
| **AUC** | ROC曲线下面积 | 排序能力、不平衡数据集 |

#### 回归任务
| 指标 | 公式 | 说明 |
|------|------|------|
| **MAE** | Σ|y_true - y_pred| / n | 平均绝对误差，鲁棒性好 |
| **MSE** | Σ(y_true - y_pred)² / n | 均方误差，对大误差惩罚大 |
| **RMSE** | √MSE | 均方根误差，和原数据同单位 |
| **R²** | 1 - (Σ(y_true - y_pred)² / Σ(y_true - y_mean)²) | 解释度，越大越好，最大值1 |

### 3. 过拟合与欠拟合
| 现象 | 原因 | 解决方法 |
|------|------|----------|
| **欠拟合** | 训练集/测试集效果都差 | 模型太简单，增加特征、增加模型复杂度 |
| **过拟合** | 训练集效果好，测试集效果差 | 模型太复杂，增加数据、正则化、早停、集成学习 |

## 核心理论

### 1. 偏差-方差权衡 (Bias-Variance Tradeoff)
```
总误差 = 偏差² + 方差 + 不可约误差
- 偏差：模型的预测值与真实值的平均差异（模型是否足够准确）
- 方差：模型在不同数据集上的预测值的差异（模型是否足够稳定）

模型复杂度低 → 高偏差、低方差 → 欠拟合
模型复杂度高 → 低偏差、高方差 → 过拟合
最优复杂度 → 偏差和方差的平衡
```

### 2. 正则化 (Regularization)
#### L1正则化 (Lasso)
- 损失函数 + λ*Σ|w_i|
- 会产生稀疏权重，适合特征选择
#### L2正则化 (Ridge)
- 损失函数 + λ*Σw_i²
- 权重不会为0，防止过拟合效果好
#### 正则化强度λ
- λ越大，模型越简单，越不容易过拟合
- λ过大 → 欠拟合
- λ过小 → 过拟合

### 3. 梯度下降 (Gradient Descent)
**核心思想**：沿着损失函数的负梯度方向更新参数，最小化损失
#### 变种
| 方法 | 说明 | 特点 |
|------|------|------|
| **批量梯度下降** | 每次用全部样本计算梯度 | 稳定但速度慢 |
| **随机梯度下降(SGD)** | 每次用单个样本计算梯度 | 速度快但不稳定 |
| **小批量梯度下降(mini-batch)** | 每次用一小批样本计算梯度 | 平衡速度和稳定性，最常用 |
| **Adam** | 自适应学习率 + 动量 | 收敛快，效果好，最常用 |

## 常用算法概览

### 1. 经典算法
| 算法 | 类别 | 优势 | 劣势 | 适用场景 |
|------|------|------|------|----------|
| **逻辑回归** | 监督/分类 | 简单、可解释、速度快 | 只能处理线性关系 | 基线模型、风控 |
| **SVM** | 监督/分类/回归 | 高维效果好、核函数强大 | 样本量大时速度慢 | 小样本高维场景 |
| **决策树** | 监督 | 可解释、不需要特征归一化 | 容易过拟合 | 集成学习的基模型 |
| **随机森林** | 监督 | 效果好、抗过拟合、不容易过拟合 | 训练速度慢、可解释性差 | 大多数结构化数据场景 |
| **XGBoost/LightGBM/CatBoost** | 监督 | 效果好、速度快、工业界标准 | 需要调参 | 竞赛、工业界首选 |
| **K-Means** | 无监督/聚类 | 简单、速度快 | 需要指定K值、对异常值敏感 | 客户分群、画像 |
| **PCA** | 无监督/降维 | 简单、可解释 | 线性降维、不保留局部结构 | 特征降维、可视化 |

### 2. 深度学习
| 模型 | 类别 | 适用场景 |
|------|------|----------|
| **DNN** | 深度神经网络 | 通用分类/回归 |
| **CNN** | 卷积神经网络 | 图像、时序 |
| **RNN/LSTM/GRU** | 循环神经网络 | 序列、文本 |
| **Transformer** | 注意力模型 | 文本、图像、多模态（大模型基础） |

## 项目流程

```
1. 需求理解 → 明确业务目标、评价指标
2. 数据采集 → 收集相关数据
3. 数据清洗 → 缺失值处理、异常值处理、去重
4. 特征工程 → 特征构造、特征选择、特征转换
5. 模型选择 → 从简单到复杂，先做基线模型
6. 模型训练 → 调参、交叉验证
7. 模型评估 → 在测试集评估，和基线对比
8. 模型部署 → 上线、监控
9. 迭代优化 → 根据线上反馈持续优化
```

## 最佳实践

### 1. 基线优先
- 先做简单模型作为基线（如逻辑回归、XGBoost默认参数）
- 复杂模型效果必须明显优于基线才有意义

### 2. 特征工程 > 模型
- 结构化数据场景：特征工程决定上限，模型只是逼近上限
- 优先花时间在特征工程上，而不是盲目堆复杂模型

### 3. 可解释性
- 风控、医疗等领域：可解释性 > 0.1个点的准确率提升
- 优先用可解释性好的模型（逻辑回归、决策树、线性模型）

### 4. 工程化考虑
- 训练速度、推理速度、内存占用都是工业界的重要考量
- 效果差不多的时候，优先选简单、速度快的模型
