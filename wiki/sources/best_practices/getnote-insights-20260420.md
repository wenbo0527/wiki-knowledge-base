## 一、Memos框架：企业级大模型记忆引擎

### 来源信息
- **课程时间**：2026-04-18 14:20:57
- **时长**：45分钟
- **领域**：计算机科学 / 大模型应用开发 / 智能体Agent
- **分享者**：唐波（快手/阿里巴巴/美团背景，现任职记忆张量）

### 核心内容

#### 1.1 大模型记忆的发展逻辑
- 从ChatGPT开始，大模型厂商持续强调记忆能力
- Agent时代到来后，长程任务对记忆的依赖度大幅提升
- **两种传统记忆方案对比**：
  - 模型层增强：直接在大模型中增加记忆能力，效果不如RAG
  - 外部记忆增强：基于应用层的记忆框架，效果好、实现简单，是当前主流路线

#### 1.2 Memos设计原则
- 结合模型驱动与应用驱动两种路线
- 模型驱动决定记忆能力上限，应用驱动决定记忆能力下限
- 通过内外协同实现深度与广度兼顾
- 支持分层记忆与多触点应用
- **从系统层面进行整体设计**

#### 1.3 记忆模块核心功能（五大功能）
1. **记忆抽取**：处理输入到记忆系统的内容
2. **记忆组织**：对抽取后的记忆进行结构化存储，可引入图存储组织实体关系
3. **记忆检索**：实现"存"与"用"的衔接，是记忆系统核心功能
4. **记忆更新**：处理记忆增量与演化
5. **记忆共享**：支持团队协作与多Agent协作，实现群体智能演化

#### 1.4 Memos分层技术架构

**记忆存储层**
- 以记忆立方体(Memory Cube)为核心组织单元
- 每个Cube对应用户/场景/知识库
- 两层存储：
  - 向量存储：基础检索存储
  - 图数据库(GraphDB)：存储复杂实体关系与因果关系
- **支持记忆模块分享**：可实现用户记忆继承，解决企业员工离职导致的知识流失问题

**记忆治理层**
- 提供记忆全生命周期管理
- 包含幻觉控制、水印、权限控制、隐私保护
- 保证记忆在不同权限范围内安全流通

**记忆调度层**
- 核心功能：工作记忆(Working Memory)管理
- **K Cache复用实现推理成本优化**：
  - 大模型推理中大量请求依赖重复知识
  - 传统检索排序不满足前缀匹配，导致K Cache无法复用
  - Memos通过对检索知识的有序存储实现K Cache复用
  - 可大幅降低调用API的成本

**记忆编码与应用层**
- 负责记忆加工与重组
- 对外提供API、MCP、本地化服务等各类接口
- 支持记忆反馈更新

#### 1.5 Memos OpenCloud插件设计

**原生OpenCloud记忆方案的痛点**：
1. 原生基于Markdown文件的记忆机制，依赖Agent自身能力实现压缩，内存压缩成本高，启动时间长
2. Token消耗量极大，复杂任务可达数百万甚至上千万级
3. 原生文件系统设计，非技术用户无法进行增删改查管理
4. 写入和检索触发机制不完善，容易重复询问用户
5. 多Agent之间记忆完全隔离，不支持协同场景下的记忆共享

**Memos插件核心优化**：
1. 提供可视化记忆管理界面
2. 引入Memos原生框架的存储、检索、记忆继承能力
3. **支持Agent进化能力**：通过与环境交互不断积累经验，规避过往踩过的坑
4. **支持多Agent记忆共享与协同进化**

**版本选择**：
- 云上版：支持高并发，50QPS下500ms内响应
- 本地版：数据保留在本地，适合对数据隐私有要求的用户

**核心增强能力**：
1. **记忆沉淀技能(Memory to Skill)**：自动从执行记录中沉淀可复用技能，支持团队共享与联合迭代
2. 局域网团队共享：支持搭建本地服务端，实现记忆、任务、技能的共享

#### 1.6 Agent时代的记忆框架演化方向

**新记忆框架的核心价值**：
- Agent执行过程中会产生大量中间轨迹
- 这些信息对后续相似任务有极高借鉴价值
- 可以帮助Agent快速定位问题、减少重复探索
- 实现Agent边运行边学习的自我演化

**核心设计思路**：
1. 分层反馈量化：整合执行过程反馈与用户反馈
2. 分层检索机制：同时召回沉淀技能、执行轨迹、环境场景认知三类信息
3. 五层记忆加工流程：任务案例构建 → 执行反馈价值回填 → 跨任务归纳总结 → 规律抽象构建认知 → 沉淀为四类知识

## 二、AI电商与商业世界模型：因果推断及京东定价实践

### 来源信息
- **课程时间**：2026-04-18 13:30:33
- **时长**：43分钟
- **领域**：人工智能 / 电商技术 / 因果推断应用

### 核心内容

#### 2.1 Agent Commerce（智能电商）概念

**核心定义**：通过AI赋能电商全链路，提升买卖双方的效率

**核心模块划分**：
- **买方端（Buy Side）**：面向消费者，AI导购、购物Agent
- **卖方端（Sell Side）**：面向经营者，品牌方/供应商的选品、定价、补货、履约决策
- **中间层**：身份认证、权限校验、交易链路打通

**代表性案例**：Asrock全AI托管自动售货机，多Agent协作完成选品设计、供应商对接、定价调价、补货全流程

#### 2.2 商业世界模型的作用与价值

**核心定义**：面向商业经营决策的仿真模拟器，为经营智能体提供虚拟训练环境

**核心原因**：
1. 经营决策的反馈周期长（定价调整后往往需要数周甚至数月才能看到效果）
2. 不成熟的决策直接上线会带来很高的经营损失

**经典案例**：
- AlphaGo在虚拟环境中完成自我对弈迭代
- 大语言模型编程能力依赖代码执行环境的快速反馈
- 自动驾驶依赖仿真路况做决策训练

#### 2.3 因果推断核心理论

**Judea Pearl的AI能力三层框架**：
1. **Seeing（预测）**：仅基于历史数据对未来结果做预测
2. **Doing（决策）**：可以推演不同决策对应的不同结果
3. **Imaging（反事实推断）**：对已经发生的历史做反思

**核心误区**：预测准确不代表决策准确
- 符合常识的因果模型预测误差反而更大
- 构建商业世界模型必须满足因果逻辑约束

**传统因果推断建模四步框架**：
1. 构建因果图
2. 数据准备（消除混淆变量偏差）
3. 模型训练（DML双重机器学习）
4. 效果评估（Uplift排序验证法）

#### 2.4 京东定价场景商业世界模型落地实践

**场景背景**：
- 京东自营万亿GMV，大量商品需要定价调整
- 电商价格竞争激烈，要求精细化快速响应
- 定价需要兼顾销售目标与供应链库存调节

**问题定义**：
- 真实商业场景中价格是动态价格策略序列
- 包含日常价、周末促销、大促降价、满减、区域优惠、人群优惠券等多种策略
- 传统单变量因果建模无法直接应用

**三维评估体系**：
1. **准确率维度**：比较模型输出的销量序列和真实销量序列的误差
2. **因果效应维度**：逐段评估价格调整带来的销量变化
3. **因果逻辑校验**：基于规则的合理性检测

**动态因果响应建模方案**：
- 商品基础信息：通过多模态大模型做特征表征
- 历史时序信息：通过时序网络编码
- 价格策略信息：创新使用大语言模型对自然语言描述的复杂价格策略做编码

**两阶段训练流程**：
1. 监督预训练（SFT）：先训练单维度平均价格的小因果模型
2. 强化学习优化：输入成对的价格序列，奖励函数结合多维度因果评估

## 三、MiniMax大模型与AI Agent产品分享

### 来源信息
- **课程时间**：2026-04-18 11:53:06
- **时长**：19分钟
- **领域**：人工智能 / 大模型产业分享 / AI Agent行业实践

### 核心内容

#### 3.1 公司基本情况
- MiniMax成立于2022年初，专注技术大模型研发
- 2026年上半年完成港股上市
- 产品矩阵：文本大模型、视频生成模型海螺、语音生成模型、音乐生成模型Music 2.6

#### 3.2 大模型迭代加速趋势
- 早期迭代周期约6个月
- 目前已缩短至每1.5-2个月迭代一次
- **加速原因**：
  - 大模型本身已经足够智能，可以辅助自身完成迭代工作
  - 公司已经形成成熟的研发pipeline循环

#### 3.3 AI参与迭代的分工机制
- **蓝色（纯人工）**：占比极低，仅负责定义下一代大模型方向、下发任务、关键决策
- **绿色（纯AI）**：大部分研发工作由AI完成
- **黄色（人机协作）**：AI给出初步结论，人工负责结论校验与最终决策

#### 3.4 MiniMax Agent产品迭代路径
1. 初始版本：支持生成完整前后端网站、客户端等开发任务
2. 技能化升级：引入Experts技能体系，支持用户通过自然语言描述创建自定义工作流（Skill）
3. 桌面端迭代：推出桌面端Workspace
4. 生态接入：结合OpenCloud、Hermes推出MaxCloud、MaxHermes

#### 3.5 AI Native公司内部落地实践

**文化定位**：全员遇到任何问题时，优先考虑让AI解决

**内部Agent实习生系统**：
- 打通公司内部所有系统，集成到飞书
- 可自动调用内部系统完成用户指令
- 覆盖技术、HR、行政、法务、财务、设计等非技术岗位

**系统架构**：通用底座+细分领域子Agent
- 通用底座复用MiniMax Agent的基础框架
- 各业务部门可自行开发对应场景的子Agent

**典型落地案例**：
1. HR招聘场景：自动登录招聘平台、筛选候选人、生成个性化打招呼内容
2. 算法研发场景：数据清洗、自动化实验、故障排查全流程引入AI
3. 技术支持场景：错误日志先由Agent实习生分析排查
4. 设计场景：交付带交互的可运行页面

#### 3.6 企业级Agent的未来发展方向
1. **安全环境隔离**：不同部门需要独立可控的Workspace
2. **企业级技能沉淀**：内部工作流技能化是核心需求
3. **多层级记忆体系**：需要支持个人记忆、团队记忆、公司记忆三个层级
4. **全时段全场景服务支撑**

## 五、Microsoft 2025年新工作未来报告

### 来源信息
- **报告名称**：New Future of Work Report 2025
- **发布时间**：2025年

### 核心洞察

#### 5.1 AI技术能力前沿进展
| 技术方向 | 核心进展 |
|---------|---------|
| 长周期任务完成 | 任务完成时间呈指数级加速，7个月翻倍 |
| 可验证推理优化 | 无需标注推理轨迹，通过RL后训练提升表现 |

- **2024年全球私人AI投资**：339亿美元（同比增长18.7%）
- **企业ChatGPT消息量**：过去一年增长8倍
- **高采用率行业**：科技/电信（72%）、专业服务（68%）、金融（65%）
