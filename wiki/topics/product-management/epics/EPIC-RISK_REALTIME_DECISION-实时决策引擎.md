## 1. 产品总览

### 1.1 一句话定位
**基于实时数据流，实现毫秒级的风险识别和智能决策，支持风险事件的即时响应和自动化处置。**

### 1.2 产品定位 & 核心价值

| 维度 | 描述 |
|:---|:---|
| **目标用户** | 风控分析师、风控运营、策略分析师、系统管理员 |
| **核心痛点** | 风险识别滞后、人工处理效率低、决策规则僵化 |
| **解决方案** | 实时流处理 + 智能决策引擎 + 规则可视化配置 + 全链路监控 |
| **核心价值** | 风险识别时效从小时级缩短到毫秒级，自动化处置率提升至 80% |

## 2. 功能结构（按模块归档）

### 2.1 FEATURE-RISK_RULE_ENGINE（规则引擎）

#### 功能入口 & 作用

**入口位置**：数字风险 → 实时决策 → 规则配置

**核心作用**：
- 可视化规则配置和编辑
- 规则版本管理和回滚
- 规则测试和模拟
- 规则执行监控和统计

#### 核心逻辑

```
规则配置流程：

Step 1: 创建规则
  - 选择规则类型（条件规则、评分规则）
  - 配置规则条件（字段、运算符、阈值）
  - 配置规则结果（命中/未命中动作）
  - 设置规则优先级

Step 2: 规则测试
  - 使用历史数据测试规则
  - 模拟规则执行结果
  - 查看命中率和效果

Step 3: 规则发布
  - 提交规则审核
  - 配置灰度发布
  - 监控规则执行

Step 4: 规则优化
  - 分析规则效果
  - 调整规则参数
  - 下线无效规则
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---|
| `rule.max_count` | 单策略最大规则数 | `100` |
| `rule.timeout` | 规则执行超时（ms）| `50` |
| `rule.cache_enabled` | 名单缓存开关 | `true` |
| `rule.version_enabled` | 版本管理开关 | `true` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **规则冲突** | 多规则对同一请求判定矛盾 | 规则优先级机制、互斥规则组 |
| **规则性能差** | 复杂规则导致决策延迟 | 规则简化、预计算、优化执行顺序 |
| **规则误杀** | 正常请求被错误拦截 | 增加人工审核兜底、规则灰度发布 |
| **规则难以理解** | 规则逻辑复杂难以维护 | 规则注释、分组管理、标准化条件 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 名单服务 | 依赖 | 黑名单、白名单查询 |
| 特征服务 | 依赖 | 决策特征获取 |
| 监控系统 | 下游 | 规则执行监控 |

**核心作用**：
- 模型版本管理
- 模型在线推理服务
- 模型效果监控
- A/B 测试支持

#### 核心逻辑

```
模型服务流程：

Step 1: 模型注册
  - 上传模型文件（PMML/ONNX/自定义）
  - 配置模型参数
  - 配置输入输出字段映射

Step 2: 模型部署
  - 选择部署版本
  - 配置资源配额
  - 灰度上线策略

Step 3: 模型推理
  - 接收推理请求
  - 特征预处理
  - 模型推理计算
  - 结果后处理

Step 4: 模型监控
  - 请求量和延迟监控
  - 模型效果监控
  - 特征漂移检测
  - 自动回滚
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---|
| `model.max_qps` | 最大 QPS | `1000` |
| `model.timeout` | 推理超时（ms）| `100` |
| `model.fallback_enabled` | 降级开关 | `true` |
| `model.cache_enabled` | 结果缓存开关 | `false` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **模型延迟高** | 复杂模型推理耗时长 | 模型压缩、GPU 加速、异步推理 |
| **模型效果下降** | 线上模型效果变差 | 效果监控、回滚机制、定期迭代 |
| **模型版本混乱** | 多版本并存难以管理 | 强制版本号、版本废弃策略 |
| **特征缺失** | 线上特征与训练不一致 | 特征一致性校验、特征监控 |

### 2.3 FEATURE-RISK_DECISION_FLOW（决策流编排）

#### 功能入口 & 作用

**入口位置**：数字风险 → 实时决策 → 决策流配置

**核心作用**：
- 可视化决策流编排
- 多阶段决策串联
- 决策分支和汇聚
- 决策结果路由

#### 核心逻辑

```
决策流执行流程：

Start → 特征获取 → 规则判断 → 模型推理 → 综合决策 → 处置执行 → End

节点类型：
- Start：决策流入口
- Feature：特征获取节点
- Rule：规则判断节点
- Model：模型推理节点
- Decision：综合决策节点
- Action：处置执行节点
- End：决策流结束

分支类型：
- IF-ELSE：条件分支
- SWITCH：多值分支
- MERGE：结果汇聚
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---:|
| `flow.max_depth` | 最大决策深度 | `10` |
| `flow.timeout` | 决策流超时（ms）| `200` |
| `flow.debug_enabled` | 调试模式开关 | `false` |
| `flow.sla_tracking` | SLA 追踪开关 | `true` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **流程复杂** | 决策流过于复杂难以维护 | 拆分子流程、模块化设计 |
| **循环依赖** | 节点之间循环依赖导致死循环 | 环检测、禁止循环配置 |
| **分支过多** | 分支太多难以覆盖测试 | 分支覆盖测试、简化分支逻辑 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 规则引擎 | 依赖 | 规则执行节点 |
| 模型引擎 | 依赖 | 模型推理节点 |
| 处置系统 | 下游 | 执行处置动作 |

## 5. 规则与配置中心

### 5.1 决策性能保障规则

```
性能保障策略：

1. 分层降级
   - L0：缓存命中，直接返回
   - L1：简单规则执行（< 10ms）
   - L2：复杂规则 + 模型（< 100ms）
   - L3：异步人工审核（> 100ms）

2. 限流熔断
   - 单接口限流：10000 QPS
   - 熔断阈值：错误率 > 10%
   - 降级策略：返回默认决策

3. 资源隔离
   - 高优请求：独立资源池
   - 普通请求：共享资源池
```

### 5.2 决策监控规则

```
监控告警规则：

1. 延迟告警
   - P99 > 500ms → P1 告警
   - P99 > 1000ms → P0 告警

2. 可用性告警
   - 成功率 < 99.9% → P1 告警
   - 成功率 < 99% → P0 告警

3. 规则告警
   - 规则命中率异常 → 告警
   - 规则执行失败 → 告警
```

### 5.3 处置动作配置

| 动作类型 | 说明 | 适用场景 |
|:---|:---|:---|
| **放行** | 通过决策 | 低风险 |
| **拦截** | 阻断请求 | 高风险 |
| **人工审核** | 转人工处理 | 中等风险 |
| **延迟处理** | 异步处理 | 需要异步确认 |
| **加强认证** | 需要额外验证 | 需要二次确认 |

## 7. 问题库 & FAQ

### 7.1 常见问题排查

#### Q: 决策延迟突然升高怎么办？

**排查步骤**：
1. 检查是哪个阶段延迟高
2. 查看是否有资源瓶颈
3. 检查是否有异常请求

**解决方案**：
- 扩容资源
- 优化规则执行
- 启用降级策略

#### Q: 规则未生效怎么办？

**排查步骤**：
1. 检查规则是否已发布
2. 查看规则测试结果
3. 确认规则优先级

**解决方案**：
- 重新发布规则
- 调整规则优先级
- 清除缓存

#### Q: 模型效果突然下降怎么办？

**排查步骤**：
1. 检查特征是否正常
2. 查看模型版本
3. 对比线上线下数据

**解决方案**：
- 回滚模型版本
- 检查特征工程
- 重新训练模型

## 8. 方法论沉淀

### 8.1 决策引擎设计原则

```
设计原则：

1. 高性能
   - 异步处理减少阻塞
   - 预计算减少实时计算
   - 缓存减少重复计算

2. 高可用
```
