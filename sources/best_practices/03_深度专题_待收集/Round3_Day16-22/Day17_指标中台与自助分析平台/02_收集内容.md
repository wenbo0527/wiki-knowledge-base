## 一、指标体系设计方法论

### 1.1 指标分类体系

```
┌────────────────────────────────────────────────────────────────┐
│                    指标分类体系                                   │
├────────────────────────────────────────────────────────────────┤
│  按业务维度分类                                                   │
│  ├── 运营指标：DAU/MAU、留存、转化                              │
│  ├── 财务指标：营收、利润、成本                                  │
│  ├── 风控指标：逾期率、欺诈率、通过率                            │
│  └── 营销指标：触达量、响应率、ROI                              │
├────────────────────────────────────────────────────────────────┤
│  按计算层级分类                                                   │
│  ├── 原子指标：最基础的业务度量，不可拆分                         │
│  │   └── 如：订单数、支付金额、注册用户数                        │
│  ├── 派生指标：原子指标+时间周期+筛选条件                         │
│  │   └── 如：今日订单数 = 订单数(时间=今日)                      │
│  └── 复合指标：多个指标计算得出                                  │
│      └── 如：转化率 = 订单数 / 访问数                            │
├────────────────────────────────────────────────────────────────┤
│  按受众分类                                                      │
│  ├── 战略指标：CEO/CRO关注，如总收入、NPS                        │
│  ├── 运营指标：业务负责人关注，如DAU、转化率                     │
│  └── 过程指标：运营人员关注，如功能点击率                        │
└────────────────────────────────────────────────────────────────┘
```

### 1.2 指标命名规范

```markdown
## 指标命名规范

### 命名结构
{业务域}_{业务主题}_{度量}_{时间周期}[_{修饰词}]

### 示例
- 营销_获客_新增用户数_日
- 营销_获客_新增用户数_日_付费渠道
- 风控_信贷_不良率_月
- 风控_信贷_不良率_月_消费金融

### 禁用词
- ❌ "实际"、"真实"、"最终"等模糊词
- ❌ 口语化表述
- ❌ 英文缩写混用

### 同义词管理
| 业务含义 | 标准词 | 禁用词 |
|---------|--------|--------|
| 日活跃用户 | DAU | 日活、DAU用户 |
| 转化客户数 | 转化客户数 | 成单数、成交数 |
| 逾期率 | 逾期率 | 坏账率、逾期比例 |
```

### 2.2 指标计算引擎核心逻辑

```python
class MetricEngine:
    """
    指标计算引擎
    支持离线、实时、预聚合三种计算模式
    """
    
    def __init__(self):
        self.offline_engine = SparkEngine()      # 离线计算
        self.realtime_engine = FlinkEngine()    # 实时计算
        self.pre_agg_engine = ClickHouseEngine() # 预聚合
    
    def calculate(self, metric_request):
        # 1. 解析指标请求
        metric_def = self.parse_metric_request(metric_request)
        
        # 2. 选择计算模式
        if self.is_pre_agg(metric_def):
            # 预聚合计算（最快的查询模式）
            return self.pre_agg_engine.query(metric_def)
        
        elif metric_request. realtime_required:
            # 实时计算（T+0）
            return self.realtime_engine.calculate(metric_def)
        
        else:
            # 离线计算（T+1）
            return self.offline_engine.calculate(metric_def)
    
    def parse_metric_request(self, request):
        """
        解析指标请求，提取：
        - 原子指标
        - 时间周期
        - 筛选条件
        - 分组维度
        """
        # 使用指标配置库解析口径
        metric_def = self.meta_store.get_metric(request.metric_id)
        
        # 应用用户指定的参数
        metric_def.apply_filters(request.filters)
        metric_def.apply_dimensions(request.dimensions)
        metric_def.apply_time_range(request.time_range)
        
        return metric_def
```

### 3.2 语义层设计

```python
class SemanticLayer:
    """
    语义层：将业务概念映射到技术实现
    
    核心功能：
    1. 业务术语 → 技术字段 的映射
    2. 自动生成 SQL
    3. 权限控制下推
    """
    
    def __init__(self):
        self.measure_registry = MeasureRegistry()    # 度量注册
        self.dimension_registry = DimensionRegistry() # 维度注册
        self.relationship_graph = RelationshipGraph() # 关系图谱
    
    def natural_language_to_sql(self, query):
        """
        自然语言转SQL
        输入: "昨天各渠道的新增付费用户数"
        输出: SELECT channel, COUNT(DISTINCT user_id) 
              FROM dwd_pay_detail
              WHERE ...
        """
        # 1. NL解析
        parsed = self.nl_parser.parse(query)
        
        # 2. 术语映射
        mapped = self.map_business_terms(parsed)
        
        # 3. SQL生成
        sql = self.sql_generator.generate(mapped)
        
        # 4. 权限控制
        sql = self.apply_permissions(sql)
        
        # 5. 优化
        sql = self.query_optimizer.optimize(sql)
        
        return sql
    
    def map_business_terms(self, parsed_query):
        """
        业务术语映射
        """
        mappings = {
            '昨天': {'time_type': 'daily', 'offset': -1},
            '新增付费用户': {
                'table': 'dwd_pay_detail',
                'field': 'user_id',
                'condition': 'first_pay_time IS NOT NULL'
            },
            '渠道': {
                'field': 'channel_name',
                'table': 'dim_channel'
            }
        }
        
        # 应用映射
        for term, mapping in mappings.items():
            if term in parsed_query:
                parsed_query.apply_mapping(term, mapping)
        
        return parsed_query
```
