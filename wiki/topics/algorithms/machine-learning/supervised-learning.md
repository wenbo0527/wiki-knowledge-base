## 1. 线性模型

### 1.1 线性回归
**模型形式**：
```
y = w0 + w1*x1 + w2*x2 + ... + wn*xn = W^T * X
```
**损失函数**：均方误差MSE
```
L(W) = Σ(y_true - W^T*X)^2 / 2n
```
**优化方法**：
- 最小二乘法：闭式解 W = (X^T X)^{-1} X^T y
- 梯度下降：迭代更新

### 1.2 逻辑回归 (Logistic Regression)
**模型形式**：用Sigmoid函数将线性输出映射到[0,1]，表示概率
```
p(y=1|x) = σ(W^T X) = 1 / (1 + e^{-W^T X})
```
**损失函数**：交叉熵损失
```
L(W) = - [y*log(p) + (1-y)*log(1-p)]  对所有样本求和
```
**优点**：
- 简单、速度快、可解释性好（系数表示特征的重要性）
- 输出是概率，可以调整阈值
- 工业界基线首选
**缺点**：
- 只能处理线性可分问题
- 对异常值敏感

### 1.3 正则化线性模型
| 模型 | 正则项 | 特点 |
|------|------|------|
| Ridge回归 | L2 | 防止过拟合，系数不会为0 |
| Lasso回归 | L1 | 特征选择，系数会变成0 |
| ElasticNet | L1 + L2 | 结合两者优点，最常用 |

## 2. 树模型

### 2.1 决策树
**核心思想**：层层分裂节点，每个节点是一个特征判断规则

**分裂准则**：
- 分类树：信息增益、信息增益率、基尼系数
- 回归树：均方误差MSE

**优点**：
- 可解释性强（可以画树图）
- 不需要特征归一化、对异常值鲁棒
- 可以处理非线性关系
**缺点**：
- 容易过拟合
- 不稳定，数据稍微变化树的结构就可能变化

### 2.2 随机森林 (Random Forest)
**核心思想**：集成多棵决策树，每棵树用随机样本和随机特征训练
```
训练过程：
1. 有放回采样N个样本（Bootstrap采样）
2. 随机选择k个特征，选择最优分裂
3. 重复以上步骤训练M棵树
预测：分类任务投票，回归任务平均
```
**优点**：
- 抗过拟合能力强
- 效果好，不容易调参
- 可以输出特征重要性
**缺点**：
- 训练速度比单棵树慢
- 比决策树可解释性差
- 对高维稀疏特征效果不如线性模型

### 2.3 梯度提升树 (GBDT)
**核心思想**：串行训练多棵树，每棵树学习之前所有树的残差
```
训练过程：
1. 第一棵树预测结果f0(x)
2. 第二棵树学习残差 y - f0(x)，得到f1(x)
3. 第三棵树学习残差 y - f0(x) - f1(x)，得到f2(x)
4. 最终预测结果是所有树的和 F(x) = f0(x) + f1(x) + ... + fm(x)
```
**优点**：
- 效果比随机森林好
- 适合结构化数据，工业界竞赛首选
**缺点**：
- 串行训练，速度慢
- 容易过拟合，需要调参

## 3. 三大梯度提升框架对比

### 3.1 XGBoost (eXtreme Gradient Boosting)
**核心改进**：
1. 损失函数加入正则项，防止过拟合
2. 二阶泰勒展开，计算更准确
3. 特征粒度的并行计算，速度快
4. 缺失值自动处理
5. 内置交叉验证、早停

**参数调优**：
| 参数 | 作用 | 常用值 |
|------|------|--------|
| `max_depth` | 树的最大深度 | 3-8 |
| `learning_rate` | 学习率 | 0.01-0.3 |
| `n_estimators` | 树的数量 | 100-1000 |
| `subsample` | 样本采样比例 | 0.7-0.9 |
| `colsample_bytree` | 特征采样比例 | 0.7-0.9 |
| `reg_alpha` | L1正则 | 0-1 |
| `reg_lambda` | L2正则 | 1-10 |

### 3.2 LightGBM (Light Gradient Boosting Machine)
**核心改进**：
1. 基于直方图的分裂，速度快、内存占用小
2. 叶子生长策略 (leaf-wise)，比XGBoost的层生长更快
3. 类别特征自动处理，不需要one-hot编码
4. 支持并行训练、GPU加速
**优点**：比XGBoost速度快3-10倍，内存占用更小，效果差不多
**适用场景**：大数据集、工业界在线训练

### 3.3 CatBoost (Categorical Boosting)
**核心改进**：
1. 自动处理类别特征，不需要手动编码
2. 有序提升，减少过拟合
3. 对类别特征的处理效果比LightGBM更好
**适用场景**：类别特征多的数据集、不需要太多特征工程

## 4. 支持向量机 (SVM)

### 4.1 核心思想
> 找到一个超平面，最大化不同类别之间的间隔（Margin）

**间隔**：超平面到最近的样本的距离的两倍
**支持向量**：距离超平面最近的样本，决定了超平面的位置

### 4.2 核函数
| 核函数 | 表达式 | 适用场景 |
|------|------|----------|
| 线性核 | K(x,z) = x^T z | 线性可分、特征多、样本少 |
| 多项式核 | K(x,z) = (γ x^T z + r)^d | 特征少、样本中等 |
| RBF核 | K(x,z) = exp(-γ ||x-z||²) | 非线性、特征少、样本中等 |
| Sigmoid核 | K(x,z) = tanh(γ x^T z + r) | 接近神经网络 |

### 4.3 优缺点
**优点**：
- 高维特征效果好（比如文本分类）
- 泛化能力强
- 有理论保证
**缺点**：
- 样本量大时训练速度极慢（O(n²)复杂度）
- 核函数选择困难
- 可解释性差
**适用场景**：小样本高维、文本分类（TF-IDF特征）

## 5. 模型选择指南

| 场景 | 首选模型 | 备选模型 |
|------|----------|----------|
| 快速做基线、需要可解释性 | 逻辑回归 | 线性SVM |
| 结构化数据、效果优先 | LightGBM/XGBoost | 随机森林 |
| 类别特征很多 | CatBoost | LightGBM |
| 小样本高维、文本分类 | 线性SVM | 逻辑回归 |
| 需要快速预测、实时推理 | 逻辑回归、轻量树模型 | 深度模型 |
| 数据量极大 | LightGBM | 深度学习 |

## 工业界最佳实践
1. **优先用树模型**：结构化数据场景，XGBoost/LightGBM几乎总是最优选择
2. **不要盲目用深度学习**：结构化数据上树模型效果更好，速度更快，更易解释
3. **基线优先**：先跑逻辑回归/XGBoost默认参数，确定效果上限
4. **特征工程第一**：特征好，简单模型也能有好效果
5. **调参性价比**：最重要的参数是max_depth、learning_rate、n_estimators，优先调这三个，其他参数默认即可
