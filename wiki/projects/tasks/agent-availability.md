## 调度规则说明

### 并行执行规则

```
可以并行的条件：
1. 任务独立性：Task A 和 Task B 无直接或传递依赖关系
2. 资源可用性：分配给不同Agent，系统资源充足
3. 最大并行度：最多3个任务同时执行
```

### 串行执行规则

```
必须串行的条件：
1. 输出依赖：Task B 需要 Task A 的输出作为输入
2. 资源冲突：同一Agent的多任务
3. 资源不足：系统资源不足时
```

### 调度优先级规则

```
优先级排序：
1. P0任务优先
2. 截止时间最早
3. 依赖链关键路径上的任务
4. 创建时间最早
```
