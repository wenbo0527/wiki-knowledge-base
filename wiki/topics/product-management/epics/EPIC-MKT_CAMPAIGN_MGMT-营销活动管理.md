## 1. 产品总览

### 1.1 一句话定位
**企业级营销活动全生命周期管理平台，覆盖活动创建、执行追踪、效果分析和复盘总结，支撑营销团队高效运作。**

### 1.2 产品定位 & 核心价值

| 维度 | 描述 |
|:---|:---|
| **目标用户** | 营销运营人员、市场经理、活动策划、产品经理 |
| **核心痛点** | 活动分散难以统一管理、执行进度不透明、效果数据分散、复盘困难 |
| **解决方案** | 活动中心 + 流程规范 + 数据聚合 + 智能复盘 |
| **核心价值** | 活动管理效率提升 70%，活动执行透明度 100%，ROI 分析效率提升 80% |

## 2. 功能结构（按模块归档）

### 2.1 FEATURE-MKT_CAMPAIGN_PLAN（活动规划）

#### 功能入口 & 作用

**入口位置**：
- 营销活动 → 活动规划 → 新建活动
- 营销活动 → 活动日历

**核心作用**：
- 管理活动需求和提案
- 制定活动方案和排期
- 协调资源和预算
- 审批活动立项

#### 核心逻辑

```
活动规划流程：

Step 1: 需求收集
  - 业务部门提交活动需求
  - 填写活动目标、预期效果、资源需求
  - 选择活动类型和优先级

Step 2: 方案设计
  - 活动策划编写活动方案
  - 设计活动规则和流程
  - 预估活动效果和预算
  - 制定风险预案

Step 3: 资源协调
  - 协调研发、设计、运营资源
  - 确认权益和奖品准备
  - 确认推广渠道和预算
  - 确认客服和售后支持

Step 4: 立项审批
  - 提交活动立项申请
  - 各相关方审批
  - 审批通过后进入创建阶段
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---:|
| `campaign.approval_required` | 预算超过多少需要审批 | `10万` |
| `campaign.min_lead_time` | 活动最小筹备周期（天）| `7` |
| `campaign.max_concurrent` | 同时进行的活动数量上限 | `20` |

### 2.2 FEATURE-MKT_CAMPAIGN_CREATE（活动创建）

#### 功能入口 & 作用

**入口位置**：营销活动 → 活动管理 → 新建活动

**核心作用**：
- 创建和管理活动基础信息
- 配置活动规则和参与条件
- 关联权益和营销素材
- 设置活动预算和投放计划

#### 核心逻辑

```
活动创建流程：

Step 1: 基础信息配置
  - 填写活动名称和描述
  - 选择活动类型
  - 设置活动时间（预热/正式/返场）
  - 设置活动标签和分类

Step 2: 活动规则配置
  - 设置参与条件（人群、门槛）
  - 配置优惠规则（折扣、满减、赠品）
  - 设置参与限制（每人限参与次数）
  - 配置活动互斥规则

Step 3: 权益和预算
  - 关联权益中心的优惠券/积分
  - 设置活动预算上限
  - 配置预算消耗预警阈值
  - 设置退款和售后规则

Step 4: 投放配置
  - 配置推广渠道（App/小程序/公众号）
  - 设置推广素材
  - 配置触达策略（营销画布联动）
  - 设置活动页落地页
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---:|
| `campaign.max_discount` | 单笔订单最大优惠金额 | `200` |
| `campaign.participation_limit` | 单用户参与次数上限 | `3` |
| `campaign.budget_warning` | 预算消耗预警比例 | `80%` |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 客群中心 | 上游 | 活动目标人群 |
| 权益中心 | 上游 | 活动优惠券/积分 |
| 触达系统 | 下游 | 活动推广触达 |
| 营销画布 | 下游 | 活动自动化营销 |
| 订单系统 | 上游 | 活动订单数据 |

### 2.3 FEATURE-MKT_CAMPAIGN_EXECUTION（活动执行）

#### 功能入口 & 作用

**入口位置**：营销活动 → 活动监控 → 活动详情

**核心作用**：
- 实时监控活动执行状态
- 及时发现和处理异常
- 动态调整活动参数
- 保障活动平稳进行

#### 核心逻辑

```
活动执行流程：

Step 1: 活动预热
  - 活动页面上线
  - 推广引流启动
  - 用户预约提醒
  - 预热数据监控

Step 2: 活动进行
  - 实时数据监控（大屏）
  - 异常告警和处理
  - 动态调整（库存、价格）
  - 用户反馈收集

Step 3: 活动高峰
  - 流量峰值应对
  - 系统容量保障
  - 紧急预案执行
  - 实时决策调整

Step 4: 活动收尾
  - 活动准点结束
  - 库存清零
  - 订单处理完成
  - 活动下线
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---:|
| `campaign.realtime_interval` | 实时数据刷新间隔（秒）| `10` |
| `campaign.alert_threshold` | 告警阈值（异常订单比例）| `5%` |
| `campaign.graceful_end` | 活动准点结束 | `true` |

### 2.4 FEATURE-MKT_CAMPAIGN_ANALYSIS（活动分析）

#### 功能入口 & 作用

**入口位置**：营销活动 → 数据分析 → 活动报告

**核心作用**：
- 聚合活动相关数据
- 计算活动核心指标
- 生成活动效果报告
- 提供活动对比和优化建议

#### 核心逻辑

```
活动分析流程：

Step 1: 数据聚合
  - 汇总活动相关数据（曝光、点击、参与、转化）
  - 关联订单和交易数据
  - 关联触达和反馈数据
  - 数据清洗和对账

Step 2: 指标计算
  - 曝光量：活动页面被看到的次数
  - 点击量：活动页面被点击的次数
  - 参与量：用户参与活动的次数
  - 转化量：完成活动目标的次数
  - GMV：活动带来的成交金额
  - ROI：投入产出比

Step 3: 效果分析
  - 目标达成率分析
  - 各渠道效果对比
  - 各阶段转化漏斗
  - 用户画像分析

Step 4: 报告生成
  - 自动生成活动报告
  - 对比历史活动
  - 输出优化建议
  - 报告分享和归档
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---:|
| `analysis.attribution_window` | 归因时间窗口（天）| `7` |
| `analysis.auto_report` | 活动结束自动生成报告 | `true` |
| `analysis.retention_days` | 报告保留天数 | `365` |

## 4. 数据相关

### 4.1 核心指标口径

| 指标 | 定义 | 计算逻辑 | 目标值 |
|:---|:---|:---|:---:|
| **曝光量** | 活动页面被看到的次数 | `count(page_view)` | - |
| **点击量** | 活动页面被点击的次数 | `count(click)` | - |
| **点击率** | 点击量 / 曝光量的比例 | `click / view * 100%` | > 10% |
| **参与量** | 用户参与活动的次数 | `count(participation)` | - |
| **参与率** | 参与量 / 点击量的比例 | `participation / click * 100%` | > 30% |
| **转化量** | 达成活动目标的次数 | `count(conversion)` | - |
| **转化率** | 转化量 / 参与量的比例 | `conversion / participation * 100%` | > 5% |
| **活动 GMV** | 活动带来的成交金额 | `sum(order_amount)` | - |
| **活动 ROI** | 活动带来的 GMV / 活动成本 | `gmv / cost` | > 5x |
| **活动客单价** | 活动订单的平均金额 | `gmv / order_count` | - |

### 4.2 埋点事件字典

| 事件名 | 触发时机 | 携带参数 |
|:---|:---|:---|
| `campaign_view` | 用户浏览活动页 | campaign_id, user_id, source |
| `campaign_click` | 用户点击活动元素 | campaign_id, user_id, element_id |
| `campaign_participate` | 用户参与活动 | campaign_id, user_id, participation_type |
| `campaign_convert` | 用户完成活动目标 | campaign_id, user_id, conversion_type, amount |
| `campaign_share` | 用户分享活动 | campaign_id, user_id, share_channel |

## 7. 问题库 & FAQ

### 7.1 常见问题排查

#### Q: 活动数据对不上怎么办？

**排查步骤**：
1. 检查各系统数据同步是否正常
2. 核对时间口径是否一致
3. 检查埋点是否完整

**解决方案**：
- 触发数据重新同步
- 修正数据口径
- 补充缺失埋点

#### Q: 活动 ROI 计算不准确怎么办？

**排查步骤**：
1. 确认归因时间窗口设置
2. 检查成本数据是否完整
3. 核实 GMV 数据来源

**解决方案**：
- 调整归因窗口
- 补充成本数据
- 核实 GMV 数据

## 8. 方法论沉淀

### 8.1 活动策划方法论

**活动策划框架**：
1. 明确目标：拉新/促活/转化/品牌
2. 确定人群：精准定向目标用户
3. 设计玩法：有趣、有价值、易参与
4. 匹配资源：权益、预算、渠道
5. 预设复盘：提前定义成功指标

### 8.2 活动效果优化方法

**提升活动效果的关键**：
1. 精准人群：选择高意向目标用户
2. 吸引力：足够有吸引力的优惠和玩法
3. 便利性：参与路径简短、操作简单
4. 时机：选择用户最可能参与的时间
5. 追踪优化：实时监控，及时调整
