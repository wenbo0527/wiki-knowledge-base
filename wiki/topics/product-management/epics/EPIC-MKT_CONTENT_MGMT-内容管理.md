## 1. 产品总览

### 1.1 一句话定位
**统一管理数字营销内容资产，支持内容的创建、存储、检索、分发和效果分析，提升营销内容的复用和管理效率。**

### 1.2 产品定位 & 核心价值

| 维度 | 描述 |
|:---|:---|
| **目标用户** | 营销运营、内容编辑、设计师、数据分析师、营销管理员 |
| **核心痛点** | 营销内容分散难以复用、版本混乱、投放效率低、效果追踪难 |
| **解决方案** | 统一内容库 + 智能检索 + 多渠道分发 + 效果分析 |
| **核心价值** | 内容复用率提升 70%，内容产出效率提升 50%，投放效果提升 30% |

## 2. 功能结构（按模块归档）

### 2.1 FEATURE-MKT_CONTENT_REPOSITORY（内容仓库）

#### 功能入口 & 作用

**入口位置**：数字营销 → 内容中心 → 内容库

**核心作用**：
- 统一存储和管理营销内容资产
- 支持多类型内容（图片、视频、文档、模板）
- 内容分类和标签管理
- 内容版本控制

#### 核心逻辑

```
内容仓库结构：

内容中心
├── 图片素材
│   ├── Banner 图
│   ├── 产品图
│   └── 图标素材
├── 视频素材
│   ├── 宣传视频
│   └── 短视频
├── 文档素材
│   ├── 文案
│   └── 活动方案
└── 内容模板
    ├── 页面模板
    └── 组件模板
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---|
| `repo.max_file_size` | 单文件最大大小(MB) | `100` |
| `repo.allowed_types` | 允许的文件类型 | `jpg,png,mp4,pdf` |
| `repo.auto_tag` | 是否自动打标签 | `true` |
| `repo.version_enabled` | 版本控制开关 | `true` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **内容重复** | 相同内容重复上传 | 相似度检测、重复提醒 |
| **分类混乱** | 内容分类不清晰难以查找 | 统一分类标准、强制分类 |
| **版本冲突** | 多人编辑导致版本冲突 | 锁机制、版本覆盖保护 |
| **存储成本高** | 大文件占用大量存储 | 分级存储、定期归档清理 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 文件存储 | 依赖 | 内容文件存储 |
| 搜索服务 | 依赖 | 内容检索 |

### 2.2 FEATURE-MKT_CONTENT_SEARCH（内容检索）

#### 功能入口 & 作用

**入口位置**：数字营销 → 内容中心 → 搜索

**核心作用**：
- 多维度内容检索
- 智能推荐和相似内容
- 筛选和排序
- 批量操作

#### 核心逻辑

```
内容检索流程：

Step 1: 搜索意图理解
  - 关键词提取
  - 标签匹配
  - 分类筛选

Step 2: 检索执行
  - 全文检索（标题、描述）
  - 标签匹配
  - 内容特征检索（颜色、风格）
  - 语义相似度检索

Step 3: 结果处理
  - 相关性排序
  - 热度加权
  - 个性化推荐
  - 分页处理

Step 4: 预览和操作
  - 缩略图预览
  - 快速信息展示
  - 收藏和下载
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---|
| `search.min_keyword_length` | 最小搜索关键词 | `2` |
| `search.max_results` | 最大返回结果数 | `100` |
| `search.similarity_threshold` | 相似度阈值 | `0.7` |
| `search.cache_enabled` | 搜索缓存开关 | `true` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **搜索不准确** | 搜索结果不相关 | 优化搜索算法、增加标签 |
| **相似内容难找** | 相似素材难以发现 | 相似推荐、智能推荐 |
| **筛选条件多** | 筛选条件复杂难以配置 | 提供预设筛选模板 |
| **检索性能慢** | 大量内容检索卡顿 | 索引优化、缓存加速 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 内容仓库 | 依赖 | 检索数据来源 |
| 推荐服务 | 依赖 | 个性化推荐 |

### 2.3 FEATURE-MKT_CONTENT_DISTRIBUTION（内容分发）

#### 功能入口 & 作用

**入口位置**：数字营销 → 内容中心 → 分发管理

**核心作用**：
- 一站式多渠道内容分发
- 分发规则配置
- 分发状态跟踪
- 渠道效果汇总

#### 核心逻辑

```
内容分发流程：

Step 1: 选择内容
  - 选择要分发的内容
  - 支持批量选择
  - 检查内容完整性

Step 2: 选择渠道
  - App 客户端
  - Web 网站
  - 小程序
  - 第三方平台

Step 3: 配置分发规则
  - 分发时间（立即/定时）
  - 分发范围（全部/指定用户）
  - 优先级配置

Step 4: 执行分发
  - 内容预处理
  - 渠道 API 调用
  - 分发状态监控
  - 结果汇总
```

#### 关键配置项

| 配置项 | 说明 | 默认值 |
|:---|:---|:---|
| `dist.sync_mode` | 分发模式 | `SYNC` |
| `dist.retry_times` | 失败重试次数 | `3` |
| `dist.timeout` | 分发超时（秒）| `30` |
| `dist.batch_size` | 批量大小 | `100` |

#### 常见坑 & 注意点

| 坑点 | 问题描述 | 解决方案 |
|:---|:---|:---|
| **分发失败** | 部分渠道分发失败 | 自动重试、失败告警 |
| **内容格式不兼容** | 渠道不支持内容格式 | 格式转换工具、模板适配 |
| **分发延迟** | 内容到达渠道时间不一致 | 异步分发、状态追踪 |
| **版本不一致** | 各渠道内容版本不同步 | 版本管理、同步机制 |

#### 关联页面 / 依赖

| 关联项 | 类型 | 说明 |
|:---|:---:|:---|
| 内容仓库 | 依赖 | 分发内容来源 |
| 渠道 API | 依赖 | 各渠道接入 |
| 消息中心 | 依赖 | 分发通知 |

## 7. 问题库 & FAQ

### 7.1 常见问题排查

#### Q: 内容上传失败怎么办？

**排查步骤**：
1. 检查文件格式是否支持
2. 检查文件大小是否超限
3. 查看网络连接

**解决方案**：
- 转换文件格式
- 压缩文件大小
- 更换网络环境重试

#### Q: 搜索不到想要的内容怎么办？

**排查步骤**：
1. 尝试简化关键词
2. 使用分类筛选
3. 查看相似内容推荐

**解决方案**：
- 使用更通用的关键词
- 浏览分类目录
- 联系管理员补充内容

#### Q: 分发到某渠道失败怎么办？

**排查步骤**：
1. 查看分发错误日志
2. 检查渠道 API 状态
3. 确认内容格式是否兼容

**解决方案**：
- 重试分发
- 转换内容格式
- 联系渠道管理员

### 7.2 问题记录

| 日期 | 问题 | 原因 | 解决方案 | 预防措施 |
|:---|:---|:---|:---|:---|
| 2026-04-10 | 大文件上传超时 | 文件过大网络不稳定 | 分片上传、断点续传 | 限制文件大小 |
| 2026-04-05 | 搜索结果不准确 | 标签不完整 | 补充内容标签 | 强制标签审核 |
| 2026-03-28 | 渠道分发延迟 | 渠道 API 限流 | 增加重试机制 | 限流配置优化 |

```
- 智能检索

L4 智能期
- AI 智能推荐
- 内容自动生成
- 效果优化闭环
```

### 8.2 内容复用最佳实践

```
内容复用策略：

1. 内容规划
   - 建立内容模板库
   - 标准化组件设计
   - 模块化内容结构

2. 内容生产
   - 基于模板快速生产
```
