# 1. 产品总览

## 1.1 一句话定位

**构建企业级产品需求管理体系，实现从业务需求到技术实现的智能拆解、结构化管理和全流程追踪。**

## 1.2 产品定位 & 核心价值

| 维度 | 描述 |
|:---|:---|
| **目标用户** | 产品经理、技术负责人、项目经理、开发人员、测试人员 |
| **核心痛点** | 需求拆解不系统、协作不透明、知识难沉淀、进度难追踪 |
| **解决方案** | 四层架构（业务入口→AI拆解→本体语义→图向量层）+ 双存储（Neo4j+飞书） |
| **核心价值** | 需求可追踪、协作可视化、知识可沉淀、决策智能化 |

## 1.4 核心角色 & 使用人群

| 角色 | 核心职责 | 使用场景 | 高频功能 |
|:---|:---|:---|:---|
| **产品经理** | 需求提出、Epic定义、验收确认 | 拆解业务需求、追踪Story状态、确认验收 | Epic创建、Story查看、状态更新 |
| **技术负责人** | 技术评估、Story拆解、进度把控 | 拆解Feature为Story、评估工作量、跟踪进度 | Story创建、工作量评估、进度看板 |
| **开发人员** | Story实现、状态更新 | 查看分配的Story、更新实现状态、提交验收 | Story查看、状态更新、评论反馈 |
| **测试人员** | 验收测试、质量把控 | 查看验收标准、执行测试用例、反馈Bug | 验收标准查看、测试记录、Bug关联 |
| **项目经理** | 全局把控、风险预警、资源协调 | 查看整体进度、识别延期风险、协调资源 | 进度看板、风险预警、资源分配 |

## 1.5 关键约束

| 约束类型 | 具体约束 | 应对策略 | 责任方 |
|:---|:---|:---|:---:|
| **数据安全** | Epic/Story 涉及业务敏感信息 | 飞书表格权限控制、Neo4j 本地部署、敏感数据脱敏 | 技术负责人 |
| **性能要求** | 飞书 API 调用频率限制（100次/秒） | 批量同步、本地缓存、增量更新、异步处理 | 开发团队 |
| **数据一致性** | Neo4j ↔ 飞书表格双向同步 | 以 Neo4j 为准、定时同步、冲突检测、人工介入 | 产品经理 |
| **可扩展性** | 支持多产品域、多 Epic 并行（目标 100+ Epic） | 模块化设计、标准化流程、模板复用、自动化工具 | 架构师 |
| **合规要求** | 数据血缘、审计追踪、变更日志 | 变更日志、操作审计、版本管理、定期审计 | 项目经理 |
| **用户体验** | 飞书表格响应 < 2s、Neo4j 查询 < 500ms | 索引优化、缓存策略、分页加载、异步渲染 | 开发团队 |

## 1.6 术语词典（本体论级别）

| 术语 | 英文 | 定义 | 示例 | 层级 |
|:---|:---|:---|:---|:---:|
| **产品域** | Product Domain | 产品的高层级分类，代表一个完整的业务领域，包含多个 Epic | PD-DFD（数据发现）、PD-MKT（数字营销） | L1 |
| **Epic** | Epic | 产品域下的业务模块，代表一个可独立交付的价值单元，包含多个 Feature | EPIC-DFD_DATA_ASSET（数据资产字典） | L2 |
| **Feature** | Feature | Epic 下的功能模块，代表一个内聚的功能单元，包含多个 Story | FEATURE-DFD_ASSET_OPERATE_TOOL（资产运营工具） | L3 |
| **Story** | User Story | 用户可感知的完整功能点，代表一个可独立交付的用户价值，可在一个迭代内完成 | STORY-ASSET_LIST_PAGE（资产列表页） | L4 |
| **故事点** | Story Point | 相对估算单位，表示完成 Story 所需的工作量，用于评估团队产能 | 1, 2, 3, 5, 8, 13 | - |
| **验收标准** | Acceptance Criteria | Story 完成必须满足的条件，用于测试验证，通常包含功能、操作、数据、交互 4 项 | 功能正常、操作正常、数据准确、交互符合预期 | - |
| **飞书表格** | Feishu Bitable | 飞书多维表格，用于需求状态同步和团队协作，支持多视图、权限控制、自动化 | Epic拆解确认-数据资产字典 | - |
| **四层架构** | Four-Layer Architecture | 产品管理体系的架构分层：业务需求入口 → AI 需求拆解 → 本体论语义 → 图与向量层 | - | - |
| **双存储** | Dual Storage | Neo4j（核心数据）+ 飞书表格（协作同步）的存储架构 | - | - |
