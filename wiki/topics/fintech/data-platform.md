## 核心案例

### 案例1: 马上消费金融 - 腾讯云TBDS数据中台

| 属性 | 内容 |
|------|------|
| 公司 | 马上消费金融 |
| 行业 | 消费金融 |
| 解决方案 | 基于腾讯云TBDS的数据中台架构 |
| 关键技术 | TBDS, Hive, Spark, Kafka |
| 效果 | 日均10亿+数据量，PB级存储，5000+日处理任务 |

**核心组件**:
- **数据采集层**: Flume + Kafka，100TB/日

- **实时处理**: Flink + Kafka Streams，<100ms延迟
- **数据服务**: RESTful API + 数据门户

### 案例2: 招联金融 - Apache Doris统一实时数仓

| 属性 | 内容 |
|------|------|
| 公司 | 招联金融（招商银行+联通） |
| 行业 | 消费金融 |
| 解决方案 | Lambda架构 → Apache Doris统一数仓 |
| 关键技术 | Apache Doris, Flink, Kafka, MySQL |

**架构演进**:

| 演进前 (Lambda) | 演进后 (Doris统一) |
|-----------------|-------------------|
| Flink实时 + Spark离线双轨 | Flink + Doris统一处理 |
| Kafka + HDFS多套存储 | 统一存储层 |
| 代码维护成本高 | 单一代码库 |
| 数据不一致风险 | 数据一致性保障 |

## 关键架构模式

### 1. Lambda架构

```
┌─────────────────────────────────────────────────────┐
│                    应用层                            │
│        (实时应用 / 离线应用 / 即席查询)               │
└─────────────────────────────────────────────────────┘
                        │
        ┌───────────────┴───────────────┐
        ▼                               ▼
┌───────────────────┐       ┌───────────────────┐
│    实时流处理      │       │    离线批处理      │
│    (Flink)        │       │    (Spark)        │
│    <1秒延迟       │       │    小时级延迟      │
└─────────┬─────────┘       └─────────┬─────────┘
          │                           │
          ▼                           ▼
┌───────────────────┐       ┌───────────────────┐
│   Kafka集群        │       │   HDFS集群        │
│   实时消息队列     │       │   离线数据湖       │
└───────────────────┘       └───────────────────┘
```

**适用场景**: 需要同时支持实时和离线分析
**代表**: 马上消费金融

### 2. Kappa架构

**适用场景**: 纯实时场景，无需离线批处理
**代表**: 招联金融演进方向

### 3. 统一实时数仓 (Apache Doris)

```
┌─────────────────────────────────────────────────────┐
│                    应用层                            │
│         (实时分析 / 交互查询 / 报表)                  │
└─────────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────┐
│              Apache Doris 统一层                     │
│         (实时写入 / 批量导入 / 查询引擎)             │
└─────────────────────────────────────────────────────┘
                        │
        ┌───────────────┴───────────────┐
        ▼                               ▼
┌───────────────────┐       ┌───────────────────┐
│    Flink实时      │       │    数据湖/OSS     │
│    数据写入       │       │    批量数据        │
└───────────────────┘       └───────────────────┘
```

## 关键技术栈

### 数据采集

| 技术 | 特点 | 适用场景 |
|------|------|----------|
| **Flume** | 分布式日志采集 | 日志、埋点数据 |
| **Kafka** | 高吞吐消息队列 | 实时数据管道 |
| **DataX** | 离线同步 | 异构数据库同步 |
| **Canal** | MySQL binlog同步 | 业务库实时同步 |

### 数据存储

| 技术 | 特点 | 适用场景 |
|------|------|----------|
| **HDFS** | PB级分布式存储 | 离线数仓 |
| **Hive** | SQL on Hadoop | 离线分析 |
| **Apache Doris** | 实时+OLAP | 统一实时数仓 |
| **ClickHouse** | OLAP引擎 | 实时分析 |
| **Iceberg/Hudi** | 数据湖格式 | 湖仓一体 |

### 计算引擎

| 技术 | 特点 | 适用场景 |
|------|------|----------|
| **Spark** | 通用计算引擎 | 离线批处理、SQL |
| **Flink** | 实时流处理 | 实时计算、流批一体 |
| **Presto** | 交互查询 | 跨源联邦查询 |

## 关键发现

### 发现1: OneData方法论是数据中台建设核心

**核心观点**: 统一数据建模、指标定义、口径是数据中台的基础。

**关键数据**:
- 阿里OneData支撑10万+数据表统一管理
- 数据开发效率提升50%
- 数据口径一致性从60%→95%+

**实施建议**:
1. 引入OneData方法论
2. 建设数据资产平台（数据地图、血缘、质量）
3. 建立统一数据服务层
4. 培养数据治理团队

---

### 发现2: 实时数仓是高级阶段

从T+1离线 → 准实时 → 秒级实时

**技术演进**:
- Lambda → Kappa → 统一数仓
- 多套存储 → 统一存储
- 双代码库 → 单一代码库
