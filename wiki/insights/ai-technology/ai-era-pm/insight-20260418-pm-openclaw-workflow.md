## 📌 核心洞察

### 龙虾11人阵型中的PM角色

**11人Agent团队配置**：

| Agent | PM使用场景 | 价值 |
|-------|------------|------|
| 📝 文档撰写 | PRD、周报生成 | 10x效率 |
| 🔍 信息收集 | 竞品分析、行业研究 | 自动化 |
| 📊 数据分析 | 数据整理、报告 | 快速洞察 |
| 🎨 原型助手 | UI描述、交互设计 | 降低门槛 |
| 💬 对外沟通 | 客户邮件、汇报 | 节省时间 |

## PM的OpenClaw工作流

```
┌─────────────────────────────────────────────────────────────┐
│  任务输入                                                    │
│  "帮我分析竞品A的产品功能"                                   │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│  Agent分发                                                  │
│  - 信息收集Agent → 抓取竞品数据                             │
│  - 分析Agent → 整理归纳                                     │
│  - 报告Agent → 生成分析报告                                 │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│  PM审核                                                     │
│  - 验证信息准确性                                           │
│  - 补充人工判断                                             │
│  - 调整结论                                                 │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│  交付输出                                                    │
│  高质量竞品分析报告                                          │
└─────────────────────────────────────────────────────────────┘
```

## PM使用龙虾的注意事项

### ✅ 适合的场景

| 场景 | PM价值 |
|------|--------|
| 日报/周报生成 | 节省1-2小时 |
| 信息收集整理 | 快速全面 |
| 文档模板化 | 批量生产 |
| 竞品初步分析 | 快速了解 |

### ❌ 不适合的场景

| 场景 | 原因 |
|------|------|
| 深度用户访谈 | 需要人际交互 |
| 复杂决策判断 | 需要业务经验 |
| 敏感信息处理 | 安全风险 |
| 创意生成 | AI缺乏原创性 |
