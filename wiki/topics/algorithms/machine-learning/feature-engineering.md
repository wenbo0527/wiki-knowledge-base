## 特征工程是什么？

**定义**：将原始数据转化为适合模型训练的特征的过程，是机器学习项目中耗时最长（通常占60%-80%时间）、影响效果最大的环节。

```
原始数据 → 特征工程 → 模型训练 → 评估上线
```

---

## 整体流程

```
1. 数据清洗 → 处理缺失值、异常值、重复值
2. 特征构造 → 从原始数据中衍生新特征
3. 特征转换 → 标准化、归一化、编码、离散化
4. 特征选择 → 选择有用的特征，去除冗余和无关特征
5. 特征监控 → 上线后监控特征分布变化
```

## 1. 数据清洗

### 1.1 缺失值处理
| 方法 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| **删除** | 缺失率<5%，样本量足够 | 简单 | 丢失信息，可能引入偏差 |
| **填充固定值** | 缺失有业务含义（如缺失代表0/无） | 简单 | 可能引入偏差 |
| **填充统计值** | 数值型特征：均值/中位数；类别型：众数 | 简单 | 没有考虑特征之间的关系 |
| **模型预测填充** | 缺失率较高（10%-30%） | 更准确 | 复杂，可能引入过拟合 |
| **特殊值标记+填充** | 缺失本身有信息（如用户没填某个字段代表某种特征） | 保留缺失信息 | 增加特征维度 |

**最佳实践**：
- 数值型特征优先用中位数填充（比均值对异常值鲁棒）
- 类别型特征加一个"Unknown"类别
- 缺失率>50%的特征直接删除，除非业务上非常重要

### 1.2 异常值处理
**识别方法**：
- 3σ原则：数值超过均值±3倍标准差的是异常值
- IQR原则：小于Q1-1.5*IQR或大于Q3+1.5*IQR的是异常值（IQR是四分位距=Q3-Q1）
- 业务规则：根据业务常识判断（如年龄>120、消费金额负数）

**处理方法**：
- 删除：异常值占比<1%，确认是错误数据
- 修正：替换为最大值/最小值/分位数
- 保留：异常值本身有业务含义（如大额消费对应高价值用户）
- 分箱：将连续值离散化，减少异常值影响

### 1.3 重复值处理
- 完全重复的样本直接删除
- 主键重复的样本保留最新的一条

## 2. 特征构造

### 2.1 统计特征
**时间窗口统计**：针对时序数据，计算过去N天/周/月的统计值
| 统计类型 | 例子 |
|------|------|
| 基础统计 | 近7天消费金额的均值、最大值、最小值、中位数、标准差、和 |
| 计数统计 | 近30天登录次数、消费次数、浏览次数 |
| 比率统计 | 近7天活跃天数/30天活跃天数、消费金额/收入、逾期次数/借款次数 |
| 趋势统计 | 近7天消费金额的环比增长率、同比增长率、斜率 |

**用户画像特征**：
- 基础属性：年龄、性别、城市、等级、注册时长
- 行为特征：浏览、点击、购买、收藏、分享的频次、偏好
- 消费特征：消费金额、消费频次、客单价、消费品类偏好
- 风险特征：逾期次数、逾期金额、风险评级

### 2.2 交叉特征
**定义**：两个或多个特征组合得到的新特征
- 类别+类别：性别+年龄段、城市+职业
- 类别+数值：年龄段+平均消费金额、城市+消费水平
- 数值+数值：消费金额/收入、浏览时长/活跃天数

**例子**：
- 25岁男性这个组合的消费能力可能远高于单独的年龄和性别特征
- 一线城市高收入这个组合的风险偏好可能远高于单独的城市和收入特征

### 2.3 业务规则特征
**定义**：根据业务经验人工构造的特征
- 风控领域：是否多头借贷、是否在黑名单、近3个月查询次数
- 电商领域：是否在大促期间消费、是否是会员、是否有优惠券
- 社交领域：是否是互关好友、共同好友数、是否在同一个群

**最佳实践**：
- 多和业务人员聊，挖掘业务经验中的有效规则
- 特征的业务含义越明确，效果越稳定，可解释性越好

## 3. 特征转换

### 3.1 数值型特征
| 方法 | 公式 | 适用场景 | 对模型的影响 |
|------|------|----------|--------------|
| **标准化 (Z-score)** | (x-mean)/std | 线性模型、SVM、神经网络、PCA | 必须做，否则特征尺度差异大，模型训练不稳定 |
| **归一化 (Min-Max)** | (x-min)/(max-min) | 神经网络、需要特征值在[0,1]之间的场景 | 对异常值敏感，适合取值范围固定的特征 |
| **对数变换** | log(x+1) | 长尾分布的特征（如消费金额、收入） | 将长尾分布压缩成近似正态分布，提高线性模型效果 |
| **平方根变换** | sqrt(x) | 右偏分布的特征 | 比对数变换温和 |
| **Box-Cox变换** | | 正态化变换 | 让特征更接近正态分布，提高线性模型效果 |

### 3.2 类别型特征
| 方法 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| **Label Encoding (标签编码)** | 类别有序（如等级：低/中/高） | 不增加维度 | 无序类别会引入错误的大小关系 |
| **One-Hot Encoding (独热编码)** | 类别无序，类别数少（<10） | 简单，效果稳定 | 类别数多会维度爆炸，特征稀疏 |
| **Target Encoding (目标编码)** | 类别数多（>10），有监督场景 | 不增加维度，保留类别信息 | 容易过拟合，需要做交叉验证平滑 |
| **Embedding (嵌入)** | 类别数极多（如用户ID、商品ID），深度学习场景 | 低维稠密，保留语义信息 | 需要大量数据训练 |

### 3.3 离散化/分箱
**定义**：将连续型数值分成若干个区间，每个区间作为一个类别
**方法**：
1. 等宽分箱：每个区间宽度相同
2. 等频分箱：每个区间样本数相同
3. 卡方分箱：有监督分箱，根据目标变量的相关性分箱
4. 业务分箱：根据业务经验分箱（如年龄：0-18/18-30/30-45/45+）

**优点**：
- 减少异常值的影响，提高模型鲁棒性
- 引入非线性，提高模型效果
- 可解释性好
**最佳实践**：
- 树模型不需要分箱，线性模型/神经网络可以分箱
- 优先用业务分箱，其次用卡方分箱，最后用等频分箱
- 分箱数控制在5-20个之间，不要太多也不要太少

## 4. 特征选择

**目标**：去除无关特征、冗余特征，减少维度，提高训练速度，降低过拟合风险

### 4.1 过滤法 (Filter)
**核心思想**：根据特征和目标变量的相关性选择特征，不依赖模型
| 方法 | 适用场景 | 说明 |
|------|----------|------|
| 方差选择 | 所有场景 | 去掉方差小于阈值的特征（方差为0的特征完全没用） |
| 互信息 | 分类任务 | 衡量特征和目标变量之间的互信息，越大越重要 |
| 相关系数 | 回归任务 | 衡量特征和目标变量之间的皮尔逊相关系数，绝对值越大越重要 |
| 卡方检验 | 分类任务，类别特征 | 衡量特征和目标变量之间的独立性，卡方值越大越相关 |

**优点**：速度快，不依赖模型，不会过拟合
**缺点**：没有考虑特征之间的相互作用，效果一般

### 4.2 包裹法 (Wrapper)
**核心思想**：用模型的效果来选择特征子集
| 方法 | 说明 |
|------|------|
| 递归特征消除 (RFE) | 每次去掉最不重要的特征，直到达到目标特征数 |
| 正向选择 | 每次加入一个效果最好的特征，直到效果不再提升 |
| 反向消除 | 每次去掉一个效果最差的特征，直到效果不再下降 |

**优点**：考虑特征之间的相互作用，效果好
**缺点**：计算速度慢，容易过拟合

### 4.3 嵌入法 (Embedded)
**核心思想**：模型训练过程中自动进行特征选择
| 方法 | 适用场景 | 说明 |
|------|----------|------|
| L1正则 | 线性模型 | L1正则会让不重要的特征的系数变成0，相当于自动选择特征 |
| 树模型特征重要性 | 树模型（随机森林、XGBoost等） | 训练后输出特征重要性，选择重要性高的特征 |
| 注意力机制 | 深度学习模型 | 模型自动学习特征的权重 |

**优点**：速度快，效果好，和模型训练结合紧密
**缺点**：依赖模型，特征重要性可能不稳定

### 4.4 最佳实践流程
```
1. 先做过滤法，快速去掉明显没用的特征（方差为0、和目标变量完全不相关）
2. 再用嵌入法，用树模型得到特征重要性，去掉重要性极低的特征
3. 最后根据需要用包裹法做更精细的特征选择
4. 特征数控制在几百到几千之间，不要太多
```

## 5. 工业界避坑指南

### 常见错误
1. **数据泄露**：用了未来的数据构造特征，导致线下效果好，线上效果差
   - 反例：用整个数据集的均值填充缺失值，而不是只用训练集的均值
   - 反例：构造特征时用到了目标变量的信息
   - 避免方法：特征工程只在训练集上做，验证集和测试集用训练集的统计值
2. **特征和目标变量强相关但业务上是因果倒置**
   - 反例：预测用户是否逾期，特征用了"是否催收"，而催收是逾期之后才会发生的
   - 避免方法：每个特征都要确认业务上的时间顺序，特征必须发生在目标变量之前
3. **特征分布线上线下不一致**
   - 现象：线下测试AUC很高，上线后效果暴跌
   - 原因：线上特征和线下训练时的特征分布差异很大
   - 避免方法：上线前对比特征的线上线下分布，上线后持续监控特征分布
4. **特征冗余**：多个特征表达同一个含义，增加训练时间，降低可解释性
   - 避免方法：计算特征之间的相关系数，去掉相关系数>0.9的冗余特征

### 特征监控
上线后必须监控的指标：
1. 特征缺失率
2. 特征均值、方差、分位数
3. 类别特征的各个类别的占比
4. 特征重要性的变化
5. 特征和目标变量的相关性变化

---

## 总结
> 好的特征应该满足：
> 1. 有明确的业务含义
> 2. 和目标变量有较强的相关性
> 3. 特征分布线上线下一致，随时间变化小
> 4. 特征之间的相关性低，冗余小
> 5. 覆盖大多数样本，缺失率低
