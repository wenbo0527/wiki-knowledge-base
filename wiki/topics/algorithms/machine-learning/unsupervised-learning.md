## 无监督学习是什么？

**定义**：在没有标签的数据中，自动发现数据的内在结构和模式。

**核心优势**：
- 不需要标注数据，标注成本高的场景下价值大
- 可以发现人类不知道的隐藏模式
- 是很多AI应用的基础（推荐、用户画像、异常检测）

**常见任务**：
| 任务 | 目标 | 典型场景 |
|------|------|----------|
| **聚类** | 将相似样本分到同一组 | 用户分群、客户画像、商品分类 |
| **降维** | 减少特征维度，保留主要信息 | 可视化、特征预处理、加速训练 |
| **异常检测** | 识别和正常样本不同的异常点 | 风控欺诈检测、运维故障监控 |
| **关联规则挖掘** | 发现变量之间的关联关系 | 购物篮分析、推荐系统 |

## 1. 聚类算法

### 1.1 K-Means
**核心思想**：将数据分为K个簇，簇内样本相似，簇间样本差异大
```
算法步骤：
1. 随机选择K个初始质心
2. 每个样本分配到最近的质心
3. 更新每个簇的质心为簇内样本的均值
4. 重复2-3步直到质心不再变化或达到最大迭代次数
```
**优点**：
- 简单、速度快、可解释性好
- 适合大规模数据集
**缺点**：
- 需要提前指定K值
- 对初始质心敏感，容易陷入局部最优
- 对异常值敏感
- 不适合非球形簇、密度差异大的簇
**调参要点**：
- K值选择：肘部法、轮廓系数
- 初始质心：用K-Means++优化初始选择
- 距离度量：常用欧氏距离，也可以用余弦距离

### 1.2 DBSCAN (Density-Based Spatial Clustering of Applications with Noise)
**核心思想**：基于密度的聚类，簇是密度相连的点的集合
```
核心概念：
- 核心点：半径ε内有至少MinPts个邻居的点
- 边界点：在核心点ε内，但自身不是核心点
- 噪声点：既不是核心点也不是边界点的点
```
**优点**：
- 不需要指定K值
- 可以发现任意形状的簇
- 自动识别噪声点（异常值）
- 对异常值不敏感
**缺点**：
- 参数ε和MinPts难调
- 密度差异大的簇效果不好
- 高维数据上效果不好
**适用场景**：空间数据、异常检测、任意形状的簇

### 1.3 层次聚类 (Hierarchical Clustering)
**核心思想**：按层次逐层合并或分裂簇，形成树状结构
```
两种类型：
1. 凝聚式（自底向上）：每个点初始为一个簇，逐层合并最近的簇
2. 分裂式（自顶向下）：所有点初始为一个簇，逐层分裂
```
**优点**：
- 不需要指定K值，可以根据树状图选择合适的层次
- 结果可视化好（树状图）
**缺点**：
- 计算复杂度高O(n²)，不适合大数据集
- 对异常值敏感
- 无法回溯，合并错了不能改
**适用场景**：小数据集、需要可视化聚类层次的场景

### 1.4 高斯混合模型 (GMM, Gaussian Mixture Model)
**核心思想**：假设数据由多个高斯分布混合而成，每个簇对应一个高斯分布
```
算法步骤：
1. 初始化K个高斯分布的参数（均值、方差、权重）
2. E步：计算每个样本属于每个高斯分布的概率
3. M步：更新高斯分布的参数
4. 重复2-3步直到收敛
```
**优点**：
- 输出是概率，软聚类，可以得到样本属于每个簇的概率
- 适合不规则形状的簇
- 可以处理重叠的簇
**缺点**：
- 需要指定K值
- 对初始值敏感，容易陷入局部最优
- 计算速度比K-Means慢
**适用场景**：需要概率输出、簇有重叠的场景

## 2. 降维算法

### 2.1 PCA (Principal Component Analysis, 主成分分析)
**核心思想**：线性降维，找到方差最大的方向作为主成分，保留最重要的信息
```
算法步骤：
1. 数据标准化，均值为0，方差为1
2. 计算协方差矩阵
3. 计算协方差矩阵的特征值和特征向量
4. 选择前K个最大特征值对应的特征向量作为主成分
5. 将原始数据投影到K个主成分上，得到降维后的数据
```
**优点**：
- 简单、速度快、可解释性好
- 无监督，不需要标签
- 广泛用于特征预处理、可视化
**缺点**：
- 线性降维，无法处理非线性关系
- 降维后可解释性下降（主成分是原始特征的线性组合）
- 对异常值敏感
**调参要点**：
- 保留的主成分数：通常选择累计方差贡献80%-95%的主成分数
- 标准化：一定要做，否则特征尺度不同会影响结果

### 2.2 t-SNE (t-Distributed Stochastic Neighbor Embedding)
**核心思想**：非线性降维，将高维空间中相似的点在低维空间中也保持接近，适合可视化
```
核心思想：
- 高维空间中相似的点的概率分布用高斯分布表示
- 低维空间中相似的点的概率分布用t分布表示
- 最小化两个分布之间的KL散度
```
**优点**：
- 可视化效果极好，能保留局部结构
- 适合发现数据的聚类结构
**缺点**：
- 计算速度慢，O(n²)复杂度，不适合大数据集
- 结果不稳定，每次运行结果可能不同
- 全局结构保留不好
- 只能用于可视化，不能用于训练前的特征降维
**适用场景**：高维数据可视化、探索性数据分析

### 2.3 UMAP (Uniform Manifold Approximation and Projection)
**核心思想**：新型非线性降维，结合了t-SNE的优点，同时保留全局结构
**优点**：
- 比t-SNE速度快，适合大数据集
- 同时保留局部结构和全局结构
- 结果更稳定
**适用场景**：大数据集可视化、降维预处理

## 3. 异常检测算法

### 3.1 孤立森林 (Isolation Forest)
**核心思想**：异常点更容易被孤立，通过随机分裂树的方式，异常点的路径长度更短
```
算法步骤：
1. 随机选择特征和分裂值，分裂节点
2. 重复直到每个节点只有一个样本
3. 异常点的路径长度更短（更少的分裂就能孤立）
4. 计算每个样本的异常得分
```
**优点**：
- 速度快，适合大规模数据集
- 不需要假设数据分布
- 对高维数据效果好
- 不需要标注数据
**缺点**：
- 对参数敏感
- 密集型异常检测效果不好
**适用场景**：大规模高维数据异常检测、风控欺诈检测

### 3.2 局部异常因子 (LOF, Local Outlier Factor)
**核心思想**：异常点的局部密度比邻居低
```
核心概念：
- 可达距离：点p到点o的距离，至少是o的核心距离
- 局部可达密度：点p的邻居的平均可达距离的倒数
- LOF得分：点p的局部可达密度与邻居的局部可达密度的比值，大于1说明异常
```
**优点**：
- 适合检测局部异常（和周围邻居不同的点）
- 不需要假设数据分布
**缺点**：
- 计算速度慢O(n²)，不适合大数据集
- 对参数k敏感
**适用场景**：小数据集、局部异常检测

## 4. 关联规则挖掘

### Apriori算法
**核心思想**：挖掘频繁项集，找到变量之间的关联规则
```
核心指标：
- 支持度(Support)：P(A∩B)，同时包含A和B的样本比例
- 置信度(Confidence)：P(B|A)，包含A的样本中也包含B的比例
- 提升度(Lift)：P(B|A)/P(B)，A对B的提升程度，大于1才有意义
```
**典型应用**：购物篮分析
- 例子：{啤酒} → {尿布}，支持度2%，置信度60%，提升度3
- 解释：2%的订单同时买啤酒和尿布，买啤酒的人60%会买尿布，买了啤酒的人买尿布的概率是普通用户的3倍

## 5. 工业界最佳实践

### 聚类落地流程
```
1. 业务理解 → 明确聚类目标、如何评价聚类效果
2. 特征工程 → 选择相关特征，标准化/归一化
3. 算法选择 → 先试K-Means，效果不好再试DBSCAN/GMM
4. 参数调优 → 选择合适的K值/参数
5. 聚类解释 → 给每个簇起业务名字，解释簇的特征
6. 落地应用 → 用于用户分层、营销、个性化推荐等
7. 迭代更新 → 定期更新聚类模型，适应数据变化
```

### 常见问题
| 问题 | 解决方案 |
|------|----------|
| K值怎么选？ | 肘部法+业务理解，优先选符合业务认知的K值 |
| 特征太多怎么办？ | 先PCA降维，再聚类 |
| 聚类结果不好解释怎么办？ | 每个簇取Top特征，和业务对齐，调整特征和K值 |
| 数据更新快怎么办？ | 定期更新模型，用流式聚类算法 |

### 算法选择指南
| 场景 | 首选算法 | 备选算法 |
|------|----------|----------|
| 大规模数据集，球形簇，需要快 | K-Means | Mini Batch K-Means |
| 任意形状的簇，需要检测异常 | DBSCAN | HDBSCAN |
| 需要概率输出，簇重叠 | GMM |  |
| 特征降维，预处理 | PCA |  |
| 高维数据可视化 | t-SNE（小数据）/ UMAP（大数据） |  |
| 大规模异常检测 | 孤立森林 |  |
| 小数据集，局部异常 | LOF |  |
| 购物篮分析，关联规则 | Apriori | FP-Growth |
