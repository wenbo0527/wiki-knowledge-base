## 📋 文档结构标准

### 每个主题的标准结构

```
专题名称/
├── 01_搜索任务.md              ← 搜索任务清单
├── 02_收集内容.md              ← 核心内容收集（4000+字）
├── 03_行业洞察.md              ← 行业趋势分析（2000+字）
├── 04_行动建议.md              ← 落地建议（1500+字）
├── 05_参考资料.md              ← 参考链接、数据来源
└── 目录.md                     ← 章节导航
```

### 文档内容要求

| 文档 | 字数要求 | 核心要素 |
|------|----------|----------|
| 01_搜索任务 | 500字 | 搜索关键词、任务清单、完成状态 |
| 02_收集内容 | 4000+字 | 具体案例、数据、对比分析 |
| 03_行业洞察 | 2000+字 | 趋势判断、监管影响、竞争格局 |
| 04_行动建议 | 1500+字 | 短期/中期/长期建议、技术选型 |
| 05_参考资料 | 300字 | 参考链接、数据来源更新时间 |

### 信息增益要求

每个主题需包含：

1. **具体案例** - 企业名称、具体数字、实施效果
2. **时间维度** - 标注信息更新时间，区分历史与最新
3. **对比分析** - 不同方案优缺点对比
4. **可操作性** - 具体的实施步骤、工具选型
